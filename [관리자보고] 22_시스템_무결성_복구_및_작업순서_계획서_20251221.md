# 📋 BitWish Network 시스템 무결성 복구 및 작업 순서 계획서
**작성일시:** 2025-12-21 20:00 (KST)
**대상 시스템:** BitWish Network Mainnet (Vultr Server)

## 1. 개요
현재 도메인 배포본(`bitwishnetwork.com`)에서 발생하는 지갑 인증 실패 및 버튼 비활성화 문제를 해결하고, 관리자 지갑을 포함한 전체 4개 지갑이 어떤 환경에서든 완벽하게 작동하도록 복구하는 것을 목표로 합니다.

## 2. 현상 분석 (장애 원인)
- **로컬 데이터 의존성:** 현재 코드가 서버 데이터보다 브라우저 로컬 스토리지를 우선하여, 새 환경 접속 시 데이터 미일치로 인증이 거부됨.
- **경로 및 프로토콜 문제:** HTTPS 도입 이후 API 호출 경로(/api)가 일부 하드코딩된 값과 충돌하여 데이터 동기화가 끊김.
- **UI 로직 오류:** 시드 문구 검증 시 브라우저 내 지갑 정보 유무를 먼저 체크하여 버튼이 활성화되지 않음.

## 3. 단계별 복구 순서 (Work Order)

### [단계 1] 로컬 검증 및 서버 데이터 무결성 확인
1.  **포트 브라우저 검증:** 현재 작동하는 포트(IP:PORT) 환경에서 4개 지갑의 접근 권한 및 데이터(BW 잔액, 추천인)가 정상인지 최종 확인.
2.  **서버 데이터 백업:** 서버의 `referrals.json` 파일을 현재 시점 기준으로 즉시 백업하여 데이터 유실 원천 차단.

### [단계 2] 핵심 로직 전면 수정 (개발 서버 적용)
1.  **서버 우선 인증 시스템 적용:** 
    - `WalletService.ts`: 시드 문구 입력 시 브라우저가 아닌 서버 API를 먼저 호출하여 지갑 주소 등록 여부 확인.
    - `MiningAuthModal.tsx`: 2차 비밀번호 검증을 서버 데이터 해시값과 직접 대조하도록 수정.
2.  **자동 복구(Restore) 기능 강화:** 인증 성공 시 서버의 최신 데이터를 브라우저 로컬 스토리지에 즉시 동기화하여 '현장 복구' 완료.
3.  **UI/UX 개선:** 24단어 형식 일치 시 버튼 즉시 활성화 및 명확한 오류 메시지(ID/PW 불일치 등) 출력.

### [단계 3] 포트 테스트 및 통합 검증
1.  **4개 지갑 전수 테스트:** 관리자 지갑 및 나머지 3개 지갑이 각각 다른 브라우저에서 시드 문구만으로 복구되는지 확인.
2.  **기능 점검:** 마이닝 시작/종료, 실시간 보상 계산, 추천인 코드 생성 기능이 서버 데이터와 동기화되는지 확인.

### [단계 4] 도메인 배포 및 실환경 적용
1.  **빌드 및 배포:** 수정된 코드를 빌드하여 Vultr 서버의 Nginx 경로로 배포.
2.  **HTTPS 최종 점검:** SSL 환경에서 모든 API 호출이 `/api` 경로를 통해 보안 연결되는지 확인.
3.  **최종 보고:** 4개 지갑 모두의 접속 성공 여부를 관리자에게 보고 후 작업 종료.

## 4. 향후 관리 지침
- 모든 중요 수정은 포트 환경에서 선검증 후 도메인에 반영한다.
- 데이터와 관련된 모든 로직은 '서버 데이터'를 최우선 순위(Source of Truth)로 둔다.
- 작업 완료 후 반드시 시스템 무결성 백업 폴더를 생성한다.
