## 1. 마이닝 진입 및 인증 브릿지 (Authentication Bridge)

사용자가 지갑 내부 혹은 홈페이지에서 '마이닝 시작' 버튼을 클릭하는 순간, 시스템은 다음의 보안 검증 회로를 가동합니다.

### 1.1 스피드 패스(Speed Pass) 세션 검크
- **로직:** `walletService.checkAuthSession()` 호출.
- **처리:** 최근 10분 이내에 니모닉 인증을 통과한 기록이 있다면, 별도의 인증 없이 즉시 마이닝 게이트웨이를 개방합니다.

### 1.2 니모닉 파편 보안 인증 (Mnemonic Fragment Auth)
- **발동 조건:** 세션 만료 혹은 미인증 상태.
- **로직:** 24단어 중 무작위로 추출된 4~6개의 단어 위치를 사용자에게 제시하고 입력을 요구합니다.
- **검증:** 입력값의 **SHA-512 지문(Fingerprint)**을 로컬 저장소와 대조하여 100% 일치 시에만 마이닝 세션을 생성합니다.

## 2. 마이닝 페이지 초기화 및 복구 시스템 (Initialization & Recovery)

마이닝 모달(`MiningStatusModal`)이 열리는 순간, 시스템은 과거의 기록을 추적하여 연속성을 확보합니다.

### 2.1 서버 데이터 직결 (State Restoration)
- `apiService.getUserStatus`를 통해 서버(MongoDB)에 저장된 사용자 상태를 호출합니다.
- **주요 항목:** `isMining`(채굴 중 여부), `miningStartTime`(채굴 시작 시점), `accumulatedReward`(기존 채굴량).

### 2.2 시간 연속성 보강 (Time Continuity)
- **공식:** `현재시간 - miningStartTime = 총 채굴 시간(초)`
- 페이지가 닫혀 있거나 네트워크가 끊겼던 시간 동안의 채굴량을 서버의 시작 시간을 기준으로 역산하여 UI에 실시간 복구합니다.

---

## 3. 초정밀 마이닝 티커 엔진 (Real-time Mining Engine)

`RealTimeSyncService` 클래스 내부에 탑재된 본 엔진은 마이닝의 심장입니다.

### 3.1 초당 증분 연산 (Per-Second Ticker)
- **메인 채굴:** `현재 채굴량 + (최종 채굴율 / 3600)`
- **추천 보너스:** `보너스 저장소 + (기본 채굴율 * 추천 보너스율 / 3600)`
- **정밀도:** 모든 연산은 `Decimal.js`를 사용하여 소수점 50자리 정밀도로 수행되며, 이진 부동 소수점 오차를 원칙적으로 차단합니다.

---

## 4. 보너스 멀티플라이어 결합 시스템 (Bonus Multiplier System)

기본 채굴량(0.25 BW/hr)에 가산되는 보너스들은 다음의 수학적 결합 과정을 거칩니다.

- **결합 공식:** `기본(1.0) + 출석보너스(0.05) + 추천인보너스(n%) + 파트너보너스(0.25)`
- **출석 시스템:** `AttendanceModal`을 통해 24시간마다 체크인 시 5%의 추가 가중치가 즉시 멀티플라이어에 합산됩니다.
- **10초 스위퍼:** `ReferralBonusService`가 10초마다 가동되어 하위 유저의 활동량을 검사하고 보너스 보관함을 업데이트합니다.

---

## 5. 데이터 무결성 동기화 회로 (Data Synchronization Circuit)

브라우저 환경의 불안정성을 극복하기 위해 하이브리드 동기화 전략을 사용합니다.

- **30초 주기 박제 (Syncing):** 로컬에서 실시간 연산된 채굴량 데이터를 30초마다 서버 API(`syncMiningData`)로 전송하여 DB를 갱신합니다.
- **네트워크 감지:** `networkStatus` 시스템이 서버와의 연결 상태를 1초마다 감시하며, 끊김 발생 시 UI 단에 즉시 경고 상태를 표시합니다.

## 6. 마이닝 페이지 내부 UI/UX 시스템 (Internal UI Systems)

- **정밀도 제어:** 잔액 표시기(`RewardBox`)는 UI 뒤틀림 방지를 위해 4자리만 표시하되, 마우스 호버 시 **8자리 원본 데이터**를 툴팁으로 제공합니다.
- **세션 관리:** 상단 'Logout' 버튼 클릭 시 `bw_mining_auth` 세션을 즉시 파기하여 타인이 채굴 상태에 접근하는 것을 물리적으로 차단합니다.
- **실시간 통계 카드:** 기본율, 최대 보상률, 추천인 수, 보관함 현황을 1초 주기로 리렌더링하여 사용자에게 '살아있는 네트워크' 경험을 제공합니다.

## 7. 마이닝 페이지 전체 UI 및 버튼 기능 인벤토리 (Functional Inventory)

본 마닝 페이지의 모든 시각적 요소와 버튼은 다음의 독립적인 코드 로직과 1:1로 매핑되어 작동합니다.

### 7.1 상단 내비게이션 그리드 (4-Button System)
- **🗓️ 출석체크 (Attendance):** 
    - **기능:** 오전 09:00 ~ 익일 08:59:59 기준의 출석 로직 가동.
    - **효과:** 체크인 성공 시 채굴 멀티플라이어에 **0.05(5%) 가속** 즉시 합산.
- **👥 추천인 (Referral):** 
    - **기능:** 하위 가입자의 지갑 주소, 가입일, KYC 상태 리스트업.
    - **텍스트 로직:** 1 BW 지급(`is1BWMePaid`) 및 2% 보너스 정산(`is2PercentMePaid`) 현황을 실시간 테이블로 출력.
- **🏢 파트너 (Partner):** 기업/기관 승인 상태(`partnerStatus`)를 확인하여 **25% 추가 가속** 상태를 관리자 승인과 연동.
- **🔒 프로필 (Profile):** 지갑의 2차 비밀번호 설정 및 계정 보안 레벨 제어 게이트웨이.

### 7.2 마이닝 상태 대시보드 (8-Status Indicators)
- **기본율/최대율 텍스트:** 현재 본인의 시간당 순수 채굴량과 보너스가 포함된 한계 생산력을 8자리 정밀도로 출력.
- **출석/파트너 상태:** `userStats` 상태값을 기반으로 ON/OFF 및 미등록/등록 상태를 실리콘 밸리 표준 색상 코드로 표출.
- **보상 보관함(Storage):** `referralBonusStorage`와 `referralRewardStorage` 텍스트가 1초 단위로 갱신되며 미확정 자산의 가치를 실시간 브리핑.

### 7.3 하단 실행 제어 시스템 (Footer Controls)
- **⚡ 마이닝 시작:** `handleStartMining` 클릭 시 서버에 `isMining` 신호를 전송하고 UI의 **'번개(lightning)'** 애니메이션을 즉시 활성화.
- **🔑 나의 지갑:** 현재 마이닝 세션을 유지한 채 지갑 관리 모드로 전환하는 **세션 유지형 브릿지**.
- **❌ 닫기:** `onClose` 호출 시 UI만 종료하고 백그라운드의 **`RealTimeSyncService`** 티커는 1초마다 계속 채굴을 수행하게 하여 사용자 편의성 제공.
- **Logout (Header):** `bw_mining_auth` 키를 로컬에서 즉시 말소하여 보안 세션을 강제 종료하는 비상 탈출 기능.

---
**최종 결론:** BitWish 마이닝 시작 시스템은 **UI 상의 단 한 줄의 텍스트, 단 하나의 버튼**조차 독립적인 금융 로직과 서버 5001번 포트의 DB에 직결되어 있는, 완벽하게 설계된 정밀 채굴 시스템입니다.

## 8. 니모닉 파편 보안 인증 모달 시스템 (Mnemonic Fragment Authentication System)

홈페이지 '마이닝 시작' 버튼 클릭 시, 보안 세션이 만료되었거나 캐시가 소멸된 환경(기기 변경 포함)에서 발동되는 비상 보안 게이트웨이입니다. 본 시스템은 사용자의 자산을 물리적으로 보호하기 위한 최후의 보루입니다.

### 8.1 2단계 정밀 인증 프로세스 (Dual-Stage Verification)
- **1단계: 지갑 주소 식별 (Address Verification)**
    - **로직:** 사용자가 입력한 BW 주소의 문자열 지문이 백업된 주소와 소수점 단위까지 일치하는지 대조.
    - **UI:** 주소 입력 전까지는 2단계(단어 입력) 구역을 비활성화하여 **무차별 대입 공격(Brute-force)**을 원천 차단.
- **2단계: 니모닉 파편 무작위 검증 (Fragment Input)**
    - **로직:** 24단어 중 시스템이 매번 무작위로 생성한 4~6개의 특정 인덱스(Word #n)를 요구.
    - **통신:** 서버 호출 없이 `WalletService` 내부의 로컬 정밀 검증 알고리즘을 사용하여 보안성을 극대화.

### 8.2 4개국어 초정밀 어순 치환 엔진 (Dynamic i18n V2 Engine)
- **기술명:** `{index}` 변수 기반 문법 보정 시스템.
- **언어별 문법 대응:**
    - **KO:** `{index}번 단어:` (한국어 특유의 후치사 구조 준수)
    - **EN:** `Word #{index}` (영어의 전치사 구조 및 직관성 확보)
    - **JA/ZH:** 각 국 언어의 고유한 수량사 및 명칭(`番目`, `号`)을 변수와 결합하여 자연스러운 안내 문구 출력.
- **성능:** `useMemo` 기반의 `LanguageManager` 인스턴스 고정을 통해 언어 전환 시 UI 지연 시간 0ms 달성.

### 8.3 보안 및 테마 무결성 (Security & Theme Integrity)
- **독립 저장소 원칙:** 모든 번역 및 인증 로직은 '개별 유저의 데이터 격리' 철학에 따라 중앙 상태 관리 없이 독립적으로 작동.
- **Premium UI Persistence:** 다국어 변경 시에도 기존의 **실리콘 밸리 표준 다크 모드** 테마와 애니메이션(Fade-in, Slide-up) 레이아웃을 100% 보존.
- **세션 지휘:** 인증 성공 시 `bw_mining_auth` 세션을 생성하여 10분간의 '스피드 패스' 권한을 부여하며, 실패 시에는 접근을 즉시 차단하고 에러 메시지를 다국어로 브리핑.
