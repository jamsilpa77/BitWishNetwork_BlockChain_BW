## 2. 핵심 기능적 결함 (Critical Functional Issues)

사용자께서 지적하신 **추천인 시스템 관련 데이터 불일치 및 로직 미반영** 문제는 다음과 같은 원인으로 추정되며, 다음 세션의 최우선 해결 과제입니다.

### 2.1 추천 보상 및 보너스 보관함 불일치
*   **현상:** "나의 지갑", 마이닝 페이지의 "추천보상보관함", "추천보너스보관함"의 수치가 서로 다름.
*   **원인 분석:**
    1.  **데이터 소스 파편화:** 각 컴포넌트(`WalletModal`, `MiningPage`)가 참조하는 상태(State)나 API 엔드포인트가 서로 다를 가능성이 높음.
    2.  **동기화 누락:** 추천인이 가입하는 순간 발생하는 이벤트가 실시간으로 모든 보관함 데이터에 전파되지 않음.
    3.  **계산 로직 중복:** 프론트엔드에서 계산하는 로직과 백엔드에서 저장하는 로직이 서로 다른 수식(소수점 처리 등)을 사용하고 있을 가능성.

### 2.2 추천인 가입 시 보상률(2%) 영구 반영 실패
*   **현상:** 가입자 1명당 기본 시간당/일일 보상률이 2%씩 즉시, 영구적으로 증가해야 하나 반영되지 않음.
*   **원인 분석:**

    2.  **DB 스키마 미비:** `User` 또는 `MiningRecord` 스키마에 "현재 적용된 보상률"을 저장하는 필드가 없거나 업데이트되지 않음.
    3.  **즉시성 부재:** 추천인 가입 트랜잭션 발생 시, 해당 유저의 마이닝 세션을 즉시 갱신하는 트리거가 백엔드에 구현되지 않음.

## 3. 반복된 오류 및 실패 원인 분석 (Root Cause Analysis of Repetitive Failures)

어제와 오늘 작업에서 동일한 오류가 반복되고 해결이 지연된 근본적인 원인은 다음과 같습니다.

### 3.1 파일 손상 및 인코딩 문제 (File Corruption)
*   **원인:** 대량의 파일 수정을 위해 사용한 PowerShell 스크립트가 UTF-8 인코딩을 명시적으로 처리하지 않거나, 정규식(Regex)이 너무 광범위하게 적용됨.
*   **결과:**
    *   한글 주석이 깨지거나 엉뚱한 문자로 변환됨 (`✅ 모든` -> `✅ ./../모든`).
    *   이로 인해 "Unterminated string literal" 같은 문법 오류가 발생하여 빌드 자체가 불가능해짐.
    *   **교훈:** 향후 대량 수정 시 스크립트 사용을 지양하고, 반드시 IDE의 안전한 리팩토링 기능을 사용하거나 파일 단위로 검증하며 수정해야 함.

### 3.2 빌드 도구 및 환경 설정의 혼선 (Build System Confusion)
*   **원인:** `react-scripts` (CRA 기본), `react-app-rewired` (설정 오버라이드), `webpack` (직접 설정) 간의 의존성 충돌.
*   **결과:**
    *   Node.js 폴리필(`crypto`, `buffer` 등)이 누락되어 Webpack 5에서 빌드 실패.
    *   `config-overrides.js`가 제대로 로드되지 않아 경로 별칭(`@/`)을 인식하지 못함.
    *   해결책을 찾는 과정에서 도구를 계속 변경하다 보니 설정 파일(`package.json`)이 꼬임.

### 3.3 포트 충돌 및 좀비 프로세스 (Port Conflicts)
*   **원인:** 서버 종료가 정상적으로 되지 않아 백그라운드에 `node.exe` 프로세스가 포트(5000, 5001)를 점유.
*   **결과:** 코드가 정상이어도 "EADDRINUSE" 오류로 서버가 켜지지 않는 현상을 코드 오류로 착각하여 불필요한 수정 반복.
