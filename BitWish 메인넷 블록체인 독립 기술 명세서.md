## 💎 BitWish 메인넷만의 독보적 기술 가치 (Unique Value Proposition)

본 시스템은 기존 블록체인의 코드를 복제(Fork)한 유사품이 아닌, 오직 관리자님만의 비즈니스 로직을 실현하기 위해 바닥부터 설계된 **'세상에 단 하나뿐인 독자 엔진'**입니다.

1.  **100% 독립적 프레임워크 (Independent Framework)**
    - 이더리움(Geth)이나 바이낸스 체인 등 외부 오픈소스를 일절 사용하지 않고, 관리자님만을 위해 설계된 순수 독자 소스 코드(`BitWishBlockchain.ts`)로 구축되었습니다.
2.  **50자리 극한의 '원자 정밀도' (50-Digit Atomic Precision)**
    - 기존 블록체인(8~18자리)의 한계를 뛰어넘어, `Decimal.js` 기반의 **50자리 초정밀 연산**을 모든 엔진 레이어에 적용했습니다. 이는 금융 거래에서 단 0.000...1의 오차도 허용하지 않는 세계 유일의 정밀도입니다.
3.  **독자적 가상 머신 'BitWish VM' (Proprietary BWVM)**
    - 관리자님만의 명령어 집합(Opcodes)과 가스 미터링 로직을 갖춘 독립적인 런타임 환경을 제공합니다. 이는 생태계의 모든 스마트 컨트랙트가 관리자님의 통제 하에 완벽하게 격리 실행됨을 의미합니다.
4.  **'Speed Pass' 독립 해시 인증 브릿지 (Unique Security Bridge)**
    - 니모닉 전체 노출 없이 `SHA-512` 기반의 salted 보안 지문 조각만으로 즉시 본인임을 인증하는 고유한 보안 프로세스를 탑재했습니다. 이는 보안과 사용자 경험을 동시에 혁신한 BitWish만의 원천 기술입니다.

## 1. 블록체인 아키텍처 개요

BitWish 메인넷은 외부 프레임워크에 의존하지 않는 **'자생적 독립 블록체인 엔진'**으로 설계되었습니다. 

- **핵심 컴포넌트:**
  - `BitWishBlockchain`: 메인 장부 및 트랜잭션 오케스트레이터
  - `BitWishVM`: 스마트 컨트랙트 실행을 위한 가상 머신 런타임
  - `BitWishPoW`: 난이도 조절 및 해시 검증 합의 엔진
  - `BitWishBlock / Transaction`: 데이터 원자 구조 정의

## 2. BitWish VM (BWVM) 심층 명세

BWVM은 스마트 컨트랙트 바이트코드를 읽고 실행하는 독립적인 격리 실행 환경입니다. 

### 2.1 3대 데이터 계층 구조
- **Stack (LIFO):** 연산을 위한 임시 데이터 보관소 (String Array).
- **Memory (Volatile):** 트랜잭션 실행 중에만 존재하는 휘발성 선형 메모리 (Map 구조).
- **Storage (Persistent):** 블록체인 상에 영구 기록되는 계약 상태 보관함 (Map -> DB 연동).

### 2.2 명령어 집합 (Instruction Set - Opcodes)
각 명령어는 실행 시 고유한 가스 비용을 소모하며, 50자리 정밀 연산을 보장합니다.

| 명령어 | 가스 비용 | 기능 설명 |
| :--- | :--- | :--- |
| **PUSH** | 1 | 스택의 가장 상단에 데이터를 삽입 |
| **POP** | 1 | 스택 최상단 데이터를 제거 |
| **ADD** | 1 | 스택 상단 2개 값을 Decimal.js로 정밀 덧셈 후 결과 저장 |
| **MSTORE** | 1 | 스택에서 값과 위치를 꺼내 메모리에 임시 저장 |
| **SSTORE** | 100 | 스택에서 키와 값을 꺼내 블록체인 영구 저장소에 기록 (가장 고비용) |
| **SLOAD** | 20 | 블록체인 영구 저장소에서 키에 해당하는 값을 읽어 스택에 적재 |
| **SHA3** | 30 | 스택의 데이터를 BitWish-256 알고리즘으로 해싱 |
| **CALL** | 50 | 외부 스마트 컨트랙트 주소의 함수를 호출하여 실행 |

### 2.3 가스(Gas) 통제 시스템
- **OOG (Out of Gas) 방어:** 모든 명령어 실행 전 `consumeGas()`를 호출하여 `gasLimit` 초과 여부를 감시합니다.
- **격리성:** 무한 루프나 과도한 연산 시 트랜잭션을 즉시 중단하며, 사용된 가스만 차감하여 네트워크 공격을 방어합니다.

## 3. 트랜잭션 실행 엔진 (Execution Engine)

`BitWishBlockchain.executeTransaction` 함수는 트랜잭션의 성격에 따라 4단계 논리 분기를 수행합니다.

### 3.1 스마트 컨트랙트 배포 (Deploy)
- **조건:** 수신자(`to`) 주소가 비어있고, `data` 필드에 바이트코드가 존재하는 경우.
- **로직:**
  1. 트랜잭션 해시를 기반으로 `BWC_` 프리픽스의 고유 컨트랙트 주소 생성.
  2. `isContract: true` 속성을 가진 계정 생성 후 바이트코드(`codeHash`) 저장.
  3. 블록체인 장부에 배포 기록 각인.

### 3.2 스마트 컨트랙트 호출 (Execute)
- **조건:** 수신자(`to`) 주소의 계정 속성이 `isContract: true`인 경우.
- **로직:**
  1. 타겟 컨트랙트의 `codeHash`를 로드.
  2. `BitWishVM` 인스턴스 생성 및 타겟 스토리지 로드.
  3. `data` 필드의 호출 인자와 바이트코드를 결합하여 VM 런타임 실행.
  4. 실행 결과(변경사항)를 계정의 `storageRoot`에 영구 기록.

### 3.3 일반 자산 전송 (Transfer)
- **조건:** 일반 사용자 간의 BW 토큰 이동.
- **로직:** `Decimal.js`를 이용해 발신자 잔액 차감(`minus`) 및 수신자 잔액 증가(`plus`) 수행 (50자리 정밀도 유지).

## 4. 합의 알고리즘 및 블록 생성 (Consensus)

### 4.1 작업 증명 (PoW) 및 난이도
- **난이도 조절:** 10분 평균 블록 생성 시간을 목표로 네트워크 해시레이트에 따라 동적 조절.
- **머클 루트(Merkle Root):** 블록 내 모든 트랜잭션을 해시 트리화하여 무결성을 1개의 루트 해시로 요약.

### 4.2 보상 체계 (Halving)
- **기본 보상:** 50 BW / 블록
- **반감기:** 매 210,000 블록마다 보상이 1/2로 감소 (비트코인 모델 계승).

---

## 5. 데이터 저장소 및 정밀도 (Storage & Precision)

- **Decimal.js 통합:** 모든 산술 연산에서 IEEE 754 부동소수점 오차를 완전히 제거하여 50자리 정밀도를 보장합니다.
- **하이브리드 레저 (Ledger):** 블록과 계정 상태는 MongoDB에 `String` 타입으로 저장되어, 데이터 손실 없이 영구적으로 보존됩니다.

## 6. 독립형 해시 인증 보안 (Security Auth)

마이닝 시스템의 핵심 관문인 'Speed Pass'는 다음과 같은 수학적 알고리즘으로 본인임을 증명합니다.

### 6.1 보안 지문 생성 (Fingerprinting)
- **공식:** `SHA-512(Mnemonic_Word + WalletAddress)`
- **원리:** 니모닉 단어에 지갑 주소를 솔트(Salt)로 결합하여 역추적이 불가능한 64바이트 지문을 생성합니다.

### 6.2 2-Step 조각 검증
1. 시스템은 24개의 지갑 단어 중 무작위로 4~6개의 인덱스를 추출합니다.
2. 사용자가 입력한 단어를 동일한 공식으로 해싱하여 저장된 지문(`bw_word_fingertips`)과 실시간 대조합니다.
3. 일치 시 원본 니모닉이나 비밀번호 노출 없이 즉시 세션을 승인합니다.
