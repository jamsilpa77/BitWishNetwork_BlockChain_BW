## 1. 현재 구조 및 데이터 흐름 (Current Architecture & Flow)

### 1-1. 전체 흐름도
`HomePage` (중앙 관제탑)가 모든 상태(`AuthenticatedAddress`, Modals)를 관리합니다.
하지만 제가 수정한 "프리패스" 경로에서 **데이터(주소)를 누락**시키는 치명적 실수를 범했습니다.

| 단계 | 컴포넌트 | 동작 설명 | 데이터 상태 (Wallet Address) | 상태 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 진입** | `MyWalletModal` | 사용자가 [나의 지갑]을 보고 있음. (`walletAddress` 보유 중) | **보유 중** (예: `BW...123`) | 정상 |
| **2. 클릭** | `MyWalletModal` | **[⛏️ 마이닝 시작]** 버튼 클릭 (`onOpenMining` 호출) | **전송 안 함 (NULL)** | **[단절]** |
| **3. 수신** | `HomePage` | `onOpenMining` 이벤트 수신 → `setIsMiningModalOpen(true)` | **수신 못 함 (NULL)** | **[단절]** |
| **4. 오픈** | `MiningStatusModal` | 열리면서 `props.walletAddress`를 참조 | **'Not Connected' (빈 값)** | **[오류]** |
| **5. 파생** | `ReferralModal` | 마이닝창에서 추천창 열기 | **'Not Connected' (빈 값)** | **[오류]** |

### 1-2. 코드 레벨 분석 (Code-Level Analysis)

#### (1) `MyWalletModal.tsx` (발신지)
```typescript
// 현재 코드 (문제점)
onClick={() => {
    if (onOpenMining) {
        onClose();

    }
    // ...
}}
```

#### (2) `HomePage.tsx` (중계지)
```typescript
// 현재 코드 (문제점)
<MyWalletModal
    onOpenMining={() => { // <--- 받는 인자가 없음
        setIsMyWalletModalOpen(false);
        setIsMiningModalOpen(true);
        // setAuthenticatedAddress(...) 가 빠져있음!
    }}
/>
```

## 2. 해결 설계 (Solution Design)

단절된 데이터 파이프라인을 복구하기 위해 다음과 같이 코드를 수정해야 합니다.

### 2-1. `MyWalletModal.tsx` 수정
1.  **Interface 변경:** `onOpenMining: () => void` → `onOpenMining: (address: string) => void`
2.  **Call 변경:** `onOpenMining()` → `onOpenMining(walletAddress)`
3.  **UI 정리:** `logout-btn` 아래에 잘못 남겨둔 `2차 비번 초기화` 버튼 삭제. (파란 헤더에 있는 것은 유지)

### 2-2. `HomePage.tsx` 수정
1.  **Handler 변경:**
    ```typescript
    onOpenMining={(address) => { // 주소를 받아서
        setAuthenticatedAddress(address); // 시스템에 "이 사람이 로그인했다"고 선포
        setIsMyWalletModalOpen(false);
        setIsMiningModalOpen(true);
    }}
    ```

## 3. 검증 시나리오 (Validation Scenario)
수정 후 다음과 같이 검증하겠습니다.

1.  **로그인:** [나의 지갑] 열기.
2.  **클릭:** 파란 헤더 우측 하단 [⛏️ 마이닝 시작] 버튼 클릭.
3.  **마이닝 모달:** 상단 "인증된 지갑 주소"에 `BW...` 주소가 뜨는지 확인.
4.  **추천 모달:** 추천 코드가 `Loading...`이 아니라 정상 코드(예: `REF...`)가 뜨는지 확인.
5.  **UI:** 로그아웃 버튼 밑에 뜬금없는 주황색 버튼이 사라졌는지 확인.
