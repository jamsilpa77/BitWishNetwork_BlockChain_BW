## 1. 현상 파악 및 직접 원인 (Root Cause Analysis)

관리자님이 보내주신 스크린샷 3장은 제가 저지른 **"알맹이 없는 껍데기 작업"**의 증거입니다.

### 현상 ①: 로그아웃 버튼 밑에 또 2차 비번 초기화 버튼이 있음 (이미지 1)
*   **원인:** 제가 UI 배치를 수정할 때, **"기존 버튼을 삭제하고 옮겼어야 했는데"**, 기존 버튼은 그대로 두고 새 버튼만 추가했습니다. 즉, 버튼이 두 개가 중복되어 UI가 엉망이 되었습니다.
*   **파일:** `MyWalletModal.tsx`

### 현상 ②: 마이닝 페이지에 "인증된 지갑 주소: Not Connected" 표기 (이미지 2)
*   **원인:** [마이닝 시작] 버튼을 클릭하면 `setIsMiningModalOpen(true)`만 실행하여 **"창"만 열었을 뿐**, **"누구의 지갑인지(walletAddress)"를 `MiningStatusModal`에 전달하지 않았습니다.**
    *   `HomePage.tsx`에서 `AuthenticatedAddress` 상태값은 업데이트되지 않은 채, 껍데기 모달만 열린 것입니다.
    *   그래서 마이닝 모달은 "난 누가 들어왔는지 모르는데?" 하며 `Not Connected`를 출력했습니다.

### 현상 ③: 추천 보너스 모달 "Loading..." (이미지 3)
*   **원인:** 위와 동일합니다. 지갑 주소(Address)가 `ReferralModal`로 전달되지 않았기 때문에, 추천 코드를 조회할 기준값이 없어 무한 로딩에 걸린 것입니다.

## 2. 왜 이런 병신 같은 실수를 했는가? (Self-Reflection)
*   **UI 중심적 사고:** "버튼을 만들어라"는 지시에만 매몰되어, 그 버튼이 눌렸을 때 **"데이터(지갑 주소)가 함께 흘러가야 한다"**는 프로그래밍의 기본 원칙을 망각했습니다.
*   **검증 소홀:** "창이 열린다"는 것만 확인하고, "데이터가 채워졌는지"는 확인하지 않았습니다.

---

## 3. 해결 방안 (Action Plan)

### 단계 1: UI 중복 제거 (`MyWalletModal.tsx`)
*   로그아웃 버튼 아래에 남아있는 **"2차 비번 초기화" 버튼 제거**. (파란 헤더로 옮긴 것만 남김)

### 단계 2: 데이터 파이프라인 연결 (`HomePage.tsx`)
*   `MyWalletModal`에서 `onOpenMining` 호출 시, **현재 조회 중인 `walletAddress`를 인자로 함께 전달**하도록 수정.
*   `HomePage`에서는 이 주소를 받아 `setAuthenticatedAddress(address)`를 실행하여, 시스템 전체에 **"이 사람이 로그인했다"**고 공표.
    *   이렇게 해야 `MiningStatusModal`과 `ReferralModal`이 주소를 인식하고 정상 데이터를 출력함.

### 단계 3: 호출 시그니처 변경 (`MyWalletModal.tsx`)
*   `onOpenMining: () => void`  
    **→** `onOpenMining: (address: string) => void` 로 변경하여 주소 전달 의무화.
