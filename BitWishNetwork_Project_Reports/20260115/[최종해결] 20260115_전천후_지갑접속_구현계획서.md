**작성일시:** 2026년 1월 15일 17:15
**작성자:** AI Assistant (Antigravity)
**수신자:** BitWishNetwork 최고 관리자
**주제:** "어디서든, 어떤 지갑이든 즉시 접속"을 위한 시스템 완전 정상화 방안

---

## 1. 핵심 목표 (Objective)
1.  **신규 지갑:** 생성 즉시 인증 절차 없이 마이닝 페이지로 자동 진입 (One-Stop Flow).
2.  **기존 지갑 & 타 지갑:** "지갑 가져오기" 절차 생략. **인증 창에 주소+비번만 넣으면 로컬/서버 어디에 있든 찾아서 로그인 시켜줌 (Anywhere Access).**
3.  **데이터 무결성:** 로컬 데이터가 없거나 깨져도, 서버 데이터를 믿고 자동으로 로컬을 복구해줌.

---

## 2. 구현 방법 및 로직 (Technical Solution)

### 과제 ①: 신규 지갑 생성 후 자동 진입 (Auto-Login Pipeline)

**현재 문제:** 지갑만 만들고 UI 연결 고리가 끊겨있음.
**해결 방안:** `HomePage.tsx`와 `CreateWalletModal.tsx`의 연결 강화.

1.  **`CreateWalletModal` 수정:**
    *   지갑 생성 완료 시, 단순 종료가 아니라 **생성된 지갑 주소**를 부모(`HomePage`)에게 전달.
    *   `onComplete(newAddress)` 형태로 변경.
2.  **`HomePage` 수정:**
    *   `onComplete` 콜백을 받으면:
        *   `setAuthenticatedAddress(newAddress)` (인증 상태 확정)
        *   `setIsCreateWalletModalOpen(false)` (생성 창 닫기)
        *   `setIsMiningModalOpen(true)` (마이닝 페이지 즉시 열기)
    *   중간에 "나의 지갑"을 거치거나 "인증 모달"을 띄우는 불필요한 단계 삭제.

### 과제 ②: "어디서든 접속" (Anywhere Access Authentication)

**현재 문제:** 로컬 스토리지에 데이터가 없으면 "일치하지 않음" 에러 내고 차단함.
**해결 방안:** `WalletService.ts`의 `verifySecondPassword`를 **"전지전능한 검증기"**로 업그레이드.

1.  **로직 변경 (하이브리드 검증):**
    *   **1단계 (로컬):** 내 브라우저에 지갑이 있고 비번이 맞으면 → **OK**.
    *   **2단계 (서버 - 핵심):** 로컬에 없거나 비번이 틀리면 → **서버에 물어봄.**
        *   서버에 해당 주소의 `secondPasswordHash`가 있는가?
        *   입력한 비번이 그 해시와 일치하는가?
        *   **일치하면?** → **"서버 인증 성공"**으로 간주하고, 로그인 처리. (동시에 서버 데이터를 로컬로 가져와서 자동 복구 수행)

이렇게 하면, PC방이든 핸드폰이든, **[주소 + 비번]**만 알면 별도의 복구 절차 없이 바로 로그인이 됩니다.
