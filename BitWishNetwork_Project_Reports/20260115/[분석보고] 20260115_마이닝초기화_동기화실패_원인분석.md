# [분석보고] 20260115_마이닝초기화_동기화실패_원인분석

**작성일시:** 2026년 1월 15일
**작성자:** AI Assistant (Antigravity)
**수신자:** BitWishNetwork 최고 관리자
**주제:** 관리자 초기화 실행 후 마이닝 페이지 미반영 원인 규명

---

## 1. 문제 상황
*   관리자 페이지에서 `마이닝 데이터 초기화` 실행 -> **"성공" 메시지 출력.** (DB는 `0`으로 바뀜)
*   그러나 마이닝 페이지(모달)는 여전히 **1023시간 진행 중**이며 누적 보상도 그대로임.

## 2. 원인 분석 (Root Cause)
**"한 손은 박수를 쳤는데, 다른 손이 가만히 있는 상황"**

1.  **Transmitter (송신자):** `AdminPage.tsx`
    *   초기화 후 `localStorage.setItem('BW_SYSTEM_RESET_TRIGGER', ...)`를 통해 "초기화됐다!"는 신호를 보냈습니다. (확인됨)

2.  **Receiver (수신자):** `MiningStatusModal.tsx`
    *   **문제점:** 이 신호를 받는 `window.addEventListener('storage', ...)` 리스너 로직이 **누락되어 있습니다.**
    *   따라서 마이닝 모달은 초기화 사실을 모르고, **자기가 기억하고 있는 로컬 데이터(캐시)나 메모리 상의 숫자를 계속 보여줍니다.**

3.  **Local Storage (잔존 데이터):**
    *   `ReferralBonusService` 등이 사용하는 로컬 데이터(`bw_referral_storage_...`)가 브라우저에 그대로 남아 있습니다.
    *   모달이 닫혔다 열리거나 할 때 이 잔존 데이터를 다시 불러와서 "1023시간"을 다시 보여주는 악순환이 반복됩니다.

## 3. 해결 방안 (Solution Plan)
마이닝 모달(`MiningStatusModal.tsx`)에 **"초기화 신호 감지기"**를 달아야 합니다.

**[구현해야 할 로직]**
1.  모달이 열려있을 때 `BW_SYSTEM_RESET_TRIGGER` 신호가 감지되면:
    *   즉시 모든 상태(`miningTime`, `accumulatedReward`)를 `0`으로 리셋.
    *   `isMining`을 `false`로 변경.
    *   `apiService.getUserStatus`를 재호출하여 **(이미 0이 된) 서버 DB 데이터**를 다시 가져옴.
    *   **중요:** 브라우저에 남은 `localStorage` 찌꺼기 데이터도 싹 지움.

## 4. 결론
지금 서버 DB는 깨끗해졌을 가능성이 높습니다. (페이지를 강력 새로고침하거나 캐시를 지우면 0이 될 수도 있음)
하지만 사용자 경험(UX)상 **"초기화 버튼 누르자마자 0이 되는"** 모습을 보려면, 프론트엔드에 **신호 수신 코드(Listener)**를 추가해야 합니다.

**대기 중:** 관리자님의 지시가 있으면 즉시 수신 코드를 작성할 준비가 되어 있습니다.
