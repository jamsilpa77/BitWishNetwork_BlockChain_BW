## 2. 1:1 마이그레이션 및 지갑 가용 금액 전송 분석 (지시사항 2 ~ 2-1)

### 💰 [지갑 시스템 분석]
- **1:1 교환 로직:** `WalletService.ts` 내에 `balance`(포인트/총액)와 `availableBalance`(사용 가능 코인) 필드가 이미 정의되어 있어, 1:1 가치 환산 기술 구현이 가능합니다.
- **15일 유예 기간 로직:** KYC 승인 시점으로부터 `createdAt`과 연동된 `locking` 로직을 통해 15일 후 자동 마이그레이션되는 배치(Batch) 로직을 적용하기에 최적화된 필드 구조를 가지고 있습니다.
- **P2P 거래 연동:** `availableBalance`로 이동된 코인은 블록체인 네트워크상의 실제 전송 함수를 통해 유저 간 실시간 P2P 거래가 가능한 구조임이 확인되었습니다.

## 3. 익스플로러 연동 및 블록 투명성 분석 (지시사항 2-2, 3)

### 🌐 [익스플로러 가시성 확보]
- **트랜잭션(Tx) 기록 (2-2):** 포인트가 실제 블록체인 코인으로 마이그레이션될 때, 시스템은 이를 **네트워크 트랜잭션(Transaction)**으로 생성합니다. 따라서 익스플로러의 트랜잭션 리스트에 해당 내역이 TXID와 함께 투명하게 기록되고 누구나 검증할 수 있게 됩니다.
- **실시간 블록 연동 (3):** 홈페이지 하단의 "실시간 생성 블록" 엔진과 익스플로러의 "Latest Blocks" 데이터 소스는 동일한 `api/stats/blocks` API를 공유합니다. 
- **투명성:** 즉, 사용자가 홈페이지에서 확인하는 실시간 블록 생성 현황은 익스플로러에 즉시 동기화되어 나타나며, 블록 내부의 채굴자(Miner)와 해시(Hash) 정보가 일치함을 분석했습니다.

- **코드 로직:** 
    - **스케줄러 주입:** 매월 말일 23:59:59에 `accumulatedReward`를 확정(Freeze)하고, 신규 컬렉션 `mining_history`에 `{year, month, amount, status: 'FINALIZED'}` 구조로 저장하는 로직을 주입합니다.
    - **자동 전환:** 1일 00:00:00에 현재 채굴량을 0으로 리셋하고 UI에는 해당 월 데이터를 즉시 표기합니다.

    - **P2P 활성화:** 해당 이동 내역은 익스플로러에 'Migration Type Transaction'으로 별도 표기되어 투명성을 확보합니다.

## 5. 수수료(0.001, 0.01) 관리 및 글로벌 벤치마킹 분석

### 5.1 블록 및 거래 수수료 분배 (60/40)
- **분석 내용:** 블록 생성 수수료(0.001 BW)와 P2P 거래 수수료(0.01 BW)는 생태계의 혈액과 같습니다.
- **권고 사항:** 재단에서 관리하는 **[전용 수수료 지갑(Fee Escrow Wallet)]**을 2개로 분리하여 관리해야 합니다.
    - **Ecosystem Wallet (60%):** 네트워크 유지보수 및 유무형 보상 자금.
    - **Foundation/Dev Wallet (40%):** 재단 운영 및 플랫폼 고도화 자산.

### 5.2 타 블록체인 벤치마킹 비교 분석
| 구분 | Ethereum (ETH) | Pi Network (Pi) | **BitWish (BW)** |
| :--- | :--- | :--- | :--- |
| **수수료 관리** | EIP-1559 기반 일부 소각(Burn) | 수수료 전용 풀(Fee Pool) 적립 | **전용 지갑 분리 관리 (60/40)** |
| **분배 방식** | 네트워크 참여자 분배 및 희소성 조절 | 추후 배포 정책에 따라 결정 | **생태계 발전(60) & 운영(40) 고정** |
| **특징** | 복잡한 가스비 산정 방식 | 폐쇄형 메인넷 내 중앙 관리 | **투명한 목적별 지갑 분리(Escrow)** |
