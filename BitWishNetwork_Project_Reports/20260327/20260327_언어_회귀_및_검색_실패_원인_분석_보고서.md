## 1. 결함 정밀 분석 (Root Cause Analysis)

### ⚠️ [결함 A] 언어 설정 회귀 (Language Reversion)
- **현상:** 영어/일어/중어로 변경 후 익스플로러 진입했다가 '뒤로 가기' 또는 '새로고침' 시 초기 값인 한국어로 강제 복구됨.
- **원인:** `HomePage.tsx` (Line 56)의 초기 상태값이 `useState('ko')`로 하드코딩되어 있습니다. 마운트 시 `localStorage`를 전혀 참조하지 않아 발생하는 전형적인 데이터 유실입니다.

### ⚠️ [결함 B] 지갑 주소 검색 실패 (Search Failure)
- **현상:** 유효한 주소를 입력했음에도 "채굴자를 찾을 수 없음" 메시지 출력.
- **원인:** 현재 스크린샷 확인 결과 `최근 블록 높이`가 `-`(Dash)로 표시되어 있습니다. 즉, **`blocks` 배열이 비어있는(Empty List) 상태**에서 검색을 시도하여 `findIndex`가 무조건 실패하는 구조적 한계입니다.

---

## 2. 세부 해결 코드 로직 (Fix Implementation)

### 🛠️ [해결 A] 홈페이지 초기 상태값 영속화
- **수정 위치:** `src/components/HomePage/HomePage.tsx` (useState 선언부)
- **코드 로직:** 
    - `useState<Language>(() => (localStorage.getItem('bw_lang') as Language) || 'ko')`
    - 컴포넌트 마운트 시 브라우저 스토리지를 즉시 읽어와, 뒤로 가기 시에도 유저가 선택한 언어가 보존되도록 보수.

### 🛠️ [해결 B] 검색 엔진 데이터 로딩 검증 강화
- **수정 위치:** `src/components/Explorer/ExplorerPage.tsx` (handleSearch 함수)
- **코드 로직:**
    - 검색 시작 시 `if (blocks.length === 0)` 조건을 추가하여 데이터가 아직 동기화되지 않았음을 안내하는 가이드 보강.
    - 데이터가 로딩된 후 검색이 이루어지도록 로직의 유효성 검증 단계를 세분화.

## 3. [긴급 추가] 검색 알림 다국어화 및 데이터 무결성 검증

### ⚠️ [결 함] 알림창 한국어 노출 및 검증 누락 (Translation & Validation Bug)
- **현상:** 영어 모드에서도 "채굴자를 찾을 수 없습니다" 등 한국어 경고창 노출. 또한 데이터가 로딩되지 않았음에도(DASH 표시 상태) "찾을 수 없음"으로 오보함.
- **원인:** 
    1. `handleSearch` 내 알림 메시지가 하드코딩된 한국어로 작성됨 (지침 미준수).
    2. `blocks.length === 0` 로직으로는 유효하지 않은 데이터가 하나라도 있는 경우를 걸러내지 못함.

### 🛠️ [보강 해결 로직]
- **수정 위치:** `src/components/Explorer/ExplorerPage.tsx` (handleSearch)
- **코드 로직:**
    - **무결성 검증 고도화:** `!blocks.some(b => b?.header)` 조건을 추가하여 실제 헤더 정보가 유효할 때만 검색 수행.
    - **알림창 100% 번역:** `alert(`${t('explorer.searchResult')}...`)` 등 모든 사용자 안내 문구를 `LanguageManager`의 번역 키로 전면 교체.
