*   **대상 파일:** `src/components/SecondPasswordModal/SecondPasswordModal.tsx`

    2.  **독립 인증:** 내 PC에 무엇이 저장되어 있든 상관없이, 방금 입력한 **[지갑주소 + 비밀번호]** 만으로 서버에 로그인 요청을 보내도록 수정합니다.

## 2. 상세 작업 계획 (Action Plan)

### **작업 A: 보안 및 로그인 인증 체계 혁신 (최우선)**
관리자님의 "어디서든 접속 가능해야 한다"는 요구를 100% 이행합니다.

#### **A-1. 2차 비밀번호 정책 대폭 강화 (Security Hardening)**
*   **대상 파일:** `src/components/SecondPasswordModal/SecondPasswordModal.tsx`
*   **구현 내용:** 
    *   비밀번호 검증 정규식(Regex) 교체: `12자리 이상` + `특수문자 3개` + `대문자 2개` + `소문자 2개` 필수.
    *   **UI 변경:** 조건 미충족 시 **[설정] 버튼 비활성화** 및 붉은색 **"보안 안내"** 문구 표시.
    *   **코드 검증:**
    ```typescript
    const strongRegex = /^(?=(?:.*[A-Z]){2,})(?=(?:.*[a-z]){2,})(?=(?:.*[!@#\\$%^&*()_+|~=`{}\[\]:";'<>?,./\-]){3,}).{12,}$/;
    ```

#### **A-2. 로그인 모달 독립성 확보 (Stateless Login)**
*   **대상 파일:** `src/components/MiningAuthModal/MiningAuthModal.tsx`
*   **구현 내용:** 
    *   **자동완성 제거:** 모달 오픈 시 `localStorage`에서 지갑 주소를 강제로 불러와 덮어씌우는 로직 삭제.
    *   **순수 검증:** 사용자가 입력한 `WalletAddress`를 그대로 서버 인증(`verifySecondPassword`)에 사용.
    *   **결과:** A지갑 사용 후 B지갑 입력 시, 아무런 충돌 없이 B지갑 마이닝 페이지로 즉시 이동.

---

### **작업 B: 홈페이지 UI/UX 고도화 (어제 미완료분)**
어제 백엔드에서 준비해둔 "살아있는 데이터(7BW)"를 시각화합니다.

#### **B-1. 역동적 숫자 카운팅 (CountUp Animation)**
*   **대상 파일:** `src/components/HomePage/HomePage.tsx`
*   **구현 내용:** 
    *   `react-countup` 등을 사용하지 않고(라이브러리 최소화 원칙), **커스텀 훅(`useCountUp`)** 또는 경량 업데이트 로직을 구현하여 숫자가 `0` -> `7.000...`으로 촤르륵 올라가는 효과 구현.
    *   `currentIssued`(발행량)와 `remainingSupply`(잔여량)에 적용.

#### **B-2. 로고 및 디자인 고급화**
*   **대상 파일:** `src/components/HomePage/HomePage.css`
*   **구현 내용:** 
    *   단순 텍스트인 `BitWishNetwork`에 **그라데이션 텍스트(Gradient Text)** 효과와 **광채(Glow)** 효과 적용.
    *   지시서의 "고급스러운 디자인" 요구사항 충족.
