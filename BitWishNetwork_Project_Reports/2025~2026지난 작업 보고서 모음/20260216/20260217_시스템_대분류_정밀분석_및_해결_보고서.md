## Step 1-1: 완벽한 추천 보상 지급 체계 구축 분석 및 계획

### 1-1-1. 현재 구현 상태 정밀 분석 (As-Is)
*   **프론트엔드 독단적 처리:** 추천 보상(1BW) 지급 로직이 서버가 아닌 `ReferralBonusService.ts` (클라이언트)에만 존재함. 브라우저 환경에 따라 보상 누락이 필연적인 구조임.
*   **데이터 파편화:** 추천 데이터가 MongoDB가 아닌 서버의 `referrals.json` 파일과 로컬 스토리지에 분산 저장되어 있어, 데이터 불일치와 유령 데이터 발생의 근본 원인이 됨.
*   **서버 API 부재:** 가입 시 추천인 관계를 맺고 보상을 지급하는 전용 서버 API가 `server/routes/user.ts` 등에 구축되어 있지 않음.

### 1-1-2. 원인 분석 (Root Cause)
*   가입 프로세스에서 유저 정보 저장(`User`)과 보관함 업데이트(`BonusRecord`)가 단일 트랜잭션으로 묶여 있지 않아, 가입은 되는데 보상은 안 꽂히는 현상 발생.

### 1-1-3. 작업 계획 및 제안 코드 (To-Be)

**[작업 목표]**
"지갑 생성 -> 추천 코드 발급 -> 가입자 발생 -> 추천인/가입자 양방향 1BW 즉시 적립" 과정을 100% 서버 DB 기반으로 일원화하여 무결성 보장.

**[작업 1: 서버 API 신설 (`server/routes/user.ts` 또는 신규 `referral.ts`)]**
추천인 코드를 실제 DB에서 찾고, 보관함을 즉시 업데이트하는 전용 엔드포인트 마련.

```typescript
    // 3. 추천인의 BonusRecord 내 referralList에 가입자 정보 영구 기록
});
```

**[작업 2: 프론트엔드 연동 (`ReferralBonusService.ts`)]**
로컬 계산 로직을 제거하고, 보상 발생 시 즉시 위의 서버 API를 호출하도록 교체.

### 1-1-4. [추가] 정밀 보완 분석 (누락된 치명적 결함)
*   **데이터 타입 불일치 (String vs Decimal):** DB상 `referralRewardStorage`는 문자열("0.0000")임. 일반 JS 연산 시 부동소수점 오차가 발생하거나 누락될 위험이 큼. ➔ **Decimal.js(50자리)를 이용한 서버 사이드 정밀 연산 필수.**
*   **보상 수취 그릇(BonusRecord) 부재:** 가입 시 `User`는 생성되나 `BonusRecord`가 누락된 경우, 보관할 곳이 없어 1BW가 불능 상태가 됨. ➔ **가입 성공 즉시 upsert(없으면 생성)를 통한 무결성 확보.**

### 1-1-5. 실제 구현할 서버 사이드 핵심 코드

**[수정 대상: `server/routes/referral.ts`]**
```typescript
/**
 * POST /api/referral/register-reward
 * 가입 시 추천인/가입자 양방향 1BW 실시간 DB 적립 (무결성 보장)
 */
router.post('/register-reward', async (req, res) => {
    const { referralCode, referredWallet } = req.body;
    
    try {
        // 1. 추천인 정보 재검증
        const referrer = await User.findOne({ myReferralCode: referralCode });
        if (!referrer) return res.status(404).json({ success: false, message: 'Invalid referrer code' });

        // 2. 추천인 보상 처리 (1BW 가산 + 추천 리스트 추가)
        const referrerBonus = await BonusRecord.findOneAndUpdate(
            { walletAddress: referrer.walletAddress },
            { 
                $push: { 
                    referralList: { 
                        childWalletAddress: referredWallet, 
                        joinedAt: new Date(),
                        rewardStatus: 'COMPLETED'
                    } 
                } 
            },
            { upsert: true, new: true }
        );

        // 정밀 연산 (Decimal.js 사용)
        const currentRefReward = new Decimal(referrerBonus.referralRewardStorage || '0');
        referrerBonus.referralRewardStorage = currentRefReward.plus('1').toFixed(8);
        await referrerBonus.save();

        // 3. 가입자 본인 보상 처리 (1BW 가산)
        const referredBonus = await BonusRecord.findOneAndUpdate(
            { walletAddress: referredWallet },
            { }, // 상위 가입이므로 리스트 추가 없음
            { upsert: true, new: true }
        );
        const currentRefedReward = new Decimal(referredBonus.referralRewardStorage || '0');
        referredBonus.referralRewardStorage = currentRefedReward.plus('1').toFixed(8);
        await referredBonus.save();

        // 4. 추천인 채굴 보너스율(2%p) 업데이트
        await MiningState.findOneAndUpdate(
            { walletAddress: referrer.walletAddress },
            { 
                $inc: { referralCount: 1 },
                $set: { referralBonusRate: (referrerBonus.referralList.length * 0.02).toString() }
            }
        );

        return res.json({ success: true, message: 'All rewards issued accurately' });
    } catch (error) {
        console.error('[CRITICAL] Reward payout failure:', error);
        return res.status(500).json({ success: false, message: 'Server transaction error' });
    }
});
```
