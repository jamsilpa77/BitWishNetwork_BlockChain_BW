## 1. 백엔드: 사용자별 독립 데이터 조회 통로 구축

백엔드에서 특정 유저의 출석 내역만 정확히 가져오는 기능이 없어 프론트엔드가 로컬 데이터에만 의존하고 있습니다.

- **위치:** `server/routes/attendance.ts` (파일 최하단 `export default router;` 바로 위)
- **수정 로직 (신설):**
```typescript
/**
 * [추가] 특정 사용자의 모든 출석 기록 조회 API
 * GET /api/attendance/history/:walletAddress
 */
router.get('/history/:walletAddress', async (req, res) => {
    try {
        const { walletAddress } = req.params;
        if (!walletAddress) {
            return res.status(400).json({ success: false, message: '지갑 주소가 필요합니다' });
        }

        // DB(BonusRecord)에서 해당 지갑의 이력만 단독 조회
        const bonusRecord = await BonusRecord.findOne({ walletAddress });

        return res.json({
            success: true,
            history: bonusRecord ? bonusRecord.attendanceHistory : []
        });
    } catch (error) {
        console.error('Get attendance history error:', error);
        return res.status(500).json({ success: false, message: '서버 오류가 발생했습니다' });
    }
});
```

## 2. 서비스 레이어: 지갑 주소별 독자적 저장소(Key) 분리

유저 간 데이터가 섞이는 근본 원인인 '공용 키' 사용을 중단하고, 지갑 주소가 포함된 유니크한 키를 사용합니다.

- **위치:** `src/services/BonusService/AttendanceBonusService.ts`
- **수정 부위 A (독립 키 생성 및 서버 동기화 로직 추가):**
```typescript
  /**
   * [추가] 지갑 주소별 고유 Storage 키 생성 (독립성 보장)
   */
  private getStorageKey(walletAddress?: string): string {
    if (!walletAddress || walletAddress === 'current_user') return 'bw-attendance-records';
    return `bw-attendance-records-${walletAddress}`;
  }

  /**
   * [추가] 서버(MongoDB) 데이터와 로컬 데이터 강제 동기화
   */
  public async syncWithServer(walletAddress: string): Promise<void> {
    try {
      if (!walletAddress) return;
      const response = await fetch(`/api/attendance/history/${walletAddress}`);
      const data = await response.json();
      if (data.success && data.history) {
        // 서버의 단독 저장소 데이터를 로컬 기록으로 이관
        this.attendanceRecords = data.history.map((h: any) => ({
          ...h,
          isCompleted: true,
          status: 'COMPLETED'
        }));
        this.saveAttendanceRecords(walletAddress); // 분리된 키로 저장
        this.calculateBonusRate();
      }
    } catch (error) {
      console.error('서버 동기화 오류:', error);
    }
  }
```

- **수정 부위 B (저장/로드 로직에 지갑 식별자 적용):**
```typescript
  // 수정 전: localStorage.getItem('bw-attendance-records');
  // 수정 후:
  public loadAttendanceRecords(walletAddress?: string): void {
    const key = this.getStorageKey(walletAddress); // 지갑별 키 생성
    const savedRecords = localStorage.getItem(key);
    this.attendanceRecords = savedRecords ? JSON.parse(savedRecords) : [];
  }

  private saveAttendanceRecords(walletAddress?: string): void {
    const key = this.getStorageKey(walletAddress);
    localStorage.setItem(key, JSON.stringify(this.attendanceRecords));
  }
```

## 3. 프론트엔드 UI: 모달 오픈 시 실시간 격리 로딩

유저가 바뀔 때마다 이전 유저의 잔상이 남지 않도록 모달이 열리는 시점에 서버 데이터를 즉시 동기화합니다.

- **위치:** `src/components/AttendanceModal/AttendanceModal.tsx` (`useEffect` 부분)
- **수정 로직:**
```tsx
    useEffect(() => {
        const initAttendance = async () => {
            if (isOpen && walletAddress) {
                // 1. 서비스 초기화 및 해당 지갑의 독자적 로컬 데이터 로딩
                attendanceService.initializeService();
                attendanceService.loadAttendanceRecords(walletAddress);
                
                // 2. 서버로부터 '진짜' 출석 내역을 가져와 로컬 저장소 강제 갱신
                await attendanceService.syncWithServer(walletAddress);

                // 3. 동기화된 데이터로 UI 렌더링
                const recordDates = attendanceService.getAttendanceRecords().map(r => r.date);
                setAttendanceData(recordDates);

                const status = attendanceService.getAttendanceStatus();
                setIsCheckedToday(status === 'COMPLETED');
                setBonusRate(attendanceService.calculateBonusRate());
            }
        };
        initAttendance();
    }, [isOpen, walletAddress, attendanceService]);
```
