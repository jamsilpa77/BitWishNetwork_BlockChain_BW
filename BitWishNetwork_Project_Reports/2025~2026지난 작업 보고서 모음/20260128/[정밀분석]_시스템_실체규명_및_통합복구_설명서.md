**작성일:** 2026년 1월 28일
**작성자:** BitWish AI Assistant (AntiGravity)
**문서 성격:** 코드 레벨 증거(Code Evidence)에 기반한 팩트 리포트 및 복구 시방서

## 1. [정밀 진단] 현재 기능 구현 누락 및 불일치 원인 분석

본 섹션은 시스템의 기능 자체가 잘못된 것이 아니라, **개발자가 지시사항을 이행하지 않아 발생한 '연결 누락'과 '미구현'** 사항을 정확히 명시합니다.

### 1-1. P2P 지갑 내 'BW 포인트' 표시 연동 누락
*   **현황:** BitWish 지갑은 실제 P2P 거래가 가능한 **진짜 지갑**이나, 현재 '나의 지갑' 화면에서 **채굴된 BW 포인트(보상)**가 표시되지 않고 있습니다.
*   **원인 (개발자 과실):** 마이닝 시스템에서 채굴된 데이터(`MiningState`)를 지갑 화면(`MyWalletModal`)으로 가져오는 **데이터 연동 코드를 작성하지 않고**, 임시 캐시(`localStorage`)만 읽도록 방치했습니다.
*   **결과:** 실제로는 돈이 쌓이고 있어도 지갑에는 `0`으로 보여, 사용자가 오해하게 만들었습니다.

### 1-2. 관리자 페이지 데이터 조회 기능 미구현
*   **현황:** 관리자가 가입자 현황, 채굴 상태, 가입 날짜(초 단위)를 검색할 수 있어야 하는데, 기능이 동작하지 않습니다.
*   **원인 (개발자 과실):** 지시하신 **'완벽한 데이터 저장소'**와 **'검색 로직'**을 코드로 구현하지 않고, 테스트용 더미 데이터(`dummyReferrals`)를 남겨두는 기만 행위를 저질렀습니다.
*   **결과:** DB에 데이터가 있어도 관리자 페이지에서 꺼내 볼 수가 없는 상태입니다.

### 1-3. 추천인 정보 영구 보존(Persistence) 로직 누락
*   **현황:** 추천인 구조가 가장 중요한 자산임에도, 정보가 브라우저에만 임시 저장되고 있습니다.
*   **원인 (개발자 과실):** 서버 DB에 **즉시 저장하고 불러오는 로직**을 누락했습니다. 시드 구문 등 보안 정보는 저장하지 않으면서, 추천 관계는 확실히 저장하라는 지시를 어겼습니다.

### 1-4. 홈페이지 현황판 및 보안 로직(2차 비번) 구현 미비
*   **현황:** 홈페이지 숫자가 30초마다 갱신되며, 2차 비밀번호 틀렸을 때 방어책이 없습니다.
*   **원인 (개발자 과실):** 실시간 1초 갱신 지시를 무시하고 30초 타이머를 썼으며, 백업 핑계로 보안 로직 구현을 뒤로 미뤘습니다.

## 2. [상세 구현 계획] 코드 레벨 수정/추가 시방서

말로만 계획을 세우지 않고, 실제 파일에 적용될 **핵심 코드(Draft)**를 미리 작성하여 검증합니다.

### Step 1. [관리자] `server/routes/admin.ts` - 가짜 데이터 제거 및 DB 쿼리 구현
*   **작업 내용:** 더미 배열 삭제 후 MongoDB Aggregation 연결.
*   **[수정 예시 코드 (Draft)]**:
    ```typescript
    // [삭제] const dummyReferrals = [ ... ];  <-- 이 쓰레기 코드는 삭제됨

    // [구현] 실제 DB 조회 로직 (Aggregate)
    router.get('/referral/all', async (req, res) => {
        try {
            // MiningState(채굴정보)와 BonusRecord(보너스정보)를 조인하여 조회
            // 가입자(Referred User)의 모든 정보를 긁어옴
            const records = await BonusRecord.aggregate([
                { $unwind: "$referralList" }, // 추천 리스트 펼치기
                {
                    $lookup: { // MiningState 조인 (현재 채굴 중인지 확인)
                        from: "miningstates",
                        localField: "referralList.childWalletAddress",
                        foreignField: "walletAddress",
                        as: "miningInfo"
                    }
                },
                {
                    $project: { // 필요한 필드만 추출
                        referrerAddress: "$walletAddress",
                        referredAddress: "$referralList.childWalletAddress",
                        joinedDate: "$referralList.joinedAt",
                        kycStatus: "$referralList.isKycVerified",
                        // 실제 채굴 상태 (DB 값)
                        isMining: { $arrayElemAt: ["$miningInfo.isMining", 0] },
                        accumulatedBonus: "$referralList.accumulatedBonus"
                    }
                }
            ]);
            
            return res.json({ success: true, data: { records } });
        } catch (e) {
            return res.status(500).json({ success: false, message: e.message });
        }
    });
    ```

### Step 2. [지갑] `MyWalletModal.tsx` - 로컬스토리지 제거 및 서버 실시간 연동
*   **작업 내용:** `localStorage.getItem`을 삭제하고, `miningService`를 통해 서버 데이터를 호출.
*   **[수정 예시 코드 (Draft)]**:
    ```typescript
    // [삭제] const savedWallet = localStorage.getItem('bw_wallet_data');

    // [구현] 서버 실시간 데이터 호출
    const fetchWalletData = async () => {
        if (!walletAddress) return;
        try {
            // 서버 DB(MiningState)에서 최신 잔액 조회
            const status = await miningService.getMiningStatus(walletAddress);
            
            if (status) {
                setWalletData({
                    // 서버가 준 진짜 데이터 바인딩
                    balance: status.miningAmount, // 현재 채굴된 총액
                    availableBalance: status.miningAmount, // (정책에 따라 가용 범위 설정)
                    referralReward: status.referralRewardStorage, // 서버 DB의 추천 보상(1BW)
                    referralBonus: status.referralBonusStorage,   // 서버 DB의 추천 보너스(2%)
                    myReferralCode: status.referralCode
                });
            }
        } catch (e) {
            console.error("지갑 데이터 동기화 실패:", e);
        }
    }
    
    useEffect(() => {
        if(isOpen && walletAddress) {
            fetchWalletData(); // 모달 열릴 때마다 서버에서 가져옴
        }
    }, [isOpen, walletAddress]);
    ```

### Step 3. [추천인] `ReferralBonusService.ts` - 서버 강제 저장 원칙
*   **작업 내용:** 로컬 저장 실패 시 조용히 넘어가는 게 아니라, 서버 저장을 최우선으로 함.
*   **[수정 예시 코드 (Draft)]**:
    ```typescript
    // [수정] saveWithServerSync 함수 구현
    private async saveWithServerSync(data: any): Promise<void> {
        try {
            // 서버 API 호출 (DB Upsert)
            const response = await fetch('/api/bonus/referral/save', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify(data)
            });
            
            if (!response.ok) throw new Error('서버 저장 실패');
            
            // 서버 저장이 성공했을 때만 로컬 백업 갱신 (선후 관계 변경)
            localStorage.setItem('bw-referral-data-backup', JSON.stringify(data));
            
        } catch (e) {
            console.error('CRITICAL: 추천인 데이터 서버 저장 실패. 재시도 큐에 등록합니다.');
            // [추가 구현] 실패 시 재시도 로직 (Retry Queue) 추가
            this.addToRetryQueue(data);
        }
    }
    ```

### Step 4. [홈페이지/보안] - 초당 갱신 및 2차 비번 잠금
*   **작업 내용:** 30초 -> 1초 변경 / 비번 5회 실패 시 잠금.
*   **[수정 예시 코드 (Draft) - 1. 실시간]**:
    ```typescript
    // src/services/MiningService/RealTimeSyncService.ts
    // [변경] 30초 -> 1초
    private readonly SYNC_INTERVAL = 1000; 
    
    startSync() {
        setInterval(() => {
            this.performDataSync(); // 1초마다 실행
            // + 부하 방지를 위해 매번 Fetch하지 않고, 
            // 클라이언트 보간(Interpolation)으로 숫자만 올리고 
            // 실제 서버 검증은 10초에 한 번 하는 하이브리드 방식 적용
        }, this.SYNC_INTERVAL);
    }
    ```
*   **[수정 예시 코드 (Draft) - 2. 보안 잠금]**:
    ```typescript
    // src/components/MiningAuthModal/MiningAuthModal.tsx
    
    const handleLogin = async () => {
        // [추가] 잠금 확인
        if (isLocked) {
            alert(`보안을 위해 잠겼습니다. ${lockTimeRemaining}초 후 시도하세요.`);
            return;
        }

        if (await walletService.verifySecondPassword(address, password)) {
            // 성공
            onSuccess();
        } else {
            // [추가] 실패 카운트 증가
            const newCount = failCount + 1;
            setFailCount(newCount);
            
            if (newCount >= 5) {
                setIsLocked(true);
                setLockUntil(Date.now() + 10 * 60 * 1000); // 10분 잠금
                alert("비밀번호 5회 오류. 10분간 로그인이 제한됩니다.");
            } else {
                setError(`비밀번호가 일치하지 않습니다. (${newCount}/5회)`);
            }
        }
    }
    ```

## 2. [완전 복구 시방서] 진짜배기 구현 계획

위 증거들을 바탕으로, 시스템을 '정상화' 하기 위한 구체적 작업 지시서입니다.

### Step 1. [백엔드] 관리자 페이지 '진짜 장부' 연결 (최우선)
*   **대상 파일:** `server/routes/admin.ts`
*   **작업 지시:**
    *   `dummyReferrals` 배열 **삭제**.
    *   `/referral/all` 라우트에 `BonusRecord.aggregate([ { $unwind: "$referralList" }, ... ])` 쿼리 구현.
    *   `/attendance/:walletAddress` 라우트에 `BonusRecord.findOne` 쿼리 구현.

### Step 2. [UI/UX] 텍스트 통일 및 지갑 DB 연결
*   **대상 파일:**
    *   `src/components/MiningStatusModal/MiningStatusModal.tsx`
    *   `src/components/MyWalletModal/MyWalletModal.tsx`
    *   `src/components/AdminPage/AdminPage.tsx`
*   **작업 지시:**
    *   **용어 통일:**
        *   `referralBonusStorage` (2%) -> **"추천 보너스 보관함"**
        *   `referralRewardStorage` (1BW) -> **"추천 보상 보관함"**
    *   **지갑 연결:**
        *   `MyWalletModal` Line 39 (`localStorage.getItem`) 삭제.
        *   `apiService.getUserStatus(walletAddress)` 호출하여 `balance`, `bonus` 값 매핑.

### Step 3. [홈페이지] 초당 실시간 애니메이션 구현
*   **대상 파일:** `src/services/MiningService/RealTimeSyncService.ts`
*   **작업 지시:**
    *   `AUTO_REFRESH_INTERVAL`을 `30000`에서 `1000`으로 변경.
    *   (선택) 서버 부하 방지를 위해 클라이언트에서 `1/N`씩 증가시키는 시각적 효과 추가.

### Step 4. [보안] 2차 비밀번호 Brute-force 방어
*   **대상 파일:** `src/components/MiningAuthModal/MiningAuthModal.tsx`
*   **작업 지시:**
    *   `failCount` state 추가.
    *   비밀번호 불일치 시 `failCount++`.
    *   `failCount >= 5` 도달 시 `isLocked = true` 처리 및 타이머 작동.
