# 20260420 BW 메인넷 제네시스 자산 교정 및 마이그레이션 스케줄러 구축 계획서
## 1. 토큰 총 발행량(210억) 제네시스 분할 입금 교정

### [문제점]
코인 생태계가 처음 켜지는 순간, 단 1개의 '재단 통장(BitWish-Foundation)'으로 210억 개가 모두 입금되는 심각한 오류가 박혀 있습니다.

### [개선 및 구현 방안]
*   처음 코인이 태어나는 순간, 총 발행량 210억 개를 곧장 3가지 몫으로 정확하게 찢어서 별도의 시스템 통장 주소로 각각 송금해 버립니다.
    1.  **개인채굴자 전용 통장 (65%):** 136.5억 개 송금
    2.  **파트너/상점 전용 통장 (15%):** 31.5억 개 송금
    3.  **관리/재단 관리 통장 (20%):** 42억 개 송금
*   **작업 파일 위치:** `src/engine/BitWishBlockchain.ts` 에 있는 블록체인 초기화 함수(`initialize`) 시작 로직 전부 해체 후 재결합.

## 2. KYC 승인 통과자의 15일 대기 자동 지급기 (스케줄러)

### [문제점]
채굴된 가상 포인트가 실제 코인으로 변환되어 송금되기 전, 15일간의 안전 심사 대기 기간(타임락)을 무인으로 체크하고 발송하는 장치가 없습니다.

### [개선 및 구현 방안]
*   **딱지표 붙이고 대기실 이동 (DB 저장):** 관리자가 유저의 KYC 확인 승인 버튼을 누르는 즉시, 백엔드 DB 대기실에 유저 채굴량 기록과 함께 "발송 기한: 오늘 날짜의 15일 뒤"라는 시간 딱지를 붙여 조용히 냅둡니다.
*   **매일 자정 자동 순찰대 가동:** 메인 서버에 봇(Bot) 스크립트를 하나 심습니다. 이 로봇은 매일 밤 자정 정각마다 대기실을 순찰합니다.
*   **실제 코인 무인 발사:** 순찰 로봇이 딱지를 확인하여 15일 기한이 도래한 유저를 발견하면, 앞선 5번의 '개인채굴자 통장(65%)'에서 실제 코인을 유저의 지갑으로 자동 입금시킨 후 기록을 완료 마킹합니다.
*   **작업 파일 위치:** 백엔드 서버쪽 `cron-scheduler.ts` 파일 신규 스크립트 작성 및 몽고 DB 데이터베이스 연결.

# 20260421 BW 메인넷 KYC 및 마이그레이션 시스템 초정밀 구축 작업 계획서

## 1. 개요 및 구축 철학 (Philosophy)
본 계획서는 빗위시 네트워크의 투명성을 대내외적으로 증명하고, 유저에게 자신의 자산이 생성되는 과정을 시각적/심리적으로 확신시키기 위한 **'신뢰의 기술'** 구현을 목표로 함. 단순한 데이터 전송을 넘어, 블록체인의 본질인 **'공인 장부'**의 기능을 익스플로러와 지갑에 완벽하게 이식함.

---

## 2. 상세 시스템 설계 보고 (Detailed System Design)

### 2-1. 실전형 KYC 신청 엔진 (Frontend Modal)
*   **접근 경로:** '나의 지갑' -> 'KYC 신청' 버튼 클릭 시 전용 대형 모달 팝업.
*   **작업 환경 필드:**
    *   **정밀 거주지:** 신분증 기재 기준 국가(Country), 지역(도시, 구, 도) 및 도로명 상세 주소.
    *   **본인 인증:** 휴대폰 실명 인증 연동 API 탑재.
    *   **증빙 자료:** 고해상도 신분증 실사진 첩부 기능 (선명도 검사 로직 포함).
*   **최종 제출:** 'KYC 신청서 제출' 버튼 클릭 시 모든 데이터 AES-256 암호화 후 재단(관리자) 서버로 전송.

### 2-2. 재단 관리자용 KYC 심사 센터 (Admin Management)
*   **승인 권한:** 재단(관리자)이 수동으로 서류 검토 후 버튼 클릭으로 결정.
*   **상태값 실시간 반영:**
    *   🟢 **대기중 (Green):** 신청 직후 단계. 유저 지갑에 '검토 중' 표시.
    *   🔵 **승인 (Blue):** 검토 통과. 즉시 익스플로러 대기자 명단 등록 및 카운팅 시작.
    *   🔴 **불승인 (Red):** 반려. 사유 노출 및 재신청 유도.

### 2-3. 지갑 내 프리미엄 마이그레이션 UI (Wallet Experience)
*   **배치 구조:** 상단 [주소 복사] 버튼 우측과 [마이닝 시작] 버튼 좌측 사이 중앙 영역에 고정.
*   **실시간 역카운팅 (Real-time Countdown):**
    *   **단위:** 일(D):시(H):분(M):초(S)
    *   **비주얼:** 1초마다 줄어드는 다이내믹 애니메이션 적용, 하이엔드 테크니컬 디자인 피팅.
*   **상태 중계:** '마그 대기 중', '마그 중', '마그 완료' 등의 텍스트를 실시간 카운팅과 병기.

### 2-4. 익스플로러 대기자 투명성 검색 시스템 (Explorer Transparency)
*   **대기자 목록:** 승인된 모든 유저의 15일 타임락 현황을 전 세계에 실시간 중계.
*   **신뢰 검색:** 특정 지갑 주소 입력 시 본인 및 타인의 마이그레이션 잔여 기간과 상태를 즉시 확인 가능한 검색 엔진 구현.
