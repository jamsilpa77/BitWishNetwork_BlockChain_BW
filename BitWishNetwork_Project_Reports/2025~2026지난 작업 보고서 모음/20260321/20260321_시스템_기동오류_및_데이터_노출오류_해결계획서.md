# 20260321_시스템_기동오류_및_데이터_노출오류_해결계획서

**작성일자:** 2026년 3월 21일
**작성자:** Antigravity 
**적용 대상:** 백엔드 서버 기동 불가 및 실시간 상태 현황 데이터 노출 오류

---

## 1. 발생 문제 및 원인 분석 (팩트 체크)

### 1) TSError: Cannot find module '../server'
- **위치:** `BitWishNetwork_MiningSystem/server/routes/stats.ts`
- **원인:** 실제 백엔드 진입 파일명이 `index.ts`임에도 불구하고, 라우터에서 `../server`라는 잘못된 경로로 블록체인 코어를 불러오려 시도함. 이로 인해 컴파일 단계에서 중단되었습니다.

### 2) TSError: Expected 1 arguments, but got 3
- **위치:** `BitWishNetwork_BlockChain/src/engine/BitWishBlockchain.ts`
- **원인:** 블록체인 엔진 내부에서 트랜잭션을 생성할 때, 1개의 객체(`{ ... }`)만 받아야 하는 생성자(Constructor)에 3개의 인자를 각각 넘겨주어 문법 오류가 발생했습니다.

### 3) 대시보드 수치 '0' 노출 현원
- **원인:** 위의 컴파일 에러로 인해 백엔드 API 서버가 현재 **죽어있는 상태**입니다. 프론트엔드(홈페이지)가 데이터를 요청했으나 서버가 응답하지 못하므로, 화면에는 데이터베이스의 실제 값이 아닌 초기 기본값인 '0'이 출력되고 있습니다.

---

## 2. 정밀 해결 방법 및 수정 코드 로직

### 🛠️ [해결 1] 라우터 모듈 경로 수정
- **파일:** `BitWishNetwork_MiningSystem/server/routes/stats.ts`
- **수정 로직:** `index.ts` 파일에서 코어 엔진을 정상적으로 가져오도록 경로를 교체합니다.

```diff
- import { bwChainCore } from '../server';
+ import { bwChainCore } from '../index'; 
```

### 🛠️ [해결 2] 엔진 내 트랜잭션 생성 규격 준수
- **파일:** `BitWishNetwork_BlockChain/src/engine/BitWishBlockchain.ts`
- **수정 로직:** 파라미터를 낱개로 보내지 않고, 엔진 규격(`BitWishTransactionData`)에 맞는 단일 객체 형태로 포장하여 전달합니다.

```typescript
// [수정 전]
const tx = new BitWishTransaction(senderAddress, receiverAddress, decimalAmount);

// [수정 후]
const tx = new BitWishTransaction({
    from: senderAddress,
    to: receiverAddress,
    amount: decimalAmount.toString(),
    gasLimit: BITWISH_NETWORK_CONFIG.GAS_LIMIT,
    gasPrice: BITWISH_NETWORK_CONFIG.GAS_PRICE.toString(),
    nonce: this.getNonce(senderAddress),
    data: '',
    timestamp: Date.now(),
    type: BITWISH_TRANSACTION_CONFIG.TYPES.TRANSFER
});
```

## 3. 작업 순서 및 정상화 기대 효과

1. **[순위 1]** `BitWishBlockchain.ts`의 파라미터 문법 오류를 수정하여 엔진의 결함을 제거합니다.
2. **[순위 2]** `stats.ts`의 잘못된 모듈 호출 경로를 `index.ts`로 바로잡습니다.
3. **[검증]** `npm run backend`를 재기동하여 컴파일 에러가 사라지는지 확인합니다.
4. **[결과]** 서버가 정상 기동되면 홈페이지 대시보드가 다시 실제 DB(13개 지갑 등)의 수치를 정확히 출력하게 됩니다.

**[최종 확인]** 본 수정 계획은 기존의 지갑 데이터나 추천 시스템 로직을 단 0.1%도 변경하지 않으며, 오직 '코드 간의 연결 고리(문법 및 경로)'만을 정상화하는 안전한 조치입니다.
