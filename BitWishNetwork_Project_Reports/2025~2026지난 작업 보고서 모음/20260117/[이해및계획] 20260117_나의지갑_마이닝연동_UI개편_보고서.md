## 1. 관리자 지시사항 이해 (Understanding)

관리자님의 요구사항을 다음과 같이 완벽하게 이해했습니다.

### ① UI 레이아웃 변경 (Top Navigation)
*   **[2차 비번 초기화]** 버튼 위치 이동:
    *   기존 우측 하단에서 → **파란색 상단 네비게이션 바(Blue Header)의 왼쪽 아래 끝**으로 이동.
    *   (정확히는 헤더 영역 내에서, 타이틀이나 주소 표시줄 근처의 적절한 빈 공간 활용)
*   **[마이닝 시작]** 버튼 신규 추가:
    *   **"2차 비번 초기화" 버튼의 바로 왼쪽**에 배치.
    *   직관적인 실행 버튼 형태.

### ② 기능 정의: "마이닝 시작" (Direct Access)
*   **상황:** 사용자가 이미 시드 구문 등으로 [나의 지갑]에 들어왔다는 것은 **"최고 권한 인증"**을 마친 상태임.
*   **동작:** 따라서, **[마이닝 시작] 버튼 클릭 시, 묻지도 따지지도 않고(비밀번호 입력 창 없이) 즉시 개인 마이닝 페이지를 열어야 함.**
    *   2차 비밀번호 설정 여부와 관계없이 프리패스(Free Pass).

### ③ 기존 "마이닝 & 보너스" 버튼 (Main Page)
*   **동작:** 홈페이지 메인에 있는 기존 버튼은 **현행 유지.**
    *   즉, 여기로 들어오면 **2차 비밀번호 인증/설정 절차를 거쳐야 함.** (보안 절차 준수)

**[요약]**
*   **홈페이지 메인 버튼:** "비밀번호 대라!" (보안 모드)
*   **나의 지갑 내부 버튼:** "주인이시군요, 어서 오십시오." (프리패스 모드)

## 2. 작업 계획 (Implementation Plan)

### 작업 1: `MyWalletModal.tsx` UI 및 로직 수정
1.  **헤더 구조 재배치:**
    *   `wallet-header` 내부의 `header-address-row` (파란색 하단부) 우측 공간 활용.
    *   `flex` 컨테이너를 사용하여 우측 끝에 **[마이닝 시작] [2차 비번 초기화]** 순서로 버튼 배치.
2.  **[마이닝 시작] 버튼 핸들러 구현:**
    *   복잡한 `verify` 로직 제거.
    *   단순히 `onClose()` 호출하여 지갑 모달을 닫고,
    *   부모(`HomePage`)에게 **"마이닝 모달을 즉시 열어라(OPEN_MINING_DIRECTLY)"**라는 신호를 보냄. (Event 또는 Callback 활용)

### 작업 2: `HomePage.tsx` 연동
1.  **직접 열기 인터페이스:**
    *   `MyWalletModal`에서 마이닝 시작 신호를 받으면,
    *   `checkAuth`나 `passwordModal`을 띄우지 않고,
    *   즉시 `setIsMiningModalOpen(true)`를 실행하여 마이닝 페이지를 렌더링.

### 결과물 예시
*   **나의 지갑 상단 파란띠 우측 하단:** `[⛏️ 마이닝 시작]` `[🔄 2차 비번 초기화]` 버튼 생성.
*   **클릭 시:** 비밀번호 묻지 않고 바로 채굴 화면 팝업.
