# [치명적 결함] BitWish 시스템 기능 장애 원인 분석 보고서

**작성일:** 2026년 2월 7일
**작성자:** BitWish AI Assistant
**목표:** 지갑 데이터 미연동, 현황판 갱신 지연, 2차 비밀번호 설정 불가 등 **시스템 핵심 기능 마비의 원인을 코드 레벨에서 규명**한다.

---

## 1. 지갑 데이터 연동 실패 (Data Sync Failure)
### 현상
*   마이닝 페이지 상단에는 채굴된 BW가 표시되나, '나의 지갑' 모달을 열면 `0 BW`로 표시됨.

### 원인 분석 (Root Cause)
*   **파일:** `src/components/MyWalletModal/MyWalletModal.tsx`
*   **코드 위치:** `fetchWalletData` 함수 내부
*   **결함 내용:**
    *   개발자가 서버 API(`apiService`)를 호출하는 코드를 누락하고, 임시 테스트용 데이터인 `localStorage.getItem('bw_wallet_data')`를 그대로 방치함.
    *   이로 인해 실제 채굴 데이터(`MiningState`)와 지갑 UI 간의 연결 고리가 끊어진 상태임.

---

## 2. 실시간 채굴 현황판 갱신 지연 (Real-time Status Lag)
### 현상
*   홈페이지 메인 화면의 '전체 채굴량', '남은 채굴량' 등의 숫자가 30초마다 뚝뚝 끊기며 변함.

### 원인 분석 (Root Cause)
*   **파일:** `src/services/MiningService/RealTimeSyncService.ts`
*   **코드 위치:** `startSync` 메서드 내부 `setInterval`
*   **결함 내용:**
    *   갱신 주기(`AUTO_REFRESH_INTERVAL`)가 `30000ms`(30초)로 설정되어 있어, 블록체인 특유의 실시간 생동감을 전혀 주지 못함.
    *   1초 단위(`1000ms`) 실시간 갱신 로직이 부재함.

---

## 3. 2차 비밀번호 설정 불가 및 접속 제한 (Security Auth Failure)
### 현상
*   **치명적:** 사용자가 보안 규칙(대/소문자/특수문자/12자)을 지켜 비밀번호를 입력해도 설정에 실패함.
*   **치명적:** PC방, 모바일 등 다른 기기에서 지갑 주소를 입력해도 로그인이 불가능함.

### 원인 분석 (Root Cause)
*   **파일:** `src/services/BlockchainService/WalletService.ts`
*   **코드 위치:** `setSecondPassword` 메서드 (Line 252 부근)
*   **결함 내용:**
    ```typescript
    const wallet = this.getWalletFromStorage();
    if (!wallet || wallet.address !== address) return false; // [원인]
    ```
    *   **논리 오류:** 해당 기기의 로컬 스토리지(`localStorage`)에 지갑 정보가 없으면, **무조건 에러(`false`)를 반환**하도록 코딩되어 있음.
    *   **결과:** 지갑을 생성하지 않은 기기(PC방 등)에서는 비밀번호 설정조차 원천 봉쇄됨. 이것이 "어디서든 접속 불가"의 직접적 원인임.

---

## 4. 추천인 데이터 휘발 (Data Persistence Failure)
### 현상
*   브라우저 캐시 삭제 시, 내가 모집한 하위 파트너 정보가 전부 사라짐.

### 원인 분석 (Root Cause)
*   **파일:** `src/services/BonusService/ReferralBonusService.ts`
*   **결함 내용:**
    *   서버 저장(`fetch` API)이 실패했을 때, 재시도(`Retry`) 없이 로컬 스토리지(`localStorage`)에만 저장하고 종료해버림.
    *   중요한 자산 데이터인 '추천인 관계'를 영구적으로 보존하지 못하는 구조적 결함.
