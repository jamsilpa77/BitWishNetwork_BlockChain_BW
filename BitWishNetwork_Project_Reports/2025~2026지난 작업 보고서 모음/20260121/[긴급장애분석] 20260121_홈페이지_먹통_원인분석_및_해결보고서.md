## 1. 장애 현상 (Symptom)
*   **현상:** 홈페이지 접속 시 화면이 붉은색 에러 창(또는 흰색 화면)으로 변하며 아무것도 표시되지 않음. (White Screen of Death)
*   **에러 메시지:** `Uncaught Error: [DecimalError] Invalid argument: undefined`
*   **발생 위치:** `HomePage.tsx` 내 `formatNumber` 및 `splitNumber` 함수.

---

## 2. 근본 원인 분석 (Root Cause Analysis)

### **핵심 원인: "데이터 필드명(변수명) 불일치"**

프론트엔드와 백엔드가 서로 다른 언어(변수명)로 대화하고 있어, 데이터가 전달되는 과정에서 증발해 버렸습니다.

#### **A. 프론트엔드 (`HomePage.tsx`)의 기대치**
`HomePage.tsx`는 `globalStats` 상태 객체에 다음 이름의 속성이 있을 것이라 믿고 렌더링을 시도합니다.
```typescript
// HomePage.tsx (Line 66)
const [globalStats, setGlobalStats] = useState<any>({
    // ...
    currentIssued: '0',  // [중요] 변수명이 'currentIssued' 입니다.
    // ...
});

// UI 렌더링 (Line 477)
const { integerPart, decimalPart } = splitNumber(globalStats.currentIssued);
```

#### **B. 백엔드 (`stats.ts`)의 실제 응답**
제가 작성한 API는 다음과 같은 이름으로 데이터를 보내고 있습니다.
```typescript
// stats.ts (Line 59)
res.json({
    success: true,
    data: {
        // ...
        currentSupply: currentSupply.toFixed(8), // [오류] 변수명을 'currentSupply'로 보냈습니다.
        // ...
    }
});
```

#### **C. 충돌 발생 메커니즘**
1.  `HomePage`가 API를 호출(`fetchGlobalStats`)하여 결과를 받아옵니다.
2.  받아온 결과(`data`)에는 `currentSupply`만 있고 **`currentIssued`가 없습니다.**
3.  `setGlobalStats(data)`가 실행되면서, 기존의 안전한 초기값(`currentIssued: '0'`)이 **사라집니다.**
4.  화면을 그릴 때 `globalStats.currentIssued`를 찾으니 `undefined`가 나옵니다.
5.  `splitNumber(undefined)`가 실행되고, 내부의 `new Decimal(undefined)`에서 **대폭발(Error)**이 일어납니다.

---

## 3. 책임 소재 및 반성
*   **책임:** 전적으로 백엔드 API를 개발할 때 프론트엔드의 기존 데이터 구조(`state`)를 정밀하게 확인하지 않고, 제 멋대로 변수명을 지어낸(currentSupply) **저의 태만과 부주의**에 있습니다.
*   **반성:** "교차 검증"을 하라고 하셨을 때, 단순히 숫자가 맞는지(5개)만 보고, 변수명이 맞는지 확인하지 않은 점 깊이 반성합니다.

---

## 4. 해결 방안 (Solution)

백엔드 코드(`stats.ts`)를 수정하여, 프론트엔드가 원하는 이름(`currentIssued`, `totalUsers`)으로 데이터를 보내주면 즉시 해결됩니다.

### **[수정 계획: server/routes/stats.ts]**
기존 코드를 아래와 같이 매핑하여 수정합니다.

| 기존 (Server) | 수정 (To Be) | FrontEnd 대응 변수 | 비고 |
| :--- | :--- | :--- | :--- |
| `totalWallets` | **`totalUsers`** | `totalUsers` | 변수명 통일 |
| `currentSupply` | **`currentIssued`** | `currentIssued` | **[핵심 원인]** 수정 |

**수정 코드 예시:**
```typescript
res.json({
    success: true,
    data: {
        totalUsers: totalWallets,         // totalWallets -> totalUsers 로 변경
        currentIssued: currentSupply.toFixed(8), // currentSupply -> currentIssued 로 변경
        remainingSupply: remainingSupply.toFixed(8),
        issuanceRate: issuanceRate.toFixed(2),
        totalBlocks,
        networkStatus
    }
});
```
