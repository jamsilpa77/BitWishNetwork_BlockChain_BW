## 1. 현재 문제 상황 (Status)
*   아까 백엔드 코드(`stats.ts`)를 수정하여 변수명을 맞췄음에도, 홈페이지는 여전히 **"흰색 화면(White Screen)"**과 **"Invalid argument: undefined"** 에러를 출력하고 있습니다.

## 2. 원인 분석 (Why?)

### **원인 1: 실행 중인 서버의 '시차' (Lag)**
*   파일(`stats.ts`)은 수정되었지만, **실행 중인 서버 프로세스(Port 5001)**가 아직 재시작되지 않았을 가능성이 매우 높습니다.
*   따라서 서버는 여전히 수정 전(구버전) 데이터인 `currentSupply`를 보내고 있고, 프론트엔드가 기다리는 `currentIssued`는 오지 않고 있습니다.

### **원인 2: 프론트엔드의 '유리몸' (Fragility) [핵심]**
*   데이터가 설령 안 오거나 틀리게 오더라도, 화면은 떠야 합니다. (예: "0" 또는 "-")
*   현재 `HomePage.tsx`의 `formatNumber` 함수는 값이 `undefined`이면 **프로그램 전체를 멈춰버리는(Crash)** 치명적인 약점을 가지고 있습니다.
*   **"데이터가 없으면 죽어라"**라고 코딩되어 있는 것이 가장 큰 문제입니다.

---

## 3. 해결 방안 (Solution)

백엔드 서버 재시작을 기다릴 필요 없이, **프론트엔드(`HomePage.tsx`)를 '강철'처럼 튼튼하게 만드는 작업**이 최우선입니다.

### **[수정 계획: HomePage.tsx]**

**기존 코드 (문제점):**
```typescript
const formatNumber = (value: number): string => {
    return precisionCalculator.formatForUI(new Decimal(value)); // value가 없으면 여기서 폭발💥
};
```

**수정 코드 (안전장치):**
```typescript
const formatNumber = (value: number | string | undefined): string => {
    // 1. 방어 코드: 값이 없으면 무조건 '0' 리턴하고 끝냄. (절대 죽지 않음)
    if (value === undefined || value === null || value === '') {
        return '0'; 
    }
    // 2. 정상 처리
    return precisionCalculator.formatForUI(new Decimal(value));
};
```

## 4. 결론
이 **안전장치(Safety Guard)**를 추가하면:
1.  서버가 꺼져 있든,
2.  데이터가 잘못 오든,
3.  아직 로딩 중이든,

**절대로 화면이 하얗게 죽지 않고 기본값 '0'을 보여주며 정상 작동**하게 됩니다.
가장 시급하고 확실한 해결책입니다.
