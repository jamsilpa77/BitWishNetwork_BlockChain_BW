## 1. 정책 vs 실제 구현 정밀 비교 분석 (Gap Analysis)

관리자님께서 제공하신 **`[지시서] 메뉴2: 추천 보너스 설정 페이지`**의 **🎁 추천인 특별 혜택** 항목 1번을 근거로 코드를 감사(Audit)했습니다.

### **[A] 정책 (Policy)**
> **"1. 즉시 보너스: 추천인코드 발급자와 가입자(지갑 생성자)는 각 1 BW 즉시 지급 (첫 지갑 생성 시)"**

### **[B] 현재 코드 (`UserController.ts`)의 결함**
```typescript
// 현재 코드 (UserController.ts line 70-71)
const bonusRecord = new BonusRecord({
    walletAddress,
    referralBonusStorage: '0.00000000...',
    referralRewardStorage: '0.00000000...', // [치명적 오류] 0으로 시작함
    // ...
});
```
*   **원인:** 지갑을 생성(`new User`)할 때 `BonusRecord`를 만들기는 하지만, 정책에 명시된 **"1 BW"**를 초기값으로 주지 않고 **"0 BW"**로 설정하고 있습니다.
*   **결과:** 가입자 5명 모두 지갑은 있지만 보상은 `0`인 상태입니다. 이 때문에 총 발행량 합계가 8BW가 아닌 0~2BW로 나오는 것입니다.

## 5. 향후 수정 계획 (Final Optimization Plan)

**기존 시스템 분석 결과:**
`MiningController.ts`의 추천 보너스(2%) 분리 및 저장 로직은 이미 **완벽하게 구현**되어 있습니다. 건드릴 필요가 전혀 없습니다.
유일한 문제는 `UserController.ts`에서 최초 가입 시 지급해야 할 **[추천 보상 1BW]**의 초기값을 `0`으로 설정한 **단순 설정 오류**입니다.

따라서, **기존 시스템의 로직을 100% 존중**하며, 오직 **데이터 초기값**만 정상화하는 방향으로 작업을 진행합니다.

### **[1단계] 코드 결함 수정 (Code Correction)**
*   **대상:** `src/server/controllers/UserController.ts`
*   **작업:**
    *   `register` 메서드 내 `BonusRecord` 생성 코드 수정.
    *   `referralRewardStorage` (보상함)의 초기값을 `'0.000...'` -> **`'1.00000000000000000000000000000000000000000000000000'`** (1BW)로 변경.
    *   이 수정만으로, **신규 가입자**는 별도의 추가 로직 없이 시스템에 자연스럽게 1BW를 보유하게 됩니다.

### **[2단계] 데이터 소급 적용 (Retroactive Repair)**
*   **대상:** 기존 가입자 5명
*   **작업:**
    *   `repair_rewards.js` 스크립트를 작성하여 1회 실행.
    *   로직: "가입일이 있으나 `referralRewardStorage`가 0인 유저"를 찾아 **1BW를 강제 주입(Injection)하는 것이 아니라, 시스템이 지급한 것처럼 업데이트(`$set`)**.
    *   추천인 관계가 있는 경우, 상위 추천인의 보상함에도 `+1` 소급 적용.
    *   **효과:** DB가 "정상적으로 보상을 지급받은 상태"가 되며, 마이닝 페이지와 나의 지갑에 즉시 **8BW**가 반영됨.

### **[3단계] 백엔드 실시간 연동 (Real-time Linkage)**
*   **대상:** `src/server/routes/stats.ts`
*   **작업:**
    *   `totalBlocks`를 `0` (상수)에서 -> **`MiningState` 모델의 동기화 로직**이나 **DB 블록 카운트**를 참조하도록 변경.
    *   현재 테스트용으로 고정된 값들을 **살아있는 DB 데이터**로 교체.

### **[4단계] 프론트엔드 UI/UX 고도화 (Frontend Polish)**
*   **대상:** `src/components/HomePage/HomePage.tsx` 및 `HomePage.css`
*   **작업:**
    *   **CountUp 애니메이션:** `currentIssued`(발행량)와 `remainingSupply`(잔여량)에 `react-countup`을 적용하여 숫자가 실시간으로 차오르는 역동적인 효과 구현. (지시서: "역동적인 디자인")
    *   **로고 스타일링:** 단순히 텍스트만 출력되는 `<h1>BitWishNetwork</h1>`를 CSS로 디자인하여 고급스러운 브랜드 이미지 구축.
    *   **데이터 검증:** 백엔드(3단계) 연동 후 `totalBlocks`가 `0`에서 실제 숫자로 바뀌는지 최종 UI 확인.
