# [2026-03-10] 관리자 페이지 'NaN %' 수치 오류 해결 및 수정 계획서

**작성일자:** 2026년 3월 10일
**현상:** 추천 보상 현황 검색 시 보너스 수치가 숫자가 아닌 "NaN %"로 표시되며, 상단 요약 바의 수치도 누락됨.

---

## 1. 문제 원인 분석 (Root Cause)
1. **백엔드(Server):** 소스 코드 수정 후 서버가 재시작되지 않아 구버전 API(보너스 필드가 없는 버전)가 응답하고 있거나, 데이터 조회 후 응답 객체에 필드를 누락함.
2. **프론트엔드(UI):** 전달받은 값이 없을 경우(undefined)에 대한 숫자 변환 예외 처리가 없어 브라우저가 "숫자가 아님(NaN)"으로 출력함.

---

## 2. 기술 수정 계획 (How to Fix)

### **(1) 백엔드 로직 보강 및 서버 재기동**
- **파일:** `server/routes/admin.ts`
- **수정:** 가입자 상세 리스트 빌드 시 `bonusRate` 및 `totalBonusRate` 필드가 반드시 숫자형 문자열로 응답 데이터에 포함되도록 확약.
- **조치:** 코드 수정 후 반드시 서버 프로세스 재시작(PM2 또는 Node 리스타트) 집행.

### **(2) 프론트엔드 숫자 정규화 안전장치**
- **파일:** `src/components/AdminPage/AdminPage.tsx`
- **수정:** 
    - `parseFloat(ref.bonusRate)` → `parseFloat(ref.bonusRate || "0")`
    - 데이터가 누락되었을 때 강제로 **0**을 대입하여 `NaN` 출력을 원천 차단.

### **(3) 상단 요약 바 수치 가독성 개선**
- **파일:** `src/components/AdminPage/AdminPage.tsx`
- **수정:** 상단 요약 정보의 `totalBonusRate` 출력 시에도 기본값(0) 처리 로직 추가.

---

## 3. 최종 검증 계획
1. 지갑 검색 시 더 이상 "NaN" 글자가 보이지 않는지 확인.
2. 가입자별 보너스율이 실제 DB 데이터와 동일하게 (예: 2 %) 표출되는지 확인.
3. 상단 헤더의 전체 보너스 합계가 정상 숫자로 계산되어 나오는지 최종 체크.
