본 보고서는 BitWishNetwork 마이닝 시스템의 두 가지 진입점(홈페이지, 나의 지갑)에 따른 코드 로직 위치와 처리 방식의 차이점을 정밀하게 분석한 결과입니다.

---

## 1. 홈페이지 입구: "마이닝 & 보너스" 버튼 경로

홈페이지 메인 화면에서 사용자가 직접 채굴장에 입장할 때 사용하는 표준 경로입니다.

### 1-1. 코드 위치 및 버튼 정의
- **파일 위치:** `src/components/HomePage/HomePage.tsx`
- **버튼 위치:** 약 500~503행 (`className="mining-bonus-button"`)
- **이벤트 핸들러:** `onClick={handleStartMining}` (299~305행)

### 1-2. 처리 로직 흐름
1.  **모달 기동:** `handleStartMining` 함수 실행 시 `setIsMiningAuthModalOpen(true)` 상태로 변경.
2.  **인증 단계:** `MiningAuthModal.tsx` 컴포넌트가 화면에 노출됨. 유저의 지갑 주소 및 2차 비밀번호 인증 수행.
3.  **최종 진입:** 인증 성공 시 `onSuccess` 콜백(672~676행)을 통해 `authenticatedAddress` 설정 후 `MiningStatusModal.tsx`를 오픈.
- **특징:** 철저한 보안 인증 절차를 우선하는 '표준 루트' 방식입니다.

## 2. 지갑 입구: "마이닝 시작" 버튼 경로 (Fast Pass)

나의 지갑 모달 내부에서 이미 인증이 완료된 사용자가 채굴 기능을 실행할 때 사용하는 급행 경로입니다.

### 2-1. 코드 위치 및 버튼 정의
- **파일 위치:** `src/components/MyWalletModal/MyWalletModal.tsx`
- **버튼 위치:** 약 196~220행 (`className="mining-start-btn"`)
- **이벤트 핸들러:** `onClick` 내 익명 함수 (210~217행)

### 2-2. 처리 로직 흐름
1.  **신호 발송:** 지갑 내 버튼 클릭 시 부모인 `HomePage`로 `onOpenMining(walletAddress)` 신호를 발송.
2.  **세션 생성:** `HomePage.tsx`의 644~654행에서 신호를 수신. 
    - `WALLET_DIRECT_ACCESS`라는 특별 세션 데이터를 `localStorage`(`bw_mining_auth`)에 즉시 저장.
3.  **모달 전환:** 지갑 모달(`isMyWalletModalOpen`)을 닫고 마이닝 모달(`isMiningModalOpen`)을 즉시 활성화.
- **특징:** 이미 지갑에서 인증을 마쳤으므로, 별도의 인증 모달을 거치지 않고 바로 채굴장으로 이동하는 '하이패스' 방식입니다.

## 3. 공통 종착지: 마이닝 엔진 실체 (`MiningStatusModal.tsx`)

위의 두 입구를 통해 들어온 사용자가 최종적으로 맞이하는 실제 마이닝 페이지의 핵심 로직입니다.

### 3-1. 핵심 코드 위치
- **파일 위치:** `src/components/MiningStatusModal/MiningStatusModal.tsx`
- **기능:** 
  - `initializeData` (66~135행): 입구와 상관없이 전달받은 `walletAddress`를 기반으로 서버 API(`apiService.getUserStatus`)와 동기화.
  - `startMiningTicker` (215~226행): `RealTimeSyncService` 엔진을 가동하여 초당 채굴량을 계산하고 화면에 표출.

## 4. 입구별 전략적 중요도 및 관리 우선순위

본 시스템의 두 입구 중 관리 및 유지보수 관점에서의 중요도는 다음과 같습니다.

### 4-1. [최상] 나의 지갑 입구 (중요도: ★★★★★)
- **이유:** 사용자가 가장 빈번하게 체류하는 지갑 화면에서 즉시 채굴장으로 이동하는 **'주력 경로'**입니다.
- **코드:** `WALLET_DIRECT_ACCESS` (하이패스 세션)를 생성하므로, 이 로직이 깨지면 지갑에서 채굴기로 가는 흐름이 차단될 뿐만 아니라 보안상 가장 취약하고 중요한 지점입니다.
- **관리 핵심:** 지갑 인증(Wallet Auth)과 채굴 인증(Mining Auth)의 연속성을 보장해야 합니다.

### 4-2. [상] 메인 홈페이지 입구 (중요도: ★★★★☆)
- **이유:** 시스템에 처음 접속하거나 장기간 미접속한 유저의 **'보안 관문'**입니다.
- **코드:** `MiningAuthModal`을 통해 2차 비밀번호를 다시 확인하는 로직이 포함되어 있어, 시스템의 전체적인 보안 신뢰도를 책임집니다.
- **관리 핵심:** 로그아웃 된 사용자나 비정상 접근 시도를 차단하는 필터 로직의 무결성이 중요합니다.
