# [장애 분석 및 복구 계획] 홈페이지 실시간 발행 현황 엔진 정상화

**작성일자:** 2026년 3월 11일
**작성자:** BitWish 전담 AI
**문서번호:** BW-HOME-20260311-01

---

## 1. 장애 현황 분석 (Root Cause)

현재 홈페이지 메인의 실시간 데이터(현재 발행량, 잔여 발행량, 발행률)가 갱신되지 않거나 극히 소량만 변동하는 현상이 발생하고 있습니다. 이는 단순한 UI 버그가 아닌, **데이터 소스(Backend)와 UI 엔진(Frontend)의 유기적 연결이 거세된 상태**이기 때문입니다.

### **(1) 백엔드 결함 (Static Data Output)**
- 서버의 `/api/stats/realtime` 엔드포인트가 정적인 DB 데이터만 반환합니다.
- DB 데이터는 유저가 지갑을 열고 명시적으로 동기화할 때만 업데이트되므로, 실시간 채굴 현황을 반영하지 못합니다.

### **(2) 프론트엔드 결함 (Engine Disabled)**
- `HomePage.tsx` 내의 `startHomePageTicker` 함수가 비어 있습니다.
- "가상 데이터 제거"라는 명목하에 숫자를 실시간으로 올려주는 로직이 완전히 삭제되어 서버에서 정지된 데이터가 오면 화면도 정지됩니다.

---

## 2. 백업 파일(`${BACKUP}/20260303`) 정밀 분석 결과

사용자님의 지시에 따라 3월 3일 자 백업 파일을 분석한 결과, **안타깝게도 해당 시점에도 엔진은 이미 파괴된 상태**임을 확인했습니다.

- **분석 파일:** `BitWishNetwork_MiningSystem/src/components/HomePage/HomePage.tsx.bak`
- **확인 결과:** `startHomePageTicker` 함수가 `// [중요] 임의의 데이터 가공 로직을 전면 삭제했습니다.`라는 주석과 함께 비어 있음.
- **결론:** 해당 백업으로 코드를 롤백해도 실시간 수치 변동 문제는 해결되지 않습니다. **새로운 정공법 로직 주입이 필수적**입니다.

---

## 3. [Step 3] 복구 이행 계획 (진짜 데이터 기반 엔진 구축)

꼼수(가상 증분)가 아닌, 백엔드와 프론트엔드를 잇는 **진짜 실시간 동기화 엔진**을 구축하겠습니다.

### **Phase 1: 백엔드 실시간 합산 로직 (Back-end)**
- `stats.ts` 수정: 현재 채굴 중인 모든 사용자(isMining: true)의 채굴 속도(`currentTotalRate`)를 합산하여 초당 발행량(`totalMiningSpeed`)을 계산하여 전송.

### **Phase 2: 프론트엔드 실시간 카운터 (Front-end)**
- `HomePage.tsx` 수정: 서버에서 받은 `totalMiningSpeed`를 근거로, 프론트엔드 타이머가 1초마다 발행량을 정밀하게 이동시킴.
- **정합성 보장:** 30초마다 서버의 실제 DB 합계와 교차 검증하여 오차 0% 유지.

- 관리자 페이지와 홈페이지 간의 데이터 괴리 완전 해소.
