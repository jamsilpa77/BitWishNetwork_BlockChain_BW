# [최종완결] BitWish 지시서 기반 시스템 정밀 구현 및 오류 해결 계획서

**수신자:** 최고 관리자님
**기반 문서:** `BW채굴 마이닝 홈페이지 및 정책 개발 지시서.md` 및 관리자님 구두 지시사항
**문서 성격:** **지시서와 현존 시스템(관리자페이지, 웹소켓)을 100% 활용한 실질적 해결책**

## 1. [보너스 정책] 지시서 기반의 영구적/동적 순환 시스템

### **(1) 정책 재확인 (지시서 기준)**
*   **대상:** 일반 가입자(자동 코드 발급) 및 추천 가입자 모두 포함.
*   **보상:**
    *   **추천인:** 가입자 1명당 **1 BW (보상)** + **채굴 속도 2% 증가 (보너스)**.
    *   **가입자:** 가입 즉시 **1 BW (보상)** + **채굴 속도 2% 증가 (보너스)**.
*   **특징:** 이 2% 보너스는 일회성이 아니라, **내 하위 가입자가 늘어날 때마다 영구적으로 누적**되어야 함. (예: 10명이면 +20% 속도 증가)

### **(2) 구현 방안 (기존 서버 활용)**
*   **서버 로직 수정:** `MiningService`가 보상을 계산할 때, 단순히 저장된 값을 쓰는 게 아니라 **DB의 `ReferralList`를 실시간 카운트**합니다.
    *   `My Referrals Count` × 0.02 = **내 추가 속도**.
*   **즉시 반영:** 누군가 내 코드로 가입하는 순간(DB Insert), 내 채굴 속도 변수가 즉시 업데이트되어 마이닝 창의 숫자가 빨라집니다.

---

## 2. [관리자 기능] 기존 시스템 복구 및 데이터 연결

### **(1) 현황 분석 (이미 구현된 기능)**
*   **프론트엔드:** `AdminPage.tsx`에 지갑 주소로 추천 보너스 내역을 조회하는 UI와 로직(`handleSearchReferral`)이 이미 완벽히 구현되어 있습니다.
*   **백엔드:** `/api/admin/referral/...` 엔드포인트가 정의되어 있으나, 내부 로직이 **실제 데이터(DB/파일)를 조회하지 않고 빈 값만 반환**하거나 연결이 끊겨 있습니다.

### **(2) 해결책 (API 로직 구현)**
*   **백엔드 수정:** `server.ts` (또는 `admin.ts` 라우트)에서 해당 API 요청을 받으면, **`PersistentStorage` (referrals.json)** 및 **MongoDB**를 실제로 조회하도록 코드를 채워 넣습니다.
*   **계산 로직 탑재:** 조회 시 단순 데이터 리턴이 아니라, **[하위 추천인 수 × 2%]** 보너스율과 **[추천 보상 합계]**를 즉석에서 계산(On-the-fly)하여 `AdminPage`로 전송합니다.
*   **결과:** 관리자가 검색 버튼만 누르면, 유저의 모든 활동 내역(가입일, 추천 수, 보너스 총액)이 화면에 즉시 뜹니다. (UI 수정 없이 백엔드만 고치면 됨)

## 3. [홈페이지] 웹소켓(Socket.io)을 이용한 초당 실시간 반영

### **(1) 문제점**
*   마이닝 페이지는 도는데 홈페이지는 멈춰 있음. (폴링 방식의 한계)

### **(2) 해결책 (이미 있는 웹소켓 활용)**
*   **서버 (`server.ts`):** `socket.emit('global_stats_update', { totalIssued: ... })` 이벤트를 1초마다, 혹은 변동 시마다 브로드캐스트.
*   **홈페이지 (`HomePage.tsx`):** `socket.on('global_stats_update')`를 통해 데이터를 받으면 **화면 숫자를 즉시 갱신**.
*   **결과:** 1천만 명이 동시에 채굴하면 홈페이지 숫자가 미친 듯이 올라가는 **역동성(Dynamics)** 구현 완료.

## 4. [원격 접속 & 2차 비번] 빗장 제거 및 프리패스 구현

### **(1) [나의 지갑 -> 마이닝 페이지] 즉시 연결 (Free Pass)**
*   **정책:** 시드 구문으로 '나의 지갑'을 열었다는 건 이미 **최고 권한 인증**을 마쳤다는 뜻입니다.
*   **수정:** '나의 지갑' 내 [마이닝 시작] 버튼 클릭 시, 추가 인증 절차를 **전면 생략하고 즉시 마이닝 페이지를 오픈**합니다. (세션 자동 연동)

### **(2) [홈페이지 -> 마이닝 로그인] 어디서든 접속 (Simple Access)**
*   **정책:** 이미 2차 비밀번호를 설정했다면, 지구 어디서든 **[지갑 주소 + 2차 비밀번호]**만 입력하면 로그인되어야 합니다.
*   **수정 (빗장 제거):** `WalletService.ts`에 있는 **`if (!localWallet)` (내 PC에 파일 없으면 차단)** 코드를 삭제합니다.
    *   파일 유무와 상관없이 서버에 비번이 맞으면 무조건 **통과(Success)**시킵니다.

### **(3) [2차 비번 설정 오류] 특수문자 Fail 해결**
*   **원인:** 비밀번호 규칙(특수문자 등)은 맞았으나, **"너 이 PC 주인 아니잖아(파일 없음)"**라는 엉뚱한 이유로 `Fail`을 리턴하고 있었습니다.
*   **해결:** `setSecondPassword` 함수에서도 **로컬 파일 확인 코드를 제거**하여, 시드 인증만 통과했다면 언제든 2차 비번을 신규 설정/변경할 수 있게 풉니다.

## 5. [DB 일치] 명칭 통일 및 정밀도 보장

### **(1) 명칭 및 데이터 통일**
*   **UI 수정:** 마이닝 창의 `추천 보너스 보관함`과 내 지갑의 `추천보너스 보상 보관함` 텍스트를 **완벽하게 일치**시킴.
*   **데이터 소스:** 두 컴포넌트 모두 **`useMiningState` (커스텀 훅)** 하나를 바라보게 하여, 항상 같은 변수(`referralBonusStorage`)를 출력하도록 강제함. (하나가 바뀌면 둘 다 바뀜)
*   **정밀도:** `Decimal.js`로 50자리 계산 후, 화면 표시는 `toFixed(8)`로 통일. (호버 시 전체 표시 기능 유지)
