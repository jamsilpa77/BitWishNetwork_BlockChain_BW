# [마이닝 채굴 시 블록 생성 및 대시보드 "실시간 생성 블록" +1 블록 카운팅 프롬프트 지시서]

## 1. Context & Overview (시스템 배경 및 목표)

이 시스템은 **가상 포인트(BW) 채굴 시스템 + PoW 메인넷** 구조입니다. 현재 대시보드(홈페이지 "실시간 BW 채굴 상태 현황 판) 화면의 "실시간 생성 블록"판에는 기본적으로 '제네시스 블록 1개'가 연동되어 표시되고 있습니다.

여기에 다음 규칙을 적용하여 **[추천 보상 자산 고정 스냅샷 + 신규 채굴 연동 + 수수료 6:4 트랜잭션 분배]** 로직을 기존 코드를 망가뜨리지 않고 완벽하게 추가 및 수정해야 합니다.

### 💡 핵심 비즈니스 로직 및 자산 규칙

1. **제네시스 설정:** 210억 BW 발행 완료 (65:15:20 황금 비율 분할 주입 완료).
2. **추천 자산 박제:** 가입자 17명의 부모-자식 가입으로 발생한 [추천 보상 총 30 BW]는 유저의 실제 자산입니다. 시스템 업데이트로 일반 가상 포인트가 초기화되더라도 이 30 BW는 절대 소멸되거나 오염되지 않아야 하며, 제네시스 블록의 '잔액 상태 스냅샷'으로 고정되어 관리되어야 합니다.
3. **블록 카운팅 규칙:** 실시간 생성 블록판은 기본 `제네시스 블록 1개` + `유저들의 추천 보상 총 30 BW`를 기반으로 작동합니다. 이후 유저가 마이닝 시작 버튼을 누르면 **동시 실시간으로 +1씩 정상 카운팅**되어 전광판에 반영되어야 합니다.
4. **네트워크 수수료 자산 순환:** * 블록 생성 시 수수료 **0.001 BW** 발생 (P2P 거래 시 0.01 BW 발생 로직 기반).
* 이 수수료를 시스템 금고로 격리한 뒤, **`Decimal.js`를 사용해 50자리 정밀도**로 **6:4 비율**(`생태계 가치 보호 기금` 60% : `재단 운영 및 커뮤니티 지원 기금` 40%)로 분산합니다.
* 모든 적립 현황은 MongoDB `network_stats` 컬렉션에 실시간 동기화하여 영구 저장 및 서버 재시작 시 0.1 BW 오차도 없는 복구를 보장해야 합니다.

## 2. 사전 문제 분석 및 예방 지침 (절대 준수사항)

* **[절대 준수 1] 완성형 코드 출력:** 구현하거나 삽입/수정할 모든 소스 코드는 생략(`// ... 기존 코드 동일` 등) 없이 **처음부터 끝까지 100% 동작하는 완성형 코드**로 출력하십시오. 가상의 라인 번호(`Line 1 ~`)를 코드 내 주석이나 텍스트로 함께 명시하십시오.
* **[절대 준수 2] 기존 로직 및 기능 보호:** 기존 PoW 메인넷 기능 및 다른 API 도로망과 충돌이 없도록 구현 전 사전 문제 분석 리포트를 요약하여 코드 출력 전에 필수 보고하십시오.
* **[절대 준수 3] 데이터 정합성 및 원자성 (Atomicity):** 블록 생성, 추천 보상 30 BW 스냅샷 유지, 채굴 보상 차감, 수수료 누적 연산 및 MongoDB 반영은 **반드시 하나의 Mongoose Session Transaction**으로 묶여야 합니다. 단 하나라도 실패할 경우 전체 작업은 롤백(Rollback)되어 자산 불일치를 원천 차단해야 합니다.

## 3. 구조적 역할 분담 및 수정 대상 파일 목록

제공된 폴더 구조 아키텍처에 기반하여 다음 4개의 파일에 코드를 작성 및 배치하십시오.

### ① 정밀 연산 유틸리티 모듈 생성

* **File Path:** `server/utils/decimalUtil.ts` (신규 생성 또는 기존 utils 폴더 내 추가)
* **역할:** `Decimal.js`를 이용한 50자리 정밀 연산 모듈 정의로 중복 코드 최소화.
* **로직:** 수수료를 오차 없이 6:4로 분할하고 정밀 덧셈을 수행하는 순수 함수 구현.

### ② 백엔드 블록 생성 및 수수료 정밀 분배 (Step 1, Step 2)

* **File Path:** `server/services/BlockchainService/BitWishNetworkService.ts`
* **역할:** 실제 PoW `generateBlock()`과 동기적으로 트리거되는 인터페이스 및 서비스 핵심부.
* **로직:** 0.001 BW 수수료 발생 $\rightarrow$ `Decimal.js` 6:4 분배 $\rightarrow$ MongoDB `network_stats` 컬렉션에 `upsert` 방식으로 원자적(Atomic) `await` 반영 $\rightarrow$ 트랜잭션 커밋 처리. 시스템 상태 매핑 인덱스를 사용하여 DB 조회 성능 최적화 포함.

### ③ 컨트롤러 연동 인터페이스 구현

* **File Path:** `server/controllers/MiningController.ts`
* **역할:** 유저의 채굴 시작 요청 신호를 받아 서비스단 장인에게 토스하는 총감독관.
* **로직:** 수수료 분배 및 블록 생성이 성공 완료되면 최신 누적 블록 카운트(`totalBlockCount`)를 응답값으로 반환.

### ④ 프론트엔드 대시보드 실시간 전광판 반영 (Step 3)

* **File Path:** `src/components/MiningPage/MiningPage.tsx`
* **역할:** 대시보드 현황판 화면 및 API 송수신 배달원(`src/services/MiningService/ApiService.ts`)과의 연동.
* **로직:** 채굴 완료 신호를 받는 즉시 '실시간 생성 블록' 카운트를 기존 제네시스 기반에서 실시간 +1씩 증가된 서버 데이터로 웹 화면에 즉각 상태(State) 반영하는 함수 구현.

내가 비개발자 이기 때문에 구조적 역할 분담 및 수정 대상 파일 목록을 명확하게 알지 못하며 이전 다른 코딩 ai가 전에 짠 구조와 다를 수 있으니 내가 이해할 수 있도록 출력 할때 요약해서 대상 파일 목록 리포트를 수정해서 제공 하도록 해라.
