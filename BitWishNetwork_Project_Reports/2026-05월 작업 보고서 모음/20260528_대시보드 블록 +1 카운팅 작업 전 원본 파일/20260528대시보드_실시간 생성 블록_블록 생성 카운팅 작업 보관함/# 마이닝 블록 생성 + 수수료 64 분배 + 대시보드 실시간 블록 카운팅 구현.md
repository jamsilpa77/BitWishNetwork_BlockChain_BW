# 마이닝 블록 생성 + 수수료 6:4 분배 + 대시보드 실시간 블록 카운팅 구현

마이닝 채굴 시 블록을 생성하고, 수수료 0.001 BW를 6:4로 정밀 분배하며, 대시보드의 "실시간 생성 블록" 카운트를 +1씩 증가시키는 기능을 기존 코드를 파괴하지 않고 추가합니다.

---

## 사전 문제 분석 리포트 (절대 준수 2)

### 충돌 위험 분석

| 영역 | 위험도 | 분석 |
|------|--------|------|
| `MiningController.startMining()` | ⚠️ 중간 | 기존 SyncGuard 로직 뒤에 블록 생성 호출을 **추가(Append)**만 하므로 기존 로직 훼손 없음 |
| `stats.ts` `/api/stats/realtime` | ✅ 안전 | `totalBlocks`는 이미 `bitwish_network.blocks` 컬렉션의 `countDocuments()`로 조회 중. 블록이 DB에 추가되면 자연 증가 |
| `HomePage.tsx` | ✅ 안전 | `globalStats.totalBlocks`를 이미 표시 중. 서버가 올바른 숫자를 반환하면 프론트 변경 최소화 |
| `BitWishBlockchain.createBlock()` | ⚠️ 중간 | P2P 송금용 `verifyAndMineTransaction()`과 별개로, 채굴 전용 블록 생성 메서드를 **서버측 서비스에서** 호출 |
| 수수료 6:4 분배 | ✅ 안전 | `network_stats` 컬렉션에 `upsert` 방식으로 저장. 기존 `BitWishBlockchain`의 `saveNetworkStats()`와 동일 패턴 사용 |
| 추천 보상 30 BW 스냅샷 | ✅ 안전 | `BonusRecord.referralRewardStorage`는 기존 독립 필드. 블록 생성 로직이 이 값을 읽거나 쓰지 않음 |

### 핵심 설계 결정

1. **블록 생성 경로**: `MiningController.startMining()` → **새 서버 서비스** `BlockMiningService.onMiningBlock()` → `bwChainCore.createBlock()` → MongoDB `bitwish_network.blocks` 컬렉션에 자동 저장
2. **수수료 분배**: 블록 생성 성공 후 `Decimal.js` 50자리 정밀도로 0.001 BW를 6:4 분할 → `bitwish_mining` DB의 `network_stats` 컬렉션에 원자적 반영
3. **프론트엔드 반영**: `stats.ts`의 `/api/stats/realtime`이 이미 `bitwish_network.blocks.countDocuments()`를 반환하므로, 블록만 정상적으로 DB에 적재되면 대시보드는 **자동으로 +1 카운팅**

## User Review Required

> [!IMPORTANT]
> **추천 보상 30 BW 스냅샷 보존**: 지시서에 따르면 17명 가입으로 발생한 추천 보상 30 BW는 제네시스 블록의 "잔액 상태 스냅샷"으로 고정해야 합니다. 현재 시스템에서 이 30 BW는 `BonusRecord.referralRewardStorage` 필드에 이미 독립 저장되어 있어, 이번 변경에 의해 절대 오염/소멸되지 않습니다. 별도의 "스냅샷 동결 컬렉션" 추가가 필요하신지 확인 부탁드립니다.

> [!WARNING]
> **Mongoose Session Transaction 범위**: 지시서의 절대 준수 3에 따르면 블록 생성 + 수수료 분배 + DB 반영을 하나의 트랜잭션으로 묶어야 합니다. 현재 MongoDB 구조상 **두 개의 DB**(`bitwish_network`와 `bitwish_mining`)에 걸쳐 데이터가 저장되어 있습니다. 단일 MongoDB ReplicaSet 환경에서만 Cross-DB Transaction이 가능합니다. **현재 MongoDB가 ReplicaSet으로 구성되어 있는지** 확인 부탁드립니다. Standalone인 경우 수수료 분배 저장소를 `bitwish_network`의 `network_stats`로 통일하는 것을 권장합니다.

### ② 블록 생성 및 수수료 분배 서비스 (신규 생성)

#### [NEW] [BlockMiningService.ts](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/server/services/BlockMiningService.ts)

핵심 서비스 클래스. 채굴 시작 시 블록체인 엔진(`bwChainCore`)과 연동하여:

1. `bwChainCore.createBlock(walletAddress)` 호출 → PoW 블록 생성 및 `bitwish_network.blocks`에 저장
2. 블록 생성 성공 후 수수료 0.001 BW 발생
3. `decimalUtil.splitFee64()`로 6:4 분할 (생태계 기금 60% / 재단 운영 40%)
4. `bitwish_network.network_stats` 컬렉션에 원자적 `updateOne($inc)` 반영
5. 최신 블록 카운트(`bitwish_network.blocks.countDocuments()`) 반환

```
주요 메서드:
- onMiningBlock(walletAddress: string, session?: ClientSession): Promise<BlockMiningResult>
- distributeFee(feeAmount: Decimal, session?: ClientSession): Promise<void>
- getTotalBlockCount(): Promise<number>
```
