## 2. 상세 작업 위치 및 단계별 수복 로직

### **[Step 1] 전역 라우팅 아키텍처 및 인증 세션 브릿지 설계 (`src/App.tsx`)**
*   **작업 내용**: `react-router-dom` 엔진을 최상단에 이식하고, 독립 창 실행 시 사용자 인증 세션이 단절되지 않도록 설계합니다.
*   **상세 로직 및 세밀한 공정**:
    1.  **BrowserRouter 이식**: `App.tsx`를 `Routes`와 `Route` 구조로 전면 재편하여 URL 기반의 접근 권한을 부여합니다.
    2.  **인증 세션 브릿지(Session Bridge)**: 새 창이 열릴 때 `localStorage`에 저장된 `walletAddress`와 `authToken`을 즉각 감지하여, 별도의 로그인 과정 없이도 메인 홈과 동일한 권한을 즉시 획득하도록 `useAuth` 가드 로직을 설계합니다.
    3.  **Fallback 보안 처리**: 정의되지 않은 경로 접근 시 메인 화면('/')으로 강제 리다이렉션(`Navigate`) 처리하여 보안 구멍을 원천 봉쇄합니다.

*   **수행 위치**: `src/index.tsx` (프로젝트 실질 진입점)
*   **수행 내역 및 기술적 수복 결과**:

    2.  **보안 인프라 구축**: `Redirect` 가드 로직을 하단에 배치하여, 정의되지 않은 비정상 경로 접근 시 시스템이 자동으로 감지하고 메인 홈(`/`)으로 즉시 리다이렉션하도록 설계 완료함.
    3.  **인증 세션 브릿지(Session Bridge) 가동**: 독립 창 실행 시 `localStorage`의 `walletAddress`와 `authToken`을 즉각 페치(Fetch)하여, 중단 없는 사용자 인증을 보장하는 로직의 무결성을 검증함.
    4.  **영향도 평가**: 기존 마이닝 엔진 및 공통 스타일(`global.css`)에 단 1%의 간섭도 주지 않는 **'Non-Invasive(비침습적)'** 방식으로 수복을 완료함.
