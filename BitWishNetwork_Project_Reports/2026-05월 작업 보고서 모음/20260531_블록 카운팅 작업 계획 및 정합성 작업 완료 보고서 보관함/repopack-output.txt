# 블록체인 실시간 생성 블록 무결점 안전 수복 및 최초 마이닝 시작/실시간 채굴 돌파 시 블록 생성 정합성 보장 구현 계획 (최종본)

유저들의 마이닝 상태가 전부 초기화(0 BW)되었으므로 실시간 생성 블록 수는 **제네시스 1블록 + 추천보상 30블록 = 총 31블록**이 되어야 마땅하나, 현재 초과 생성된 블록(16개)이 DB에 존재하여 47블록으로 표시되고 있습니다.

유저의 기존 지갑/주소 등 소중한 개인 정보나 가입 데이터는 **100% 보존**하면서, 초과 생성된 블록 데이터 및 기금 장부 정합성을 완벽하게 수복(31블록화)합니다. 또한 최초 채굴을 시작할 때 즉시 +1블록을 지급하고 1BW를 돌파할 때마다 +1블록이 실시간 생성되도록 정밀 수선합니다.

## 핵심 수정 설계

### 1. 일회성 자산 및 블록 무결점 안전 수복 로직 추가 (`server/index.ts`)
서버가 기동할 때 `bwChainCore.initialize()`가 MongoDB에서 기존 데이터를 로드하기 **전에** 실행되어 완벽한 안전 수복을 수행하는 비동기 함수 `runOneTimeCleanup()`을 점화합니다.

- **중복 블록 제거**: `bitwish_network.blocks` 컬렉션에서 `blockHeight > 0`인 모든 블록들을 완전 삭제하여 **제네시스 블록(0번)만 보존**합니다.
- **계정 잔액 안전 리셋 (계정 삭제 절대 없음)**: `bitwish_network.accounts`에서 3대 기본 금고(`BitWish-Miner-Pool`, `BitWish-Partner-Pool`, `BitWish-Foundation`)를 제외한 유저 계정을 **삭제하지 않고, 오직 잔액(balance)만 `'0.0'`으로 안전하게 리셋**합니다.
- **기본 금고 원상복구**: 3대 금고 잔액을 제네시스 주입량(65%, 15%, 20%)으로 완벽하게 복구합니다.
- **글로벌 기금/수수료 리셋**: `bitwish_network.network_stats`의 기금 누적 수치를 전부 `0`으로 원상복구합니다.
- **마이닝 기준선 리셋**: `bitwish_mining.miningstates`의 모든 유저 채굴량을 `'0.0'`으로 셋팅하고, **`lastBlockRewardThreshold` 필드를 완전히 제거(`$unset`)**합니다.
  - 이렇게 해야 유저가 새로 채굴을 시작할 때 최초 +1블록을 즉시 지급받을 수 있는 무결점 상태가 됩니다.
- **중복 실행 방지**: `one_time_cleanup_v3` 플래그를 `bitwish_mining.system_configs`에 저장하여 중복 실행을 완벽 차단합니다.

### 2. 마이닝 시작 시 최초 즉시 +1블록 지급 (`MiningController.ts`)
우리 마이닝 시스템은 정지 없이 영원히 채굴하는 방식이므로, 시작 로직을 깔끔하고 단순하게 구성합니다.

- **최초 채굴 시작 시**: `lastBlockRewardThreshold` 필드가 존재하지 않거나 null인 경우에만 `BlockMiningService.onMiningBlock`을 호출하여 즉시 +1블록을 지급하고, 기준점을 `'0'`으로 지정 및 저장합니다.

### 3. 실시간 채굴량에 따른 부동소수점 돌파 카운팅 정밀화
`syncMiningData`, `getUserStatus` 내부의 1BW 경계 돌파 로직은 기존의 50자리 정밀도(`Decimal.js`) 카운팅에 따라, 누적 채굴량이 `lastBlockRewardThreshold + 1` 이상이 되는 즉시 정확하게 추가 블록을 마이닝하고 `lastBlockRewardThreshold`를 업데이트합니다.
- 채굴 시작 즉시: +1블록 (Threshold = 0)
- 채굴량이 1.0 BW 돌파 즉시: +1블록 추가 (Threshold = 1)
- 채굴량이 2.0 BW 돌파 즉시: +1블록 추가 (Threshold = 2)
