# [2026-05-16] BW 멀티윈도우 드래그 UI 전환 및 UX 정밀 수복 계획서

## 1. 현 구조 분석 및 문제점 (Current State Analysis)

### **1.1 고정형 모달(Modal)의 한계**
*   **파일**: `MyWalletModal.tsx`, `MiningPage.tsx` (및 관련 CSS)
*   **구조**: `.my-wallet-modal-overlay`가 화면 전체를 `rgba(0,0,0,0.6)`으로 덮고 중앙 정렬(`flex`)을 강제함.
*   **문제점**:
    1.  **상호 차단(Blocking)**: 하나의 모달이 열리면 배경 및 다른 UI와의 상호작용이 불가능함.
    2.  **강제 폐쇄 구조**: 마이닝 페이지를 보려면 지갑을 닫아야 하고, 지갑을 보려면 마이닝 페이지를 닫아야 하는 비효율적 UX.
    3.  **공간 활용 제약**: 창의 위치를 옮길 수 없어 정보가 겹칠 경우 확인할 방법이 없음.

## 2. 정밀 수복 공정 계획 (Technical Implementation Plan)

### **Step 1: 모달에서 '윈도우(Window)' 시스템으로 전환**
*   **변경 내용**:
    *   배경을 어둡게 하는 `overlay` 속성 제거 및 `pointer-events: none` 처리 (창 외부 클릭 가능하게 변경).
    *   모달 본체를 `position: fixed`로 유지하되, `top`/`left` 좌표를 상태값(`state`)으로 관리.
*   **위험 방지**: 창이 화면 밖으로 나가지 않도록 `window.innerWidth/Height` 범위 내에서 좌표 제한(Constraint) 로직 삽입.


### **Step 2: 정밀 드래그 엔진(Draggable Engine) 이식**
*   **대상**: `.wallet-header` 및 마이닝 페이지 헤더 영역.
*   **로직 단계**:
    1.  **`onMouseDown`**: 클릭 시점의 마우스 좌표와 창의 현재 좌표 간의 오프셋(Offset) 계산 및 `isDragging` 활성화.
    2.  **`onMouseMove`**: 마우스 이동 거리에 오프셋을 적용하여 실시간 좌표 업데이트. `user-select: none` 속성을 적용하여 드래그 중 텍스트 선택 방지.
    3.  **`onMouseUp`**: 드래그 종료 및 좌표 고정.
*   **최적화**: `requestAnimationFrame`을 사용하여 드래그 시 끊김 없는 60fps 부드러움 확보.


### **Step 3: 정밀 멀티 포커스(Dynamic Z-Index) 관리 시스템**
*   **기술적 정의**: 여러 개의 독립된 윈도우가 겹칠 때, 사용자의 클릭 상호작용에 따라 실시간으로 최상위 레이어(Top-most Layer)를 교체하는 동적 포커스 제어 시스템.
*   **정밀 작업 로직**:
    1.  **Global Focus State**: `HomePage.tsx`에서 현재 활성화된 창의 ID(예: 'wallet', 'mining', 'referral')를 추적하는 `activeModal` 상태값 운용.
    2.  **Z-Index Escalation**: 모든 윈도우의 기본 `z-index`를 10000으로 통일하되, `activeModal`과 일치하는 창에만 즉시 가중치(+100)를 부여하여 레이어 스택의 최상단으로 강제 격상.
    3.  **Event Capture**: 각 모달의 최상위 컨테이너에 `onMouseDownCapture` 리스너를 배치하여, 내부의 어떤 버튼이나 영역을 클릭하더라도 즉시 부모에게 포커스 변경 신호(Signal)를 전송.
*   **시각적 피드백(Visual Cue)**:
    *   **활성 창**: 그림자 농도 강화(`box-shadow: 0 20px 50px rgba(0,0,0,0.3)`) 및 헤더 투명도 제거.
    *   **비활성 창**: 반투명 처리(opacity: 0.85) 및 그림자 약화하여 시각적 거리감(Depth) 부여.


#### **[핵심 추가 공정: 오버레이 레이어 정합성 수복 (Stacking Context Resolution)]**
*   **문제 분석**: 각 모달 컴포넌트의 최상위 부모인 `overlay` div에 CSS 고정 `z-index`(10000~10002)가 부여되어 있어, 내부 모달의 우선순위를 높여도 부모 레이어의 한계에 갇혀 창 순서가 바뀌지 않는 '레이어 격리' 현상 발견.
*   **수복 방법**:
    1.  **우선순위 제어권 이전**: 모달 본체에 적용하던 동적 `z-index` 로직을 최상위 부모인 `overlay` div로 전면 이전.
    2.  **동적 스택 스왑**: `isActive` 상태에 따라 부모 오버레이의 `z-index`를 10100(활성) 또는 기본값(비활성)으로 실시간 교체하여 전체 스태킹 컨텍스트를 동기화.
    3.  **포커스 전파 최적화**: 오버레이 영역에 `onMouseDownCapture` 센서를 배치하여, 창의 배경이나 가장자리를 클릭하더라도 즉시 해당 윈도우 스택 전체가 최상단으로 격상되도록 처리.


### **Step 4: 상태 보존 및 UX 고도화**
*   **위치 기억**: 창을 닫았다가 다시 열 때, 마지막으로 배치했던 위치를 `localStorage`에 저장하여 복구.
*   **독립 제어**: '나의 지갑'에서 '마이닝 시작' 클릭 시 지갑이 자동으로 닫히지 않고, 마이닝 창이 옆에 새로 뜨도록 이벤트 흐름 수정.


### **Step 5: 초기 배치 최적화 및 드래그 제약 해제 (Initialization & Free Drag)**

#### **5.1: 분석 및 요구사항 정의**
*   **현황 분석**: 기존 180px, 250px 등의 좌표가 터무니없이 부족하여 텍스트와 버튼의 허리를 잘라버리는 치명적 결함 발생.
*   **수정 목표 (사용자 최종 시각 지시안)**: 
    *   **지갑**: "실시간 BW 채굴 상태 현황" 텍스트를 완전히 벗어나 그 아래 빈 공간부터 창이 시작되도록 대폭 하강.
    *   **마이닝/추천인**: 주황색 "마이닝 시작" 버튼까지 완전히 벗어나 그 아래 회색 카드 상단 라인에 맞춰 창이 시작되도록 극하단 강하.
    *   **공통**: 정확한 CSS Width(800, 600, 450)를 사용해 가로 정중앙 완벽 배치.

#### **5.2: 정밀 작업 공정**
1.  **윈도우별 초기 좌표 정밀 튜닝 (오차 제로 확정)**:
    *   **나의 지갑 (`MyWalletModal`)**: 초기 Y 좌표를 `300px`로 조정. 250px과 350px의 오류를 시정하여, 상단 텍스트는 건드리지 않고 튀어나온 주황색 마이닝 시작 버튼만 완벽히 덮어버리는 정확한 위치.
    *   **마이닝 페이지 (`MiningStatusModal`)**: 초기 Y 좌표를 `450px`로 대폭 하향 조정. 마이닝 시작 버튼을 온전히 살리고 하단 대시보드 카드 윗선과 일치하는 위치.
    *   **추천인 페이지 (`ReferralModal`)**: 마이닝 페이지와 동일한 뷰포인트를 유지하기 위해 Y 좌표 `450px`로 통일.
2.  **오픈 시 원위치 정렬(Reset) 로직**:
    *   기존의 '마지막 위치 기억' 코드를 완전히 제거. 
    *   창을 닫고 다시 열 때마다 무조건 **위에서 정의한 지정 좌표(300px / 450px)로 강제 복귀**하여 정돈된 UI 유지.
3.  **드래그 경계 제한(Constraint) 완전 철폐**:
    *   `handleMouseMove` 내의 화면 경계 제한 로직(`Math.max/min`)을 100% 삭제.
    *   사용자가 모니터 구석이나 브라우저 밖으로 창을 절반쯤 걸쳐놓고 싶을 때 어떤 저항도 없이 부드럽게 이동하도록 '무제한 드래그' 엔진 완성.
#### **5.3: 오픈 잔상(Ghosting) 현상 심층 분석**
*   **발생 증상**: 창을 구석으로 드래그한 뒤 닫고 다시 열 때, 구석에서 정중앙(지정 좌표)으로 창이 빠르게 날아오는 궤적(잔상)이 선명하게 노출됨.
*   **기술적 원인 (React LifeCycle & CSS Transition 충돌)**:
    1.  **State 잔류**: 창이 닫혀도 언마운트되지 않고 숨겨지기만 하므로, 마지막 드래그된 `position(x, y)` 상태값이 그대로 유지됨.
    2.  **렌더링 순서의 맹점**: 다시 열릴 때, 브라우저는 가장 먼저 **마지막 드래그 위치**에 창을 렌더링함.
    3.  **후행 초기화**: 직후 Step 5의 `useEffect`가 발동하며 `position`을 중앙 지정 좌표(300px, 450px)로 확 잡아당김(2차 렌더링).
    4.  **CSS 트랜지션 개입**: 위치가 구석에서 중앙으로 순간 이동할 때, 모달의 CSS `transition` (부드러운 열림 효과) 속성이 이 '좌표 이동'까지 애니메이션으로 처리해버려 화면을 가로지르는 유령 잔상이 만들어짐.

---

### **Step 6: 잔상(Ghosting) 버그 완벽 제거 및 렌더링 타이밍 최적화**

#### **6.1: 작업 목표**
*   창이 열릴 때 어떠한 이동(날아옴) 없이 즉시 지정된 위치(300px, 450px)에서 깔끔하게 등장하도록 보장.
*   오픈 시의 부드러운 투명도(Opacity) 효과는 살리되, 좌표 이동(Left/Top) 애니메이션은 원천 차단.

#### **6.2: 작업 대상 파일 및 위치**
1.  **React 로직**: `MyWalletModal.tsx`, `MiningStatusModal.tsx`, `ReferralModal.tsx` 
2.  **CSS 스타일**: 각 모달의 최상위 `.css` 파일 (Transition 속성 분리 제어)

#### **6.3: 상세 작업 공정 (2중 방어선 구축)**
1.  **1차 방어 (닫기 시점 사전 초기화 - 비동기 State 제어)**:
    *   현재 `if (isOpen)`일 때만 작동하는 초기화 로직을 보강하여, **`if (!isOpen)` (창이 닫힐 때)에도 `position` 상태를 미리 리셋**시킴.
    *   단, 창이 닫히는 도중 중앙으로 날아가는 닫힘 잔상을 막기 위해, 닫힘 애니메이션 시간(약 300ms)이 끝난 직후 쥐도 새도 모르게 좌표를 리셋시키는 `setTimeout` 비동기 클린업(Clean-up) 로직 도입.
2.  **2차 방어 (드래그 좌표 Transition 분리 - CSS 강제화)**:
    *   드래그를 담당하는 인라인 속성인 `left`, `top` 값에 CSS `transition`이 간섭하지 못하게 함.
    *   리액트 컴포넌트의 인라인 `style` 객체에 `transitionProperty: 'opacity, visibility'`를 강제 주입하여 좌표 이동 시의 애니메이션을 물리적으로 제거함.
    *   (부가 효과) 드래그 중 마우스 포인터를 미세하게 늦게 따라오던 딜레이 현상까지 완벽히 사라지고 100% 즉각적으로 따라붙게 됨.

### **Step 7: 모달 계층 구조(Layering) 최적화 및 연쇄 닫힘 현상 제거**

#### **7.1: 문제 분석 (Analysis)**
*   **현상**: `나의 지갑` 모달에서 `마이닝 시작` 버튼 클릭 시, 지갑 창이 즉시 닫히고 마이닝 현황 창이 열림.
*   **원인**: `MyWalletModal.tsx` 내부의 버튼 핸들러에 `onClose()`가 하드코딩되어 있어, 멀티 윈도우 환경임에도 불구하고 단일 모달 방식의 구식 로직이 잔존함.
*   **개선 방향**: 지갑 창을 닫지 않고 유지하여 유저가 자신의 자산 현황과 실시간 채굴 증분을 동시에 모니터링할 수 있는 **'데스크톱 앱 급 멀티 레이어 UX'**를 구축.

#### **7.2: 작업 목표**
*   `나의 지갑` -> `마이닝 시작` 클릭 시 지갑 모달을 닫지 않고 유지.
*   새로 열리는 `마이닝 현황` 모달이 지갑 모달보다 높은 `z-index`를 점유하여 자연스럽게 '앞으로' 배치되도록 강제 포커싱.

#### **7.3: 상세 작업 공정**
1.  **`MyWalletModal.tsx` 로직 수정**:
    *   `.mining-start-btn`의 `onClick` 이벤트 핸들러에서 `onClose()` 호출부를 삭제.
    *   이를 통해 `onOpenMining` 트리거 시 지갑의 `isOpen` 상태가 `false`로 변하지 않도록 차단.
2.  **`HomePage.tsx` 상위 통제 레이어 강화**:
    *   `onOpenMining` 콜백 함수 내에서 `setIsMiningModalOpen(true)` 실행 직후, 즉각적으로 `handleFocus('mining')`를 호출하여 새로 열린 창에 최상위 우선순위 부여.
3.  **Z-Index 인터랙션 검증**:
    *   지갑(`y: 300`)과 마이닝(`y: 450`)의 고유 좌표 오프셋을 활용하여, 두 창이 겹쳤을 때 하단 마이닝 창이 앞쪽에서 정보를 강조하고 지갑 창은 상단에서 배경 역할을 수행하는 입체적 레이아웃 확인.

## 3. 발생 가능한 부작용 및 방어 전략 (Risk Mitigation)

| 위험 요소 | 상세 분석 | 방어 전략 |
| :--- | :--- | :--- |
| **Z-Index 인플레이션** | 무한히 `z-index`를 높일 경우 다른 UI 요소(토스트, 로딩바)를 침범할 위험. | 수치를 계속 높이지 않고, `Base(10000)`와 `Active(+100)` 두 단계로만 고정 관리. |
| **이벤트 충돌** | 드래그 시작 시점과 클릭 시점의 중복 트리거로 인한 버벅임. | `isDragging` 상태와 포커스 전환 로직을 분리하여 드래그 중에는 포커스 연산을 최소화. |
| **모바일 호환성** | 좁은 화면에서 여러 창이 겹칠 경우 조작 불가능. | `window.innerWidth < 768px` 감지 시 모든 드래그 및 멀티 윈도우 기능을 끄고 기존 팝업 모드로 자동 강제 전환. |
| **메모리 누수** | 리스너 미해제로 인한 브라우저 성능 저하. | `useEffect`의 Cleanup 함수에서 모든 `window` 이벤트 리스너를 100% 제거 검증. |

## 4. 작업의 기대 효과 (Expected Benefits)

1.  **가시성**: 채굴 현황과 지갑 잔고를 한 화면에 나란히 띄워놓고 실시간으로 비교 확인 가능.
2.  **실효성**: 불필요한 창 닫기/열기 반복 횟수를 80% 이상 감소시켜 유저 피로도 급감.
3.  **효율성**: 여러 메뉴를 동시에 조작할 수 있는 멀티태스킹 환경을 제공하여 전문가용 플랫폼 느낌의 프리미엄 UX 완성.

### **[Step 7 완료 보고서: 모달 계층 구조 최적화 및 멀티 레이어 UX 구현]**

#### **1. 작업 개요 및 성과**
*   **작업 목표**: `나의 지갑`에서 `마이닝 시작` 클릭 시 지갑이 닫히는 단일 모달 방식의 한계를 극복하고, 두 창을 동시에 띄워 정보를 교차 확인할 수 있는 멀티 윈도우 환경 구축.
*   **달성 결과**: 지갑 창이 유지된 상태에서 마이닝 창이 그 위(Z-Index 10100)로 부드럽게 레이어링되는 고품격 UX 달성.

#### **2. 기술적 구현 세부 내역**

**✅ MyWalletModal 로직 정규화 (연쇄 닫힘 차단)**
*   **기존 문제**: 마이닝 시작 버튼의 `onClick` 핸들러에 `onClose()`가 하드코딩되어 있어 강제 종료가 발생함.
*   **조치 내용**: `MyWalletModal.tsx:L342` 영역에서 `onClose()` 호출부를 완전 삭제하고, 오직 마이닝 모달 오픈 콜백(`onOpenMining`)만 트리거되도록 수정 완료.

**✅ HomePage 포커스 동기화 강화 (Z-Index 제어)**
*   **구현 로직**: 지갑 창에서 마이닝 명령 수신 시, `setIsMiningModalOpen(true)` 호출과 동시에 `handleFocus('mining')`를 실행하도록 상위 통제 레이어를 정렬.
*   **효과**: 새로 열리는 마이닝 모달이 즉각적으로 '활성 상태(Active)'가 되어 지갑 모달을 시각적으로 덮으면서도 뒤에 잔존한 지갑 데이터를 확인할 수 있는 입체적 구조 형성.

**✅ 레이아웃 오프셋 시각적 검증**
*   **검증 결과**: 지갑(`y: 300`)과 마이닝(`y: 450`)의 수직 거리 차이로 인해, 두 모달이 겹쳤을 때 정보가 뭉개지지 않고 계단식으로 배열되어 사용자에게 '멀티 태스킹'의 쾌감을 제공함.

#### **3. 작업 최종 확인**
*   **독립성 준수**: 전역 상태나 공통 함수 없이 순수 Props 및 Local State만으로 멀티 레이어링 구현 완료.
*   **잔상 제거 패리티**: Step 6에서 구축한 잔상 제거 로직이 멀티 윈도우 환경에서도 완벽히 작동하여 위치 이동 없는 깨끗한 렌더링 확인.
