## 🛠️ 1. 1공정: Prisma DB 전체 Schema 설계 (물리적 격리)

**[작업 위치 (신규 생성)]**
*   `c:\BitWishNetwork_BlockChainMainnet\BW_Community_Server\prisma\schema.prisma`
*   *사유: 마이닝 백엔드(`BitWishNetwork_MiningSystem`)와 서버 폴더 자체를 물리적으로 완전히 분리하여 마이닝 DB 훼손 확률을 0%로 만들기 위함입니다.*

**[기존 코드 로직 (현재 상태)]**
*   커뮤니티 전용 사용자 및 게시글 관계형 스키마 부재. (기존은 마이닝을 위한 단일 지갑 `MiningState` 위주)

**[수술/신설 로직 요약]**
*   **User 모델:** `email`, `password`, `nickname`, `role` (구글 소셜 로그인 지원을 위한 `provider` 필드 추가)
*   **Post 모델:** `title`, `content`, `category`, `hotScore` 등 (다크모드/다국어는 프론트엔드단 처리이므로 DB는 순수 데이터만 저장)
*   **Comment & Reaction 모델:** 게시글과 1:N 관계로 묶어 실시간 핫스코어(좋아요, 하트 등)를 정밀 계산할 수 있는 구조 확립.

## 🔐 2. 2공정: JWT 및 소셜 로그인(구글) 구현

**[작업 위치 (신규 생성)]**
*   `BW_Community_Server\src\controllers\authController.ts`
*   `BW_Community_Server\src\routes\authRoutes.ts`
*   `BW_Community_Server\src\utils\jwtHelper.ts`

**[기존 코드 로직 (현재 상태)]**
*   기존 마이닝 시스템은 지갑 주소 자동 인식 기반이므로, 표준화된 JWT 보안 토큰 생성 로직 및 구글 OAuth 로직이 없음.

**[수술/신설 로직 요약]**
*   **authController.ts:** `register`(회원가입), `login`(이메일 로그인), `googleOAuth`(소셜 로그인) 로직을 전역 변수 없이 캡슐화하여 구현.
*   **jwtHelper.ts:** 인증 시 보안을 위해 15분짜리 `AccessToken`과 7일짜리 `RefreshToken`을 듀얼로 발급하는 최고 등급 보안 아키텍처 로직 구현. (전역 클래스 공유 금지 원칙에 따라 독립 유틸로 작성)

---

## ⚠️ 3. 절대 준수 규칙 크로스체크
1.  **기존 로직 간섭 금지:** 기존 마이닝 폴더(`BitWishNetwork_MiningSystem`) 내부 코드는 단 1byte도 수정/간섭하지 않고 `BW_Community_Server`라는 분리된 우주(폴더)를 생성하여 작업하므로 완벽히 안전합니다.
2.  **마이닝 DB 절대 보호:** 커뮤니티 Prisma는 별도의 Database URL을 향하도록 설정하여 하이브리드 MongoDB와 절대 섞이지 않습니다.
3.  **전역 상태/모달 배제:** 백엔드 API 서버이므로 프론트엔드 전역 상태와 무관하며, 완벽한 RESTful 무상태(Stateless)로 구현됩니다.
