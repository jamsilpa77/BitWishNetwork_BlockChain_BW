# BW 커뮤니티 Phase 4 정밀 수술 계획서 (헤더 연동 및 관리자 시스템 복구 - 개정판)

**작성일**: 2026년 05월 19일
**목표**: 
1. 마이닝 홈페이지 상단 헤더의 [BW 커뮤니티] 버튼 클릭 시, 화면을 벗어나지 않고 독립된 멀티윈도우 창(`BWCommunityWindow`)이 마운트 및 팝업되도록 통합 연동합니다.
2. 커뮤니티 창 내부가 '더미 로딩'에 그치지 않고, 실제 게시글 CRUD, 댓글/대댓글 작성, Socket.io 실시간 랭킹 갱신이 시각적으로 완벽하게 구동되는 실물 컴포넌트를 이식합니다.
3. 커뮤니티 내부에 ⚙️ 관리자 기능을 수행하는 `CommunityAdminDashboard` 컴포넌트를 구축하고 백엔드 관리자 API(`adminController`, `adminRoutes`)를 설계하여 독자적인 커뮤니티 관리 기능을 실현합니다.
4. 커뮤니티 서버(`BW_Community_Server`)를 로컬에서 구동하여 실제 테스트를 진행할 수 있도록 서버 엔트리(`index.ts`), `package.json`, `tsconfig.json` 파일 구조를 신설합니다.

## 🔗 1. 8공정: 기존 마이닝 홈페이지 헤더(Header) 연동 및 실물 UI 구현

### [수술 대상 1] `BitWishNetwork_MiningSystem\src\components\BWCommunityWindow\BWCommunityWindow.tsx`
*   **기존 로직:** 닫기 버튼이 없고 API 연동 없이 더미 로딩 상태만 렌더링함.
*   **수복 교체 로직:** 
    *   창을 닫을 수 있는 `onClose?: () => void` Props 인터페이스 추가 및 헤더 우측 다국어/테마 선택 드롭다운 옆에 `❌` 버튼 이식.
    *   **실물 게시판 UI 구현:** `communityFetch` 유틸리티를 호출하여 게시글 전체 목록, 카테고리 필터, 페이지네이션 렌더링.
    *   **글작성/상세보기/댓글(대댓글) 뷰 추가:** 윈도우 내부 로컬 State(`currentView: 'list' | 'detail' | 'write' | 'admin'`)에 의존하여 뷰를 전환하는 자체 라우팅 구조 구현.
    *   **실시간 웹소켓 연동:** `socket.io-client` 패키지를 활용해 백엔드 서버(포트 4000)와 웹소켓 커넥션을 수립하고, `hot-post-update` 이벤트를 감지하여 리액션 발생 시 새로고침 없이 화면의 게시글 순위와 Hot Score가 실시간으로 재정렬되는 연동 로직 추가.

### [수술 대상 2] `BitWishNetwork_MiningSystem\src\components\HomePage\HomePage.tsx`
*   **기존 로직 (L524-L526):**
    ```typescript
    <button className="nav-button hover-button" onClick={() => window.location.href = '/community'}>
        {getTranslation('navigation.community')}
    </button>
    ```
*   **수복 교체 로직:**
    *   상단 임포트에 `import { BWCommunityWindow } from '../BWCommunityWindow/BWCommunityWindow';` 정밀 추가.
    *   로컬 토글 상태 선언: `const [isCommunityOpen, setIsCommunityOpen] = useState(false);`
    *   버튼 클릭 이벤트 수정: `onClick={() => setIsCommunityOpen(true)}`
    *   JSX 하단부 모달 렌더링 영역에 삽입:
        ```tsx
        {isCommunityOpen && (
            <BWCommunityWindow onClose={() => setIsCommunityOpen(false)} />
        )}
        ```
    *   **안전 조치:** 마이닝의 전역 상태(Redux, Zustand) 및 백그라운드 채굴 타이머 엔진에는 절대 영향이 가지 않는 순수 로컬 State 토글 방식 적용.

## ⚙️ 2. 9공정: 커뮤니티 전용 관리자(Admin) 시스템 및 서버 구동 인프라 신설

### [수술 대상 3] `BW_Community_Server\src\controllers\adminController.ts` (신규 생성)
*   **기능:** 
    *   `createSystemNotice`: 커뮤니티 전체 공지사항을 등록하고 상단 고정(Pin) 처리하는 API.
    *   `blindPost`: 불법/악성 게시글을 화면에서 블라인드(삭제) 처리하는 API.
    *   `banUser`: 악성 활동 유저를 차단(밴) 상태로 전환하는 API.
*   **특징:** 요청 헤더의 JWT 페이로드에서 `role === 'ADMIN'`을 강하게 검증하여 어드민 이외의 사용자는 절대 차단.

### [수술 대상 4] `BW_Community_Server\src\routes\adminRoutes.ts` (신규 생성)
*   **기능:** 관리자 전용 URL 매핑 (`POST /api/admin/notice`, `PUT /api/admin/post/blind`, `PUT /api/admin/user/ban`).

### [수술 대상 5] `BW_Community_Server\src\index.ts` (신규 생성 - 백엔드 기동)
*   **기능:** 포트 `4000`에서 Express 서버와 Socket.io 엔진을 융합 기동하고 CORS(3000/5001포트 화이트리스트 지정), body-parser, 모든 라우터(auth, post, comment, reaction, upload, admin)를 통합 바인딩.

### [수술 대상 6] `BW_Community_Server\package.json` (신규 생성)
*   **기능:** `express`, `socket.io`, `@prisma/client`, `cors`, `dotenv`, `typescript`, `ts-node-dev` 등 필수 의존성 정의.

### [수술 대상 7] `BW_Community_Server\tsconfig.json` (신규 생성)
*   **기능:** 백엔드 TypeScript 컴파일 환경 구성.

### [수술 대상 8] `BitWishNetwork_MiningSystem\src\components\BWCommunityWindow\BWCommunityWindow.tsx` (보강)
*   **기능:** 어드민 권한 파싱 및 ⚙️ 관리자 대시보드 UI 구현.
    *   JWT 토큰 정보를 로컬 스토리지 또는 세션에서 읽어와 권한이 `ADMIN`이면 헤더 영역에 관리자 전용 탭 노출.
    *   탭 전환에 따라 다크모드와 4개국어 번역을 유지하는 공지글 생성/신고글 관리용 프리미엄 관리자 페이지 연동.

## ⚠️ 3. 절대 준수 규칙 크로스체크
1.  **마이닝 시스템 안전 보호:** 마이닝 홈페이지(`HomePage.tsx`)를 수정할 때 오직 로컬 State 토글과 컴포넌트 렌더링 코드만 이식하므로 메인넷 MongoDB나 트랜잭션 검증 모듈에는 0%의 피해도 발생하지 않습니다.
2.  **독립성(Air-Gap):** 커뮤니티 전용 관리자 기능은 오직 커뮤니티 전용 Prisma DB(`schema.prisma`)에서만 쓰기 연산을 처리하며 메인넷 지갑 데이터에는 절대 읽기/쓰기를 하지 않습니다.
3.  **파일 생성 제한 준수:** 계획서에 지정된 5개 신규 생성 파일(`adminController.ts`, `adminRoutes.ts`, `index.ts`, `package.json`, `tsconfig.json`) 이외의 임의 파일 생성은 일절 금지합니다.
