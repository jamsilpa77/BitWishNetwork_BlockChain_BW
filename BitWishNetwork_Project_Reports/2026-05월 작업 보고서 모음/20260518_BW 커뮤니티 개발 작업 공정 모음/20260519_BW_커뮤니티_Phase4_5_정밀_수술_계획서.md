# BW 커뮤니티 Phase 4.5 정밀 수술 계획서 (스타일 수복 및 CORS 연동)

**작성일**: 2026년 05월 19일
**목표**:
1. 마이닝 프론트엔드 빌드 환경에 존재하지 않는 Tailwind CSS 유틸리티 클래스를 React 표준 **바닐라 인라인 스타일(inline style)** 객체로 전면 수복하여 UI 화면 깨짐 및 투명 겹침 현상을 완벽하게 극복합니다.
2. 커뮤니티 서버(`index.ts`)의 CORS 허용 포트를 기존 오타/기본값인 `3000`에서 실제 마이닝 프론트엔드가 동작 중인 `5000`으로 수정하여 네트워크 통신을 완벽히 개통합니다.

---

## 🔗 1. 수술 대상 및 기존 vs 수복 교체 로직 요약

### 🛠️ [수술 대상 1] `BW_Community_Server\src\index.ts` (CORS 포트 수복)
*   **기존 코드 로직 (L18-L23):**
    ```typescript
    app.use(cors({
        origin: ['http://localhost:3000', 'http://localhost:5001'],
        credentials: true,
        methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS']
    }));
    ```
*   **수복 교체 로직:**
    ```typescript
    app.use(cors({
        origin: ['http://localhost:5000', 'http://localhost:5001'], // 3000 포트를 실제 마이닝 프론트엔드 포트인 5000으로 교체
        credentials: true,
        methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS']
    }));
    ```

### 🛠️ [수술 대상 2] `BitWishNetwork_MiningSystem\src\components\BWCommunityWindow\BWCommunityWindow.tsx` (바닐라 CSS 스타일 수복)
*   **기존 코드 로직:**
    컴포넌트 전체가 `className="w-[850px] h-[650px] bg-gray-900 fixed ..."` 형태의 Tailwind 유틸리티 클래스로 도배되어 있어, 빌드가 되지 않아 스타일이 통째로 씹히고 투명하게 겹쳐 렌더링됨.
*   **수복 교체 로직:**
    모든 DOM 요소의 Tailwind CSS 클래스(`className`)를 삭제하거나, React 표준 인라인 스타일 객체(`style={{ ... }}`)로 전격 교체 수복합니다.
    *   **메인 프레임 컨테이너:** 
        ```tsx
        // 변경 후 인라인 스타일 이식
        style={{
            position: 'fixed',
            left: position.x,
            top: position.y,
            width: '850px',
            height: '650px',
            borderRadius: '16px',
            boxShadow: '0 25px 50px -12px rgba(0,0,0,0.25)',
            overflow: 'hidden',
            display: 'flex',
            flexDirection: 'column',
            border: theme === 'dark' ? '1px solid #1f2937' : '1px solid #e5e7eb',
            backgroundColor: theme === 'dark' ? '#111827' : '#ffffff',
            color: theme === 'dark' ? '#ffffff' : '#111827',
            zIndex: 99999,
            transition: 'background-color 0.2s, color 0.2s'
        }}
        ```
    *   **헤더 바 (Header Bar):**
        ```tsx
        style={{
            padding: '16px',
            display: 'flex',
            justifyContent: 'space-between',
            alignItems: 'center',
            cursor: 'move',
            userSelect: 'none',
            borderBottom: theme === 'dark' ? '1px solid #374151' : '1px solid #e5e7eb',
            backgroundColor: theme === 'dark' ? '#1f2937' : '#f3f4f6',
            transition: 'background-color 0.2s'
        }}
        ```
    *   **목록 카드 및 입력 폼 요소:**
        각 카테고리 필터 버튼, 글 목록 카드, Input 입력 칸 역시 테마(Light/Dark)와 4개국어 번역 상태를 완벽히 유지하면서, Tailwind CSS 의존성을 100% 제거한 바닐라 CSS 형태의 style 객체로 교체합니다.

## ⚠️ 2. 절대 금지 규칙 및 중요 준수 사항 자가 검증
1.  **다른 코드 로직에 대한 간섭 금지:** 이미 수술이 완료되어 연동 완료된 `HomePage.tsx` 스위치 로직 및 `authController.ts` 등은 1byte도 수정 또는 삭제하지 않습니다.
2.  **다른 파일 생성 절대 금지:** 본 계획서 이외의 불필요한 임시 파일이나 스크립트 생성을 일절 차단합니다.
3.  **전역 상태 오염 차단:** 어떠한 전역 상태 관리 라이브러리도 쓰지 않으며 오직 캡슐화된 `BWCommunityWindow` 내부의 로컬 State만 사용합니다.
4.  **다국어 및 다크모드 완벽 대응:** 인라인 CSS 수복 중에도 다국어 번역 함수(`getTranslation`) 및 다크모드 색상 분기 로직(`theme === 'dark' ? ... : ...`)은 완전무결하게 유지합니다.
