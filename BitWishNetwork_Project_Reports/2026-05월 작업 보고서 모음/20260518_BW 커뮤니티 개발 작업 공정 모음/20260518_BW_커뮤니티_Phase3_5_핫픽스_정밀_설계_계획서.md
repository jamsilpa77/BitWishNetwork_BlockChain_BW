# BW 커뮤니티 Phase 3.5 긴급 수복(Hotfix) 정밀 수술 계획서

**작성일**: 2026년 05월 18일
**목표**: 원본 설계서 대비 누락되었던 3대 핵심 코어 기능(대댓글 계층 구조, 디스코드 로그인, 외부 이미지 업로드 로직)을 마이닝 시스템에 절대 피해가 가지 않는 선에서 `BW_Community_Server` 내부에 완벽히 이식(수복)합니다.

## 🔧 1. 긴급 1공정: DB 및 인증 스키마 보강

**[작업 위치 (기존 파일 수정)]**
*   `c:\BitWishNetwork_BlockChainMainnet\BW_Community_Server\prisma\schema.prisma`
*   `BW_Community_Server\src\controllers\authController.ts`
*   `BW_Community_Server\src\routes\authRoutes.ts`

**[수술(교체) 로직 요약]**
*   **대댓글(Nested Comment) 스키마 이식:** 기존 `Comment` 모델에 `parentId Int?` 필드를 추가하고 자기 자신을 참조하는 `@relation("CommentToComment")` 옵션을 걸어줍니다. 이렇게 하면 1차 댓글 밑에 무한 뎁스(Depth)의 대댓글을 달 수 있는 구조가 완성됩니다.
*   **디스코드(Discord) 소셜 로그인 추가:** `authController.ts` 내부에 `discordOAuth` 전용 컨트롤러를 신설합니다. 지갑이나 마이닝 인증과는 100% 무관한, 순수 커뮤니티 전용 JWT 토큰 발급 체계로 캡슐화합니다.

## ☁️ 2. 긴급 2공정: 이미지 업로드 코어 구축

**[작업 위치 (신규 생성)]**
*   `BW_Community_Server\src\controllers\uploadController.ts`
*   `BW_Community_Server\src\routes\uploadRoutes.ts`

**[수술(신설) 로직 요약]**
*   **외부 클라우드 전용 업로드 API:** 마이닝 시스템이 구동되는 서버의 물리적 디스크 용량을 커뮤니티 이미지들이 갉아먹으면 서버가 터질 위험이 있습니다. 이를 원천 차단하기 위해, 업로드되는 모든 이미지를 AWS S3 또는 Cloudinary 외부 스토리지로 던져버리는 `multer` 기반의 Headless API 라우터를 신설합니다.
*   이를 통해 프론트엔드에서 이미지를 업로드하면 절대 로컬에 남지 않고 클라우드 URL만 반환되어 `Post` 테이블의 `thumbnail`에 깔끔하게 저장됩니다.

## ⚠️ 3. 절대 준수 규칙 크로스체크
1.  **다른 코드/파일 간섭 절대 금지:** Phase 1, 2에서 만들어둔 파일 중 `schema.prisma`와 `auth...` 관련 파일만 정밀하게 절개하여 살을 붙이며, 멀쩡히 돌아가는 게시글/댓글 라우터는 1byte도 건드리지 않습니다.
2.  **마이닝 시스템 절대 보호:** 이미지 업로드를 로컬 경로(`/uploads`)가 아닌 클라우드 프록시로 설계하여 마이닝 메인넷 서버의 I/O 부하율 0%를 유지합니다.
