    ```tsx
    // 뼈대 깜빡임 효과(Skeleton Loading)에 썸네일용 네모 박스 애니메이션을 추가합니다.
    <div style={{ display: 'flex', justifyContent: 'space-between' }}>
        <div style={{ flex: 1 }}>
            <div className="bw-skeleton-box" style={{ width: '70%' }}></div>
            <div className="bw-skeleton-box" style={{ width: '40%' }}></div>
        </div>
        <div className="bw-skeleton-box" style={{ width: '84px', height: '84px', borderRadius: '16px' }}></div>
    </div>
    ```

    **3. 리액션 타격감(Bouncing) 극대화 애니메이션 (Step 3.3 대응):**
    ```css
    /* 기존의 단순한 크기 변경을 넘어, 누를 때 뒤틀리며 튀어 오르는 타격감을 주입 */
    @keyframes popBouncing {
        0% { transform: scale(1) translateY(0); }
        40% { transform: scale(1.25) translateY(-4px) rotate(-8deg); }
        70% { transform: scale(1.1) translateY(2px) rotate(4deg); }
        100% { transform: scale(1) translateY(0) rotate(0); }
    }
    .bw-reaction-btn { 
        transition: all 0.2s cubic-bezier(0.25, 0.46, 0.45, 0.94); 
    }
    .bw-reaction-btn:active { 
        animation: popBouncing 0.5s cubic-bezier(0.175, 0.885, 0.32, 1.275); 
    }
    ```

## 🏗️ 2. 신규 개발 공정 (Phase & Step 상세)

모든 작업은 마이닝 메인 코어 파일과의 충돌을 피하기 위해 **`bw_community_`** 접두사를 붙인 플러그인(Plugin) 형태로 격리 개발됩니다.

### 🟡 Phase 1: 백엔드 안전 격리 및 DB 설계 (포트 5001 연동)
**목표:** 기존 `index.ts`를 어지럽히지 않고, 커뮤니티 전용 DB 모델과 라우터를 플러그인처럼 안전하게 마운트.

*   **Step 1.1: 커뮤니티 전용 통합 스키마 설계**
    *   **대상 파일:** `server/models/bwCommunityModels.ts` `[신규 생성]`
    *   **구현 로직:** 사용자 스키마(이메일, 소셜ID), 게시물 스키마(6대 카테고리 지원), 리액션 스키마 통합 정의. 마이닝 지갑 주소 필드는 철저히 배제합니다.
*   **Step 1.2: 프리미엄 비즈니스 로직(라우터) 구축**
    *   **대상 파일:** `server/routes/bw_community_api.ts` `[신규 생성]`
    *   **구현 로직:** 구글 로그인 페이로드 검증 로직, 핫 게시물 점수(Like*3 + Heart*5...) 실시간 계산 로직, 게시글 CRUD.
*   **Step 1.3: 마이닝 코어에 안전하게 마운트 (포트 5001)**
    *   **대상 파일:** `server/index.ts` `[정밀 수정]`
    *   **수복 기법:** 기존 마이닝 라우터들은 절대 건드리지 않고, 파일 맨 끝(DB 연결 코드 직전)에 단 2줄만 추가하여 플러그인처럼 연결.
