승인이 떨어지면, **오직 아래 명시된 위치와 코드대로만** 안전하게 텍스트 파일 단위로 수술하겠습니다.

### 🟢 [수술 부위 1] `src/components/BWCommunityWindow/BWCommunityWindow.tsx` (리스트 뷰 개편 및 애니메이션 강화)
*   **작업 목적:** Phase 2에서 구축한 기초 UI에 생명력을 불어넣습니다. 리스트 화면 우측에 '썸네일(Thumbnail) 이미지 레이아웃'을 신설하고, 리액션 버튼 타격감을 최고조로 끌어올립니다.
*   **기존 코드 로직의 한계:** 
    - 텍스트 위주의 단순 리스트 렌더링 (리치 미디어 대응 불가)
    - 밋밋한 버튼 클릭 효과

*   **수복될 신규 코드 (핵심 요약):**

    **1. 썸네일 이미지 동적 렌더링 레이아웃 신설 (Step 3.2 대응):**
    ```tsx
    // 단순 텍스트 박스를 넘어, 우측에 썸네일이 들어갈 공간을 확보한 프리미엄 리스트 UI로 교체합니다.
    <div className="bw-card-inner" style={{ display: 'flex', justifyContent: 'space-between', alignItems: 'flex-start', width: '100%' }}>
        <div style={{ display: 'flex', flexDirection: 'column', gap: '10px', flex: 1 }}>
            {/* 카테고리, 제목, 작성자 등 기존 메타 정보 */}
        </div>
        
        {/* 리치 미디어 썸네일 전용 규격 (이미지가 없다면 기본 글래스모피즘 플레이스홀더 출력) */}
        <div className="bw-thumbnail" style={{ 
            width: '84px', height: '84px', 
            borderRadius: '16px', 
            backgroundColor: 'var(--bw-input-bg)',
            border: '1px solid var(--bw-border-color)',
            marginLeft: '20px',
            flexShrink: 0
        }}>
            <span style={{ opacity: 0.3, fontSize: '24px' }}>🖼️</span>
        </div>
    </div>
    ```

    **2. 스켈레톤 로딩 고도화 (썸네일 동기화):**
    ```tsx
    // 뼈대 깜빡임 효과(Skeleton Loading)에 썸네일용 네모 박스 애니메이션을 추가합니다.
    <div style={{ display: 'flex', justifyContent: 'space-between' }}>
        <div style={{ flex: 1 }}>
            <div className="bw-skeleton-box" style={{ width: '70%' }}></div>
            <div className="bw-skeleton-box" style={{ width: '40%' }}></div>
        </div>
        <div className="bw-skeleton-box" style={{ width: '84px', height: '84px', borderRadius: '16px' }}></div>
    </div>
    ```

    **3. 리액션 타격감(Bouncing) 극대화 애니메이션 (Step 3.3 대응):**
    ```css
    /* 기존의 단순한 크기 변경을 넘어, 누를 때 뒤틀리며 튀어 오르는 타격감을 주입 */
    @keyframes popBouncing {
        0% { transform: scale(1) translateY(0); }
        40% { transform: scale(1.25) translateY(-4px) rotate(-8deg); }
        70% { transform: scale(1.1) translateY(2px) rotate(4deg); }
        100% { transform: scale(1) translateY(0) rotate(0); }
    }
    .bw-reaction-btn { 
        transition: all 0.2s cubic-bezier(0.25, 0.46, 0.45, 0.94); 
    }
    .bw-reaction-btn:active { 
        animation: popBouncing 0.5s cubic-bezier(0.175, 0.885, 0.32, 1.275); 
    }
    ```

### 🛠️ 수술 완료 부위 및 검증 내역

1. **썸네일 동적 렌더링 레이아웃 이식 완료**
   - 밋밋했던 텍스트형 리스트 레이아웃의 우측 영역을 분할하여, 향후 이미지 첨부 시 표시될 `84x84` 규격의 썸네일 블록(`div.bw-thumbnail`)을 신설했습니다.
2. **리액션 마이크로 애니메이션 강화 완료**
   - Phase 2의 기초적인 애니메이션을 폐기하고, 클릭 시 아이콘이 4도 가량 뒤틀리며 위아래로 역동적으로 튀어 오르는 타격감 높은 `popBouncing` Keyframes 로직으로 전면 교체했습니다.
3. **썸네일 뷰어용 스켈레톤 로딩 동기화 완료**
   - 리스트 로딩 시 텍스트 뼈대뿐만 아니라 썸네일용 네모 박스(`84x84`) 뼈대까지 깜빡거리도록 UI 스켈레톤 로직을 확장하여 시각적 안정감을 확보했습니다.
