## 🔍 2. BitWish 추천 보너스 정책의 정의 및 4대 원인 규명

### 2.1. 💡 BitWish 정규 추천 보너스 정책 (Referral Bonus Policy)
추천 보너스율(`referralBonusRate`)은 회원의 계정 계층 상태에 따라 다음과 같이 유기적이고 동적으로 누적 합산되어야 합니다.

1. **부모 연결 보너스 (Parent-Link Bonus)**:
   * 회원이 가입할 때 추천인을 등록하여 부모가 존재하는 경우, **기본 2% (0.02)의 속도 보너스**를 영구히 보장받습니다.
   * 이는 본인의 자식이 0명이더라도 영원히 유지되며, 채굴 시 실시간으로 본인의 보관함에 적립되어야 합니다.
2. **자식 추천 보너스 (Child-Referral Bonus)**:
   * 회원이 자신의 코드로 다른 회원(자식)을 가입시킨 경우, **추천인 1명당 추가로 2% (0.02)의 속도 보너스**를 받습니다.
   * 이 보너스는 부모의 존재 여부와 관계없이 가입된 실제 자식 수만큼 유기적으로 누적 합산됩니다.
3. **최종 통합 공식 (Unified Formula)**:
   * **최종 추천 보너스율** = `(부모가 존재하면 0.02, 없으면 0.00) + (본인의 실제 자식 수 * 0.02)`
   * *예시 1*: 부모가 있고 자식이 0명인 회원 = `0.02 + 0.00 = 0.02` (2%)
   * *예시 2*: 부모가 있고 자식이 1명인 회원 = `0.02 + 0.02 = 0.04` (4%로 자동 증가)
   * *예시 3*: 부모가 없고 자식이 2명인 회원 = `0.00 + 0.04 = 0.04` (4%)
   * *예시 4*: 부모가 없고 자식이 0명인 회원 = `0.00 + 0.00 = 0.00` (0%)

