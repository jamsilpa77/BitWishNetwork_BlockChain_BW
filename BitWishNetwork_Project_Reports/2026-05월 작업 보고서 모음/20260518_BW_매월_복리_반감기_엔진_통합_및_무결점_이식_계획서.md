## 1. 🔍 [상세 분석] 무결점 이식 전략 (Top-Down 데이터 흐름 설계)

사용자님의 **"기존 채굴 시스템이나 랭킹게시판 익스플로러 등 어떤 계산 형식의 모든 시스템에 어떠한 영향도 없이 수치만 딸깍 적용되도록 하라"**는 엄명은 아키텍처 관점에서 매우 정확하고 날카로운 통찰입니다.

### 1-1. 기존 시스템의 상태 (병목점)
현재 랭킹게시판, 관리자 페이지, 익스플로러는 유저의 실시간 상태를 계산할 때 데이터베이스(`MongoDB`의 `MiningState` 컬렉션)에 저장된 `currentBaseRate`(기본 채굴률)와 `currentTotalRate`(합산 채굴률) 값을 읽어옵니다. 
문제는 현재 백엔드 로직 곳곳에 `0.25`라는 문자가 하드코딩되어 있어 시간이 지나도 DB 값이 변하지 않는다는 점입니다.

### 1-2. 무결점 수복 전략 (Single Source of Truth)
랭킹 서비스나 익스플로러 파일들은 **절대 건드리지 않습니다.** 
오직 사용자에게 마이닝 데이터를 제공하고 30초마다 갱신하는 심장부인 **`MiningController.ts` 및 관련 상태 업데이트 모듈에만 "반감기 엔진(Halving Engine)"을 장착**합니다.

*   30초 주기 동기화(`syncMiningData`)가 돌 때마다, 서버는 현재 날짜(2027년 6월 이후 경과 월수)를 체크하여 현재 시점에 맞는 '정확한 반감기 기본 채굴률'을 계산합니다.
*   계산된 수치를 DB(`currentBaseRate`)에 즉각 반영합니다.
*   그러면 기존의 랭킹게시판, 어드민 페이지 등은 평소 하던 대로 DB 값만 읽어가면 되므로 **자동으로 반감된 수치가 반영**되어 사이드 이펙트(부작용) 발생 확률이 0%가 됩니다. 이것이 완벽히 톱니바퀴가 맞물리는 방식입니다.

## 2. 🛠 [상세 작업 위치 및 코드 로직] 단계별 공정 계획

### 📍 1공정: 독립된 정밀 반감기 계산 엔진(Halving Calculator) 신조
기존 코드를 오염시키지 않기 위해, 오직 날짜를 기반으로 50자리 정밀도로 반감된 수치만 계산해 반환하는 독립 유틸리티 클래스를 생성합니다.
*   **생성 위치**: `server/utils/HalvingCalculator.ts`
*   **상세 로직**:
    ```typescript
    import Decimal from 'decimal.js';

    export class HalvingCalculator {
        // 기준일: 2027년 6월 1일 AM 00:00:00 (KST)
        private static readonly HALVING_START_DATE = new Date('2027-06-01T00:00:00+09:00');
        private static readonly INITIAL_BASE_RATE = new Decimal('0.25');
        private static readonly DECAY_FACTOR = new Decimal('0.95'); // 매월 5% 감소

        public static getCurrentBaseRate(): Decimal {
            const now = new Date();
            // 2027년 6월 이전이면 무조건 0.25 반환
            if (now < this.HALVING_START_DATE) {
                return this.INITIAL_BASE_RATE;
            }

            const yearsDiff = now.getFullYear() - this.HALVING_START_DATE.getFullYear();
            const monthsDiff = now.getMonth() - this.HALVING_START_DATE.getMonth();
            const totalMonths = Math.max(0, yearsDiff * 12 + monthsDiff);

            // 0.25 * (0.95 ^ 경과월수)
            return this.INITIAL_BASE_RATE.mul(this.DECAY_FACTOR.pow(totalMonths));
        }
    }
    ```

### 📍 2공정: 마이닝 심장부(Mining Controller) 하드코딩 절제 및 엔진 이식
가장 중요한 채굴 코어에서 `0.25` 하드코딩을 전부 뽑아내고, 1공정에서 만든 `HalvingCalculator`의 리턴값을 꽂아 넣습니다.
*   **작업 파일**: `server/controllers/MiningController.ts`
*   **수술 위치**: `startMining`, `syncMiningData`, `getUserStatus` 메서드 내부.
*   **상세 로직**:
    ```typescript
    // [기존 코드 - 삭제]
    // const baseRate = new Decimal(state.currentBaseRate || '0.25');

    // [신규 코드 - 이식]
    // 1. 현재 시간에 맞는 정확한 반감기 Base Rate 계산 (톱니바퀴 1)
    const baseRate = HalvingCalculator.getCurrentBaseRate();
    
    // 2. 출석/가맹점/추천인 등 보너스 비율은 그대로 둔 채 합산 (톱니바퀴 2)
    const currentTotalRate = baseRate.mul(보너스_가산_비율);

    // 3. 계산된 값을 DB에 저장하여 랭킹 및 외부 시스템이 읽어가도록 세팅 (톱니바퀴 3)
    state.currentBaseRate = baseRate.toString();
    state.currentTotalRate = currentTotalRate.toString();
    ```

### 📍 3공정: 회원가입 및 출석 모듈의 하드코딩 잔재 제거
유저가 최초 가입하거나, 다음 날 출석 보너스를 받을 때 상태를 초기화하는 로직에도 `0.25`가 박혀있습니다. 이를 일괄 교정합니다.
*   **작업 파일 1**: `server/controllers/UserController.ts` (가입 시 지갑 생성 부분)
*   **작업 파일 2**: `server/routes/attendance.ts` (출석 갱신 부분)
*   **상세 로직**:
    초기화 시 `'0.25'` 문자를 주입하던 것을 `HalvingCalculator.getCurrentBaseRate().toString()`으로 변경하여 시스템 전역에 일관성을 강제합니다.
