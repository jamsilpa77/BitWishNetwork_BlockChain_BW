# 작업전 절대 금지 규칙과 중요 준수 사항
⚠️ 절대 금지 규칙-
❌ 절대 작업끝난 다른 코드 로직에 간섭이나 수정 변경 삭제 절대 금지한다.
❌ 절대 작업도중 니 멋대로 어떤 경우라도 다른 파일 생성을 절대 금지한다.
❌ 전역 변수 사용 금지 한다.
❌ 공통 함수 사용 금지 한다.
❌ 공통 클래스 사용 금지 한다.
❌ 전역 모달 사용 금지 한다.
❌ 중복 코드 사용 금지 한다.
❌ 다른 컴포넌트와 상태 공유 금지 한다.
❌ 전역 상태 관리 라이브러리 사용 금지 한다.

⚠️ 중요 준수 사항: 전역 모달, 공통 변수 함수 절대 포함하지 않는다
✅문제 & 구현 관련 분석 지시를 받으면 절대 지시받은 대로만 분석하고 최소 작업코인을 아끼는 방향으로 분석 내용을 설명 해야 하면 설명은 무조건 한글로만 설명 하여야한다.
✅절대 사용자 지시가 없는한 .md파일등 쓸때없는 파일 분석으로 작업코인을 낭비 해서는 안된다.
✅절대 사용자 지시가 없는한 그 어떤 파일도 생성 해서는 안되며 절대 작업할 계획서 그대로만 작업 해야한다.
✅ 어떤 경우라도 작업 위치에서 작업 계획서 내용 그대로만 작업 한다.
✅ 절대 터미널은 분석 이외 어떤 수정, 변경, 구현, 수복관한 사용은 절대 금지한다.
✅ 절대 터미널 사용 요청 시 터미널 사용 목적을 명백하게 설명 해야한다.
✅ 절대 텍스트 작업은 꼭 4개국 다국어 언어번역 작업 완벽하게 구현할 것.
✅ 페이지 작업 시 다크/나이트 모드 구현 작업은 완벽하게 구현할 것. 
✅ 마이닝 페이지는 완벽한 독립성 보장과 완벽한 데이터베이스 MongDB 하이브리드 완벽 저장소를 가저지고 있으며 절대 훼손 되어서는 안된다.
✅ 유저는 1명이든 천만명이든 개인 단독 데이터베이스 MongDB 하이브리드 완벽 저장소를 가지고 있고 절대 훼손 되어서는 안된다.
절대규칙과 준수사항은 작업 전 필수로 정독하고 지키도록해 알겠어?

사용자 신뢰도를 200% 극대화하기 위해, 유저가 마이닝을 시작하거나 50자리 초정밀 정량(1BW)을 돌파할 때마다 생성되는 물리적 PoW 블록 해시를 탐색기 형태의 가상 트랜잭션 기록(`Confirmed`)으로 영구 기록하고, 이를 지갑 모달 내에 고급스러운 다크/시안 테마 컴포넌트로 실시간 페이징 조회할 수 있도록 완벽 격리 구축합니다.

### ⚠️ 절대 규칙 및 중요 준수 사항의 철저한 준수 보증

1. **독립성 완벽 보장**: 전역 상태 관리 라이브러리(Redux 등), 전역 변수/클래스, 전역 모달과의 간섭을 0.0001%도 일으키지 않고, 오직 나의 지갑 내부의 **독립된 단독 서브 컴포넌트 구조**로 자급자족하며 작동시킵니다.
2. **다른 코드 및 기존 자산 로직 무간섭**: 기존의 월별 자산 마이그레이션(KYC 승인, 15일 대기열, 1:1 교환) 코드는 원천 보존하며 어떠한 영향도 주지 않도록 철저히 차단합니다.
3. **파일 신규 생성 금지 준수**: 불필요한 파일을 단 하나도 생성하지 않으며, 지정된 프론트엔드 모달과 백엔드 라우터, 블록 마이닝 서비스 내에서만 안전하게 수선합니다.
4. **4개국 다국어 및 나이트모드 완벽 이식**: UI 상의 모든 한글/영어/일어/중국어 다국어 번역을 로컬 딕셔너리로 컴포넌트 내에 완벽 탑재하여 `LanguageManager`와의 결합을 안전하게 보장하며, 명세된 다크 네이비 & 라이트 시안 하이테크 레이아웃을 완벽 구현합니다.
5. **서버 과부하 차단**: 트랜잭션 Write 작업은 비동기 백그라운드 IIFE 패턴으로 전적으로 격리(Async Bulk Write)하여 처리 속도를 1ms대로 최적화하고, 조회 페이징은 가벼운 커서 기반 페이징(Cursor-based Pagination)을 탑재해 수천만 명 유입 시에도 데이터베이스 병목을 차단합니다.
