## 🛑 1. [현황 분석] 왜 되던 것들이 안 되는가?

사용자님이 제시하신 이미지와 현재 전산망의 로직을 대조한 결과, 시스템이 **'박제된 데이터'**와 **'실시간 데이터'**를 구분하지 못해 발생하는 병목 현상을 식별했습니다.

- **지갑 내 "채굴 보상 내역" 실시간분 누락:** 지갑 UI가 `MonthlySettlement` DB(과거 달)만 조회하므로, 아직 말일이 되지 않아 박제되지 않은 **[이번 달의 실시간 채굴량]**은 목록에 나타나지 않는 것입니다.
- **관리자 페이지 가입자 목록 "과거 고착":** `formatDateRange` 유틸리티가 날짜 범위를 무조건 **1일 단위(am~pm)**로 고정하고 있어, 한 달 치 합산 수량이 아닌 가입 첫날의 0원 데이터만 보여주는 기술적 착오입니다.

## 🛠️ 2. [복구 및 구현 계획] (Implementation Plan & Task List)

사용자님의 비즈니스 원안(`매월 1일 00시 ~ 말일 24시`)을 시스템적으로 완벽히 보관하기 위한 각 위치별 세부 작업 리스트와 그에 따른 실행 효과(Phase)의 **[무결 정렬]**입니다.

### 2.1 지갑 UI 실시간 보상 내역 주입 (MyWallet Modal)
- **위치:** `src/components/MyWallet/MyWalletModal.tsx` 내 `MiningRewardDetails` 렌더링부.
- **작업 리스트:**
    - [ ] `miningStore`에서 현재 실시간으로 올라가는 `accumulatedReward`를 가져오는 로직 추가.
    - [ ] 박제된 내역 목록 상단에 **"2026.03 (진행 중)"** 가상 아이템을 생성하여 실시간 수치를 연결.
    - [ ] 8자리 정밀도 엔진을 동일하게 적용하여 목록 내에서도 초당 숫자가 변하는 것을 시각화.
- **[Phase 2]:** 지갑 UI 목록에 실시간 채굴 현황을 박시분과 병합하여 유저에게 실시간 무결성을 증명함.

### 2.2 Admin 페이지 정산 기간 포맷팅 정상화 (Admin Route)
- **위치:** `server/routes/admin.ts` 내 `formatDateRange` 함수 (L11-18).
- **작업 리스트:**
    - [ ] 전달받은 날짜로부터 해당 월의 **[1일 am 00:00:00]**과 **[말일 pm 23:59:59]**를 동적으로 계산하는 로직 주입.
    - [ ] 고정된 1일치 범위를 삭제하고, 해당 월 전체를 아우르는 문자열로 반환하도록 정정.
- **[Phase 1]:** 관리자 페이지의 날짜 범위를 월 단위로 확장하여, 숨겨진 한 달 치 채굴 데이터를 즉시 수면 위로 끌어올림.

### 2.3 Admin 가입자 목록 실시간 티커 보정 (Admin API)
- **위치:** `server/routes/admin.ts` 내 `recordsWithRealTime` 매핑부 (L238-257).
- **작업 리스트:**
    - [ ] `lastSyncTime`과 현재 서버 시간의 차이를 소수점 50자리 정밀도로 계산하는 `increment` 엔진 강화.
    - [ ] `MiningState` 데이터가 0으로 고착된 경우에도 서버 시간을 강제 대입하여 티커가 멈추지 않도록 보정.
- **[Phase 3]:** 지갑과 관리자 페이지의 실시간 수치가 0초의 오차도 없이 동일하게 동기화되는지 최종 전수 검증함.
