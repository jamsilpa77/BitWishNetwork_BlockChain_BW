## 1. 신규 장애 원인 분석 및 해결책 (Bug Report)

### **(1) 런타임 에러: `Cannot read properties of undefined (reading 'toLowerCase')`**
- **원인:** 백엔드 API에서 특정 가입자의 KYC 상태 데이터가 누락되어 전달될 때, 프론트엔드가 이를 고려하지 않고 `.toLowerCase()`를 강제 호출하여 화면이 충돌함.
- **해결책:** 
    - **프론트엔드:** Optional Chaining(`?.`)을 적용하여 데이터가 없을 경우 에러 없이 빈 값을 반환하도록 수정.
    - **백엔드:** 가입자 목록 조회 시 모든 결과에 기본 KYC 상태(`NOT_APPLIED`)가 포함되도록 서버 로직 방어 강화.

### **(2) 데이터 누락: 18자리 코드 찌꺼기 문제 (`BW958A...` 지갑 건)**
- **원인:** 본인 코드는 15자리로 정규화되었으나, 가입자(`BWD6CC...`)가 가입할 때 썼던 **가입 기록(`referrerCode`)**에 여전히 18자리 찌꺼기가 남아 있어 매칭에 실패함.
- **해결책:** 
    - **데이터 정규화 (소급):** DB의 `users` 및 `bonusrecords` 컬렉션을 전수 조사하여, 18자리로 기록된 모든 `referrerCode`를 15자리로 강제 정규화 집행.
    - **효과:** `BW958A...` 지갑 검색 시 그동안 찌꺼기 때문에 가려졌던 가입자 목록이 타 유저들처럼 정상 표출됨.
