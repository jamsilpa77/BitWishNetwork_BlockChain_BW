## 1. "전체 지급 보너스 (2%)" 데이터 반영 오류 수리 (0.00% 제거)

### **[오점 정밀 분석]**
1.  **백엔드(Backend) 결함**: 기존 로직이 `BonusRecord` 컬렉션만 조회하고 있었음. 실제로 추천에 따른 2% 보너스율 데이터는 `MiningState`에 저장되므로, 두 상자를 모두 열어서 합산해야 함.
2.  **프론트엔드(Frontend) 결함**: 서버가 진짜 데이터를 보내줘도 관리자 페이지 화면(`AdminPage.tsx`)이 이를 받아먹는 '입(변수)'이 막혀 있음. 서버 데이터인 `totalBonusRate`를 화면의 '전체 지급 보너스' UI와 연결해야 함.

### **[작업 계획]**
- **서버 수리**: `admin.ts`에서 `BonusRecord`와 `MiningState`를 교차 집계하여 진짜 합계 산출.
- **화면 수리**: `AdminPage.tsx`에서 서버 데이터를 수신할 상태 변수를 신설하고 UI 카드에 연동.

### **[구현될 상세 코드]**

#### **(1) 백엔드: server/routes/admin.ts**
```typescript
/* /api/admin/rewards/total 엔드포인트 수리 (양방향 합산 로직) */
// 1. 추천 보상(1BW) 합산
const rewardResults = await BonusRecord.aggregate([
    { $group: { _id: null, total: { $sum: { $toDouble: { $ifNull: ["$referralRewardStorage", "0"] } } } } }
]);

// 2. [추천인 측] 보너스율(2%p) 합산 - MiningState에서 추출
const bonusRateResults = await MiningState.aggregate([
    { $group: { _id: null, total: { $sum: { $toDouble: { $ifNull: ["$referralBonusRate", "0"] } } } } }
]);

// 3. [가입자 측] 보너스율(2%) 합산 - referrerCode 보유 유저 수 추출
const joinerCount = await User.countDocuments({ referrerCode: { $ne: null } });

// 4. 양방향 합산 처리 (추천인 합계 + 가입자 합계)
const referrerSum = bonusRateResults.length > 0 ? bonusRateResults[0].total : 0;
const joinerSum = joinerCount * 0.02;
const finalTotalRate = (referrerSum + joinerSum) * 100;

return res.json({
    success: true,
    totalIssued: rewardResults.length > 0 ? rewardResults[0].total : 0,
    totalBonusRate: finalTotalRate.toFixed(2)
});
```

#### **(2) 프론트엔드: src/components/AdminPage/AdminPage.tsx**
```tsx
/* 데이터 수신 및 UI 카드 연동 수리 */
// 1. 전체 보너스율 합계를 저장할 전용 상태 변수 신설
const [totalBonusRateSum, setTotalBonusRateSum] = useState<string>("0.00");

// 2. 서버가 보낸 진짜 데이터를 변수에 저장

    const response = await fetch('http://localhost:5001/api/admin/rewards/total');
    const data = await response.json();
    if (data.success) {
        setTotalRewardIssued(data.totalIssued);
        setTotalBonusRateSum(data.totalBonusRate); // [수리] 0.00% 원인 해결
    }
};

// 3. UI 카드에 변수 연결 (824라인 부근)
<div className="total-summary-card bonus-card">
    <span className="total-label">전체 지급 보너스</span>
    <span className="total-value" style={{ color: '#a855f7' }}>
        {totalBonusRateSum} %
    </span>
</div>
```

## 2. 홈페이지 "실시간 BW 발행률" 카운팅 엔진 수리

### **[문제 분석]**
- **현상:** 발행량과 잔여량은 실시간으로 올라가지만, '발행률'만 0.00%로 멈춰 있음.
- **분석 건:** 
    - `HomePage.tsx` 내의 실시간 티커 엔진(`startHomePageTicker`)에서 발행량(`currentIssued`)은 계산하여 올리고 있으나, UI 렌더링 시 발행률(`issuanceRate`)에 대한 애니메이션 처리가 누락됨.
    - 발행률 계산식 (`발행량 / 2.1억`)이 티커 내부에서만 돌고, 실제 화면에 보일 때는 소수점 변화를 감지하지 못하는 상태.
- **해결책:** 발행률 필드에도 `useCountUp` 애니메이션 효과를 적용하고, 초당 증분 로직을 발행량과 정비례하게 연결.

### **[작업 계획]**
- **파일:** `src/components/HomePage/HomePage.tsx`
- **로직:** `globalStats.issuanceRate`를 애니메이션 대상에 포함시키고, 소수점 4자리까지 실시간 변화가 보이도록 수정.

### **[구현될 상세 코드]**
```tsx
/* HomePage.tsx - 발행률(%) 전용 애니메이션 훅 적용 */
const animatedRate = useCountUp(globalStats.issuanceRate, 800);

// JSX 렌더링 부분 수정
<div className="card-value">
    {/* 0.00% 고정 문제 해결: 애니메이션 값이 적용된 변수 사용 */}
    {parseFloat(animatedRate).toFixed(2)}%
</div>
```
