## 🏗️ 대분류 1: 관리자 시스템 (Admin System)

### 1-1. 현재 구현 상태 (As-Is Analysis)
*   **검색:** 지갑 주소를 모르면 검색 불가. (목록 조회 불가)
*   **데이터:** 채굴 정보만 표시됨. 가입일, 추천인, KYC 정보 누락.
*   **초기화 (Reset) 로직의 치명적 결함:**
    *   **과잉 초기화:** `reset` 시 마이닝 보상뿐만 아니라, **유저가 획득한 `1BW`(추천 보상금)까지 0으로 날려버림.** (자산 소멸)
    *   **유령 잔존:** 정작 지워야 할 **숨겨진 좀비 필드(6 BW 등)**는 지우지 않음.

### 1-2. 해결 방안 (To-Be Solution)
*   **통합 조회:** `User` + `MiningState` + `BonusRecord` 3개 컬렉션을 조인하여 전체 목록(`GET /api/admin/users/all`) 제공.
*   **초기화 정책 재정립 (1BW 보존):**
    *   **초기화 대상:** 마이닝 누적 보상, 2% 추천 보너스 누적치, 출석 기록 등 "활동 기반 데이터".
    *   **보존 대상:** **`referralRewardStorage` (추천인 코드 보상 1BW)는 초기화하지 않고 영구 보존.**
    *   **완전 삭제:** `reset` 로직에 **`$unset`**을 적용하여 숨겨진 좀비 필드 영구 삭제.

## 🏗️ 대분류 2: 마이닝 및 지갑 연동 (Mining & Wallet Sync)

### 2-1. 현재 구현 상태 (As-Is Analysis)
*   **타이머:** 마이닝 페이지 숫자는 올라가지만 서버로 전송하지 않음(휘발성).
*   **지갑 화면:** 모달을 열 때 1회만 스냅샷 데이터를 가져옴. 마이닝 중이어도 숫자가 멈춰 있음.
*   **코드 구조:** 블록체인 지갑(`walletService`)과 마이닝 표시(`apiService`)가 혼재되어 있으나, 상태 관리는 분리되어 있음.

### 2-2. 해결 방안 (To-Be Solution) - ⚠️ 무결성 보장 원칙 준수
**"나의 지갑은 블록체인 P2P 지갑이며, 마이닝 표시는 부가 기능이다."**라는 원칙을 절대 준수합니다.

1.  **[비침해성] 실시간 동기화 구현 (Safe Synchronization):**
    *   **Core Logic 보존:** 지갑 생성, 키 관리, 송금, 주소 생성 등 `walletService`의 핵심 코드는 **단 1줄도 건드리지 않습니다.**
    *   **Display Layer만 연동:**
        *   지갑 모달(`MyWalletModal`)에 `RealTimeSyncService` 리스너를 추가합니다.
        *   이 리스너는 마이닝 타이머가 보내는 "채굴량 숫자"만 수신하여, 화면의 **단순 텍스트(`balance`)만 갱신**합니다.
        *   지갑의 실제 잔액(트랜잭션 기반)에는 전혀 영향을 주지 않습니다.
2.  **서버 동기화:** 마이닝 타이머가 10초마다 서버로 데이터를 전송하도록 수정하여 데이터 휘발 방지.
3.  **UI 포맷팅:** 추천 보상(정수), 채굴량(4자리) 표준화.

### 🛠️ Step 4. 지갑 화면 실시간 연동 (프론트)
**대상 파일:** `src/components/MyWalletModal/MyWalletModal.tsx`
**목표:** **지갑 핵심 로직(보안, 송금)은 건드리지 않고**, 오직 화면의 숫자만 갱신.

```typescript
// [수정] useEffect 내부
useEffect(() => {
    if (!isOpen) return;
    fetchWalletData(); // 1. 기존 데이터 로드 (유지)

    // 2. [신규] 실시간 마이닝 숫자만 구독 (안전한 리스너)
    // 이 리스너는 마이닝 페이지가 보내는 '숫자'만 받아서 화면에 그려줍니다.
    const unsubscribe = realTimeSyncService.subscribe((data) => {
        // 지갑의 실제 잔액(DB)을 건드리는 게 아니라, 화면 표시용 state만 업데이트
        if (data && data.currentIssued > 0) {
            setWalletData(prev => prev ? {
                ...prev,
                balance: data.currentIssued // 화면 갱신
            } : null);
        }
    });

    return () => {
        unsubscribe(); // 모달 닫으면 리스너 해제 (메모리 누수 방지)
    };
}, [isOpen]);
```
