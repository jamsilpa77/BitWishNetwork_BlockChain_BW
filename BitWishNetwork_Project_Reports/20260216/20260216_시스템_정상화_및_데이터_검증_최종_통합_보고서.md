## 1. 정밀 현상 분석 (Problem Diagnosis)

### 🔴 문제 1: "초기화 해도 6개가 안 지워져요" (Ghost Data 6 BW)
*   **현상:** 관리자 페이지에서 `BWD6...` 지갑을 초기화(Reset) 시켰음에도, 전체 발행량 통계에는 `6 BW`가 그대로 남아있습니다.
*   **원인 (숨겨진 데이터):**
    *   초기화 버튼은 현재 보이는 `referralRewardStorage`라는 **'새 장부'**만 지우고 있습니다.
    *   하지만 시스템의 통계 계산기(`stats.ts`)는 옛날에 쓰던 **'헌 장부'(`bonusStorage`)**까지 몰래 합산하고 있습니다.
    *   즉, 6 BW는 헌 장부에 적혀 있어서 지워지지 않고 좀비처럼 계속 합산되는 것입니다.
*   **해결책:** 초기화 버튼을 눌렀을 때, **'헌 장부'(`bonusStorage`)까지 몽땅 찾아서 강제 삭제($unset)**하도록 수정해야 합니다.

### 🔴 문제 2: "추천 보상은 정수인데 소수점 8자리가 왜 나와?" (UI Format)
*   **현상:** 1명당 1 BW를 주는 추천 보상은 딱 떨어지는 정수인데, 화면에는 `1.00000000 BW`처럼 불필요하게 길게 표시됩니다.
*   **원인 (무조건 8자리):**
    *   화면을 그리는 코드(`MyWalletModal.tsx`)에 "무조건 소수점 8자리로 보여줘라(`toFixed(8)`)"라고 쇠고집처럼 박혀 있습니다.
*   **해결책:** "보너스 보관함" 칸에는 **소수점 없이 정수(`toFixed(0)`)**로만 깔끔하게 나오도록 수정합니다.

### 🔴 문제 3: "지갑에 내 추천 코드가 안 떠요" (Missing Referral Code)
*   **현상:** '추천 보너스' 창에는 내 코드가 있는데, '나의 지갑' 창에는 `-` (없음)으로 나옵니다.
*   **원인 (통신 두절):**
    *   추천 코드는 브라우저(내 컴퓨터)에서 만들어서 저장했습니다.
    *   하지만 정작 **서버(본사 DB)**에는 이 코드를 "나 이거 만들었어"라고 알려주지 않았습니다.
    *   '나의 지갑' 창은 서버(본사)에 물어봐서 보여주는데, 서버는 코드를 들은 적이 없으니 "모름(-)"이라고 답하는 것입니다.
*   **해결책:** 코드가 생성되면 **즉시 서버에도 "내 코드 이거야"라고 저장**하도록 연결선을 이어야 합니다.

### 🔴 문제 4: "관리자가 전체 회원을 볼 수가 없어" (No Master View)
*   **현상:** 관리자 페이지에서 10만 명이 가입해도 목록을 볼 수가 없고, 지갑 주소를 알아야만 검색이 됩니다.
*   **위험성:** 이대로라면 나중에 코인을 1:1로 바꿔줄 때(에어드롭), **누가 언제 가입했는지 전수 조사가 불가능**하여 대형 사고가 날 수 있습니다.
*   **해결책:** 관리자 페이지에 **"모든 회원 명부(Total User List)"**를 엑셀처럼 쫙 뽑아주는 기능을 반드시 만들어야 합니다. (가입일시 초단위 포함)

## 2. 단계별 작업 지시서 (Step-by-Step Execution Plan)

위 문제들을 해결하기 위해 아래 순서대로 작업을 진행합니다.

### [Step 1] 유령 데이터(6 BW) 및 추천 코드 동기화 해결
*   **대상 파일:** `server/routes/admin.ts`, `server/routes/user.ts` (신규 코드 필요)
*   **작업 내용:**
    1.  관리자 리셋(`reset`) 함수에 **`$unset: { bonusStorage: "" }`** 코드를 추가하여 헌 장부를 파쇄합니다.
    2.  서버(`user.ts`)에 **`POST /update-code`** 기능을 만들어, 지갑이 추천 코드를 보내면 저장하도록 만듭니다.

### [Step 2] UI 포맷팅 정상화 (소수점 정리)
*   **대상 파일:** `src/components/MyWalletModal/MyWalletModal.tsx`, `src/components/MiningStatusModal/MiningStatusModal.tsx`
*   **작업 내용:**
    1.  `referralReward` (1 BW) 표시 부분: `toFixed(8)` ➔ **`toFixed(0)` (정수)**
    2.  `miningAmount` (채굴량) 표시 부분: `toFixed(8)` ➔ **`toFixed(4)` (기본)** + 마우스 올리면 8자리 툴팁.

### [Step 3] 관리자 통합 원장 조회 기능 개발 (에어드롭 준비)
*   **대상 파일:** `server/routes/admin.ts`, `src/pages/AdminPage.tsx`
*   **작업 내용:**
    1.  **서버:** `GET /api/admin/users/all` API 개발.
        *   기능: 모든 회원의 지갑주소, **가입일시(초단위)**, 추천인, 총 채굴량을 한 번에 조회합니다.
    2.  **화면:** 관리자 페이지에 [전체 회원 조회] 탭을 만들고, 표(Table) 형태로 데이터를 보여줍니다.

### [Step 4] 데이터 연동 복구 (마이닝 타이머)
*   **대상 파일:** `RealTimeSyncService.ts`
*   **작업 내용:**
    *   마이닝 페이지의 숫자가 올라갈 때마다, `updateCurrentMiningAmount`를 호출하여 서버와 지갑에도 "지금 이만큼 캤어"라고 실시간 동기화합니다.
