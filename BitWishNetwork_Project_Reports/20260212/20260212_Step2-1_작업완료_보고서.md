# [Step 2-1 완료] 시스템 정상화 작업 경과 및 작동 설명서

**작성자:** BitWish AI Assistant
**작업 내용:** `server/controllers/MiningController.ts` 수정 (백엔드 데이터 반환 로직 정상화)

## 1. 작업 개요 (Overview)
본 작업은 **"추천 보너스 보관함" 데이터가 프론트엔드에서 0으로만 표시되던 핵심 문제**를 해결하기 위한 **백엔드(Server) 필수 선행 작업**입니다.
프론트엔드가 사용자 정보를 요청(`/mining/status/:walletAddress`)할 때, 백엔드가 **User 테이블만 조회**하고 정작 보너스 데이터(`BonusRecord` 테이블)는 **빼먹은 채 응답**하던 결함을 수정했습니다.

## 2. 작업 전 상태 (Before)
*   **파일:** `server/controllers/MiningController.ts`
*   **코드:** `const user = await User.findOne({ walletAddress });` 만 실행.
*   **문제점:**
    *   `User` 스키마에는 `referralBonusStorage`(2% 보너스), `referralRewardStorage`(1BW 보상) 필드가 **없음.**
    *   결과적으로 백엔드는 `user` 객체만 반환하고, 프론트엔드는 해당 필드가 없으니 `undefined` 또는 `0`으로 처리함. (데이터 누락)

## 3. 작업 후 상태 (After - Current)
*   **수정 내용:** `getUserStatus` 메서드 내부에 **`BonusRecord` 모델 조회 로직**을 추가하고, `user` 객체에 해당 필드들을 **병합(Merge)**하여 반환하도록 수정함.

**[적용된 핵심 코드]**
```typescript
// 1. BonusRecord 데이터 조회
const bonusRecord = await BonusRecord.findOne({ walletAddress });

// 2. 응답 시 필드 병합
res.status(200).json({
    success: true,
    user: user ? {
        ...user.toObject(),
        // [중요] BonusRecord 값이 있으면 그것을, 없으면 '0'을 할당
        referralBonusStorage: bonusRecord?.referralBonusStorage || '0',  // 2% 채굴분
        referralRewardStorage: bonusRecord?.referralRewardStorage || '0', // 1BW 가입보상
        referralList: bonusRecord?.referralList || []
    } : null,
    miningState
});
```

## 4. 작동 원리 및 기대 효과 (How it Works)
1.  **프론트엔드 요청:** `MyWalletModal`이 열릴 때 지갑 주소로 상태 조회를 요청합니다.
2.  **백엔드 처리:**
    *   `User` 테이블에서 기본 정보(지갑주소, 2차비번 등) 조회.
    *   **[신규] `BonusRecord` 테이블에서 보관함 정보(2% 보너스, 1BW 보상) 조회.**
    *   두 데이터를 하나로 합쳐서(`Merge`) JSON으로 응답.
3.  **결과:** 프론트엔드는 `response.data.referralBonusStorage` 필드에 접근하여 **실제 DB에 저장된 보너스 수량**을 정확하게 표시할 수 있게 됩니다.
