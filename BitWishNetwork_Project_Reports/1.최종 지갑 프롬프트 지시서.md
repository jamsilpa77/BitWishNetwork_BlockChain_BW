AI 개발자용 프롬프트 지시서: BW(BitWish) Network 지갑 시스템 개발
1. 프로젝트 개요 (Project Overview)
프로젝트명: BitWishNetwork 지갑 시스템

2. BitWishNetwork 지갑 주소: BW + 40자 16진수 = 42자리 주소로 구성되고 
지갑이 생성되면 실제 P2P 실제 거래가 가능한 BitWish블록체인 지갑이 생성된다.

3. 현재 개발완성된 BiWishNetwork_BlockChain(PoW) Mainnet 프로젝트를 기반으로 모든 기능이 개발 되어야 한다.

핵심 목표: 사용자가 비수탁(Non-custodial) 방식으로 BW(BitWish) Network 블록체인 기반의 개인 지갑을 생성하고 관리할 수 있는 완전한 시스템을 구축합니다. 모든 개인 키(시드 문구)는 사용자가 직접 관리하며, 서버는 이를 저장하지 않습니다. 시스템은 실시간 노드 보상 데이터와 연동되고, 엄격한 인증 절차를 통해 자산의 송금/수신 기능을 제공합니다.

기술 스택:

블록체인: BW(Bitwish) 네트워크

핵심 라이브러리: BitWish SDK (예: BiWishNetwork-sdk for JavaScript), BIP39 Mnemonic Library

인증: Google Authenticator (OTP) 연동

2.2. 백엔드 (Backend & Blockchain Integration)
지갑 생성 로직 (POST /api/wallet/create)

요청: { "verificationWords": { "3": "apple", "8": "banana", ... } }

로직:

서버는 시드 문구를 저장하지 않으므로, 클라이언트에서 생성된 시드 문구를 기반으로 검증 단어만 받아 확인합니다. (보안상 시드 자체를 서버로 보내지 않습니다.)

검증 성공 시, 클라이언트에서 BitWish SDK를 사용하여 **Keypair(공개키, 비밀키)**를 생성합니다. 비밀키(Secret Key)는 절대 서버로 전송되지 않습니다.

생성된 **공개키(Public Key, G...)**만 서버로 전송하여 사용자 계정과 매핑하여 DB에 저장합니다.

해당 사용자가 이전에 지갑을 만든 기록이 없는지 확인하고, 첫 생성이라면 서버의 마스터 지갑에서 해당 

생성 및 서명하여 BW 네트워크에 제출합니다.

잔액 조회 로직 (GET /api/wallet/balance)

BitWish Horizon API 서버에 직접 요청하여 사용자의 공개키에 해당하는 계정 정보를 가져옵니다.

노드 보상, 추천, 출석 등으로 발생한 모든 BW 토큰의 총량을 나의 BW로 계산하여 반환합니다.

사용자의 인증 상태(KYC, OTP)를 확인하고, 정책에 따라 사용 가능 금액을 계산하여 함께 반환합니다.

트랜잭션 생성 및 전송 로직 (POST /api/wallet/transaction)

요청: { "destination": "G...", "amount": "100", "memo": "...", "otp": "123456" }

로직:

요청한 사용자의 Google OTP 코드가 유효한지 검증합니다.

가장 중요: 이 API는 트랜잭션을 **클라이언트 측에서 서명(Client-Side Signing)**하는 것을 전제로 설계되어야 합니다.

백엔드는 트랜잭션의 기본 구조(Source Account, Destination, Amount, Fee 등)를 만들어 클라이언트에 전달합니다.

클라이언트는 사용자가 안전하게 보관 중인 **비밀키(또는 시드 문구에서 파생된 키)**를 사용하여 해당 트랜잭션에 서명합니다.

서명된 트랜잭션(XDR 형태)을 다시 백엔드로 보내고, 백엔드는 이를 BitWish 네트워크에 최종 제출합니다. 이는 서버가 사용자의 비밀키를 절대 알 수 없게 하는 핵심 보안 조치입니다.

지갑 관리 로직

OTP 재등록: 시드 문구 확인 절차를 다시 거쳐 성공하면, 기존 OTP 설정을 초기화하고 새로 등록할 수 있는 API를 제공합니다.

새 지갑 만들기:

사용자가 새 지갑 만들기를 최종 확인하면, DB에 저장된 기존 공개키를 비활성화(또는 삭제)하고 새로운 공개키를 등록할 수 있도록 허용합니다.

3. 데이터 모델 정의 (Data Model Definition)
Users Table (기존 모델에 추가):

wallet_public_key (Varchar, Nullable)

is_wallet_created_before (Boolean, Default: false)

google_otp_secret (Varchar, Encrypted, Nullable)

kyc_level (Integer, Default: 0)

Transactions Table (BW 블록 익스플로러용):

hash (Primary Key, Varchar)

from_address (Varchar)

to_address (Varchar)

amount (Decimal)

memo (Varchar, Nullable)

created_at (Timestamp)

4. 최종 결과물 (Final Deliverables)

클라이언트 측 서명을 포함한 최고 수준의 보안 아키텍처 구현.

모든 사용자 플로우(생성, 확인, 송금, 복구)가 매끄럽게 동작하는 UI/UX.

실시간 잔액 업데이트 및 트랜잭션 내역이 BW 블록 익스플로러와 즉시 동기화되는 기능.

- **비수탁 지갑**: BitWishNetwork블록체인 완전 개발 성공으로 BitWish기준으로 현재 개발된 BitWish블록체인

  - 24단어 시드 문구(BIP39) 생성.
  - 서버는 비밀키 저장 안 함, 클라이언트 측에서 키페어 관리.
  - KYC/OTP로 사용 활성화,
  - 시드 문구 복구는 구글 OTP(TOTP) 인증 필수.

비수탁 지갑 시스템
**목표**: 24단어 시드 문구(BIP39) 생성, KYC/OTP 통합, 시드 문구 복구 시 구글 OTP 필수.  
**프론트엔드 (UI/UX)**:
- **가입 즉시 지갑 생성**:
  - 이메일 OTP 인증 완료 후 `/wallet` 페이지로 이동.
  - 클라이언트에서 `BitWish-sdk`로 키페어 생성(`Keypair.random()`), 24단어 시드 문구 생성(`bip39`).
  - 첫 가입 시 `t('wallet.congrats')` 토스트(4초).
- **지갑 페이지 (`/wallet`)**:
  - **상단 오른쪽**: `t('wallet.getSeed')` 버튼, 클릭 시 시드 문구 모달 표시.
  - **시드 문구 모달**:
    - 제목: `t('wallet.seedPhrase')`.
    - 표시: 24단어 시드 문구(12개씩 2열 X, 현재 BW 그대로 사용 하면됨 O, MUI Grid).
    - 경고: `t('wallet.seedWarning')` (빨간색, MUI Alert severity="warning").
    - 버튼: 복사(📋, `navigator.clipboard.writeText`), 취소, 확인.
    - 확인 클릭 시 모달 닫힘, 시드 문구는 `localStorage`에 AES-256 암호화 저장(구글 OTP 비밀키 사용).
  - **지갑 대시보드**:
    - 공개키: QR 코드(zxing), 복사 버튼(📋).
    - 잔액: `t('wallet.total')`: X.XXXX BW, `t('wallet.available')`: X.XXXX BW.
    - 안내: KYC 미완료 시 `t('wallet.availableNote')` (MUI Alert severity="info").
    - 최근 트랜잭션: 최대 5개(TXID, 금액, 날짜, MUI Table).
    - 버튼:
      - 송금받기(↓, 파란색): 공개키/QR 모달.
      - 송금하기(↑, 빨간색): 주소/금액/메모 입력, 구글 OTP 검증.
      - KYC 신청: `/wallet/kyc`로 이동.
      - OTP 설정: `/wallet/otp`로 이동, 구글 OTP QR 코드 표시.
      - 시드 문구 복구: `/wallet/recover`, 구글 OTP 입력 후 시드 문구 표시.
- **KYC/OTP 통합**:
  - **KYC 신청** (`/wallet/kyc`):
    - 입력: 모바일(`+821012345678`), 이메일, 소셜 ID(선택).
    - 버튼: `t('wallet.kycSubmit')`, 제출 후 관리자 승인 대기.
    - 승인 시: `Users.kyc_level = 1`, 사용 가능 잔액 해제.
  - **구글 OTP**:
    - 설정 (`/wallet/otp`): `speakeasy`로 TOTP 비밀키 생성, QR 코드(zxing) 표시.
    - 검증: 송금, 시드 문구 복구 시 6자리 OTP 입력.
    - 복구: `/wallet/recover`에서 OTP 검증 후 시드 문구 표시.

**백엔드 (API)**:
- **POST `/api/wallet/create`**:
  - 요청: `{ "email": "user@example.com" }`
  - 로직:
    1. 클라이언트에서 `BitWish-sdk`로 키페어 생성, 공개키 전송.
    2. MongoDB `Users.wallet_public_key`에 저장.
  - 응답: `{ "success": true, "publicKey": "BW..." }`
- **GET `/api/wallet/balance`**:
  - 쿼리: `?address=G...`
  - 로직: Horizon API(`/accounts/G...`)로 잔액 조회, KYC 상태 적용.
  - 응답: `{ "totalBW": 10.5, "availableBW": 5.0, "transactions": [...] }`
- **POST `/api/wallet/transaction`**:
  - 요청: `{ "destination": "G...", "amount": "100", "memo": "Payment", "otp": "123456" }`
  - 로직:
    1. `speakeasy`로 OTP 검증.
    2. 클라이언트에서 시드 문구로 트랜잭션 서명(`BitWish-sdk`).
    3. 서버가 XDR을 Horizon API(`/transactions`)로 제출.
  - 응답: `{ "success": true, "hash": "tx_hash" }`
- **POST `/api/wallet/kyc`**:
  - 요청: `{ "mobile": "+821012345678", "email": "user@example.com", "socialId": "ID123" }`
  - 로직: MongoDB `Users.kyc_data`에 저장, 관리자 승인 대기.
  - 응답: `{ "success": true, "status": "pending" }`
- **POST `/api/wallet/otp/setup`**:
  - 로직: `speakeasy`로 TOTP 비밀키 생성, QR 코드 반환(zxing).
  - 응답: `{ "success": true, "qrCode": "data:image/png;base64,..." }`
- **POST `/api/wallet/otp/verify`**:
  - 요청: `{ "otp": "123456" }`
  - 로직: TOTP 검증, `Users.google_otp_secret` 저장.
  - 응답: `{ "success": true }`
- **POST `/api/wallet/recover`**:
  - 요청: `{ "otp": "123456" }`
  - 로직: OTP 검증 후 AES-256으로 암호화된 시드 문구 복호화.
  - 응답: `{ "success": true, "seedPhrase": "word1 word2 ..." }`

**데이터 리셋 방지**:
- 시드 문구: AES-256으로 암호화(구글 OTP 비밀키 사용), `localStorage` 저장.
- 잔액/트랜잭션: Horizon API(`/accounts`, `/transactions`)로 실시간 동기화.
- Redis 캐싱: `wallet:{address}` 키로 잔액/상태 캐싱(TTL 1h).

- **P2P 네트워크**: WebRTC 기반 `libp2p`로 트랜잭션 전파, 블록 동기화, SCP 합의.

반감기 정책 (Halving Policy) - 수정된 로직
정책: baseRate는 매년 정해진 비율만큼 감소하며, 이 연간 감소량은 12개월에 걸쳐 매월 균등하게 적용됩니다.

1년차: 연초 대비 연말에 **50%**가 되도록, 매월 약 4.17%씩 점진적으로 감소.

2년차: 연초(1년차 말) 대비 연말에 **45%**가 되도록, 매월 점진적으로 감소.

3년차: 연초(2년차 말) 대비 연말에 **40%**가 되도록, 매월 점진적으로 감소.

이후 매년 연간 감소율이 5%p씩 줄어들어 0%가 될 때까지 적용됩니다.

계산 로직 (AI 필독):

현재가 프로젝트 시작일로부터 몇 년차, 몇 개월차인지 계산합니다.

현재 연도의 시작 시점 baseRate와 다음 연도의 시작 시점 baseRate를 계산합니다.

rate_start_year1 = initialBaseRate

rate_start_year2 = rate_start_year1 * (1 - 0.50)

rate_start_year3 = rate_start_year2 * (1 - 0.45)

rate_start_year4 = rate_start_year3 * (1 - 0.40)

...

현재 연도의 '총 감소량'을 계산합니다.

total_reduction_this_year = rate_start_this_year - rate_start_next_year

'월별 감소량'을 계산합니다.

monthly_reduction_amount = total_reduction_this_year / 12

현재 월의 최종 baseRate를 계산합니다.

currentBaseRate = rate_start_this_year - (monthly_reduction_amount * (current_month_of_year - 1)) (단, current_month_of_year는 1~12 사이의 값)

 * @param {Date} projectStartDate - 프로젝트 시작일
 * @param {Date} currentDate - 현재 날짜
 * @returns {number} 현재의 기본 채굴률
 */
function calculateCurrentBaseRate(initialBaseRate, projectStartDate, currentDate) {
  const totalMonthsElapsed = (currentDate.getFullYear() - projectStartDate.getFullYear()) * 12 + (currentDate.getMonth() - projectStartDate.getMonth());
  const currentYearIndex = Math.floor(totalMonthsElapsed / 12); // 0부터 시작 (0 = 1년차)
  const currentMonthOfYear = (totalMonthsElapsed % 12) + 1;

  let rateStartThisYear = initialBaseRate;

  // 현재 연도의 시작 Rate까지 계산
  for (let i = 0; i < currentYearIndex; i++) {
    const annualReductionPercent = Math.max(0, 0.50 - i * 0.05);
    rateStartThisYear *= (1 - annualReductionPercent);
  }

  // 다음 연도의 시작 Rate 계산
  const nextYearAnnualReduction = Math.max(0, 0.50 - currentYearIndex * 0.05);
  const rateStartNextYear = rateStartThisYear * (1 - nextYearAnnualReduction);

  const totalReductionThisYear = rateStartThisYear - rateStartNextYear;
  const monthlyReductionAmount = totalReductionThisYear / 12;

  const currentBaseRate = rateStartThisYear - (monthlyReductionAmount * (currentMonthOfYear - 1));

  return currentBaseRate;
}
