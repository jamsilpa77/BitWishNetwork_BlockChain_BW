# 추천 보너스 부모-자식 트리(Tree) 연결 무결성 아키텍처 및 구현 보고서

## 1. 기존 발생 문제의 뼈대적 분석과 근본적 한계
관리자님의 준엄한 지적을 깊이 통감합니다. 단순히 "UI 상단에 코드 한 줄 넣겠다"는 얕은 발상이었습니다. 
현재 상황은 단순한 오류가 아니라 **"데이터베이스에 점(Node)으로만 존재하는 사용자들을 어떻게 무한한 상속의 끈(Chain)으로 영구적으로 묶어주느냐"** 하는 아키텍처(구조)의 부재에서 파생된 치명적인 결함이었습니다. 
이를 해결하기 위해, 데이터 계층부터 프론트엔드 계층까지 하나로 관통하는 완벽한 래퍼럴 트리(Referral Tree) 구조를 설계했습니다.

## 2. 끊어지지 않는 영구적 상속 구조 설계 (Deep Architecture)

누구라도 가입하는 순간 부모를 절대 잃지 않고, 본인 역시 타인의 부모가 되어 무한 계승되는 **3단 보장 시스템**을 구축합니다.

### A. 출생의 보장 (가입 즉시 데이터 고착화 - **Phase 11 이식 완료**)
과거 10명이 실종되었던 참사는, 사용자가 '지갑 주소'를 잘못 입력했을 때 시스템이 부모를 연결해주지 못하고 튕겨내 버린 구조적 허점 때문이었습니다.
이제 가입 데이터 처리 시 `$or` 다중 검색망을 통해 가입자가 엉터리로 코드를 입력하더라도 **서버가 부모의 `myReferralCode`와 `walletAddress`를 쌍끌이 전수조사**하여 진짜 부모를 찾아내 강제로 묶어버립니다. 즉, **미아 발생 생성을 원천 차단**하는 데이터 무결성을 확보했습니다.

### B. 뿌리의 고정 (내 부모가 누구인지 역추적 결합 - **신규 백엔드 이식**)
나의 지갑을 열어 [가입자 목록]을 호출할 때, 서버(`referral.ts`)는 더 이상 자식(referralList)만 가져오지 않습니다.
내 도큐먼트의 진실의 방인 `User.referrerCode`를 역추적하여 나를 낳아준 진짜 부모 객체(Parent Info)를 동적으로 창조해 냅니다. 이 객체를 모든 자식 데이터보다 앞선 **배열의 [0번째 인덱스]에 콘크리트처럼 타설**(`unshift`)합니다. 이로써 나의 응답 데이터 구조는 무조건 **[ 0번: 내 부모, 1번~N번: 내 자식들 ]** 로 확고하게 굳어집니다.

### C. 자손의 수호 (Healer 엔진의 실시간 병합 - **정상 작동 중**)
동시에, 가입 당시 혹여라도 지연되어 끊어진 자식들이 있다면 Healer 엔진이 `User` 컬렉션을 전수 스캔하여 누락된 내 새끼들을 즉시 `BonusRecord.referralList`에 영구 병합(Upsert)해냅니다.

## 3. UI/UX의 직관적 계층 분리 (지시사항 2-4, 2-5 구현 로직)

위에서 가공되어 내려온 강력한 다단계 배열을 프론트엔드(`MyWalletModal.tsx`)에서 색채와 형태로 완벽히 분리해 렌더링합니다.

*   **[최상단 부모 영역 / 지시사항 2-4]**
    *   **식별:** 서버가 붙여준 `isParentRow: true` 플래그를 감지합니다.
    *   **렌더링:** 글자 색상을 강렬한 **빨간색(#DC2626)**으로 굵게 처리합니다.
    *   **코드 원형 존치:** 지갑 주소를 `BW...` 형식으로 점 처리하여 자르는 기존 룰을 무시하고, 부모 코드의 원본 텍스트 전부를 최상단에 우뚝 솟게 고정 출력합니다.
    *   **상태 값:** 가입날짜, 지급(1BW), 지급(2%), 승인(KYC) 등은 모두 기존 양식의 열(Column)에 맞추어 오와 열이 틀어지지 않게 완벽히 동일한 구조로 출력합니다.

*   **[하단 자식 영역 / 지시사항 2-5]**
    *   **식별:** `isParentRow` 플래그가 없는 나머지 무한대의 자식들을 렌더링합니다.
    *   **렌더링:** 기존 묻혀있던 검은색 텍스트를 모두 푸른빛의 **파란색(#2563EB)**으로 일괄 변경합니다.
    *   **압축 출력:** 좁은 화면을 고려하여 기존 방식대로 `BW+6자리+...` 축약 방식을 적용하며, 모든 보상 상태는 시스템의 진실과 정확히 일치하여 출력됩니다.

## 4. 최종 결론
이 구조가 이식되는 순간, 사용자는 로그인 즉시 **1대(나의 붉은 부모) - 2대(나 자신) - 3대(나의 푸른 자식들)** 로 이어지는 자신의 정확한 위상을 직관적으로, 그리고 단 한 명의 누락도 없이 보게 됩니다.
