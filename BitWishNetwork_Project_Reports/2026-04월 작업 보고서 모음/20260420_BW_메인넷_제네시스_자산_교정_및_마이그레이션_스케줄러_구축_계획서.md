# 20260420 BW 메인넷 제네시스 자산 교정 및 마이그레이션 스케줄러 구축 계획서

## 1. 토큰 총 발행량(210억) 제네시스 분할 입금 교정

### [문제점]
코인 생태계가 처음 켜지는 순간, 단 1개의 '재단 통장(BitWish-Foundation)'으로 210억 개가 모두 입금되는 심각한 오류가 박혀 있습니다.

### [개선 및 구현 방안]
*   처음 코인이 태어나는 순간, 총 발행량 210억 개를 곧장 3가지 몫으로 정확하게 찢어서 별도의 시스템 통장 주소로 각각 송금해 버립니다.
    1.  **개인채굴자 전용 통장 (65%):** 136.5억 개 송금
    2.  **파트너/상점 전용 통장 (15%):** 31.5억 개 송금
    3.  **관리/재단 관리 통장 (20%):** 42억 개 송금
*   **작업 파일 위치:** `src/engine/BitWishBlockchain.ts` 에 있는 블록체인 초기화 함수(`initialize`) 시작 로직 전부 해체 후 재결합.

## 2. KYC 승인 통과자의 15일 대기 자동 지급기 (스케줄러)

### [문제점]
채굴된 가상 포인트가 실제 코인으로 변환되어 송금되기 전, 15일간의 안전 심사 대기 기간(타임락)을 무인으로 체크하고 발송하는 장치가 없습니다.

### [개선 및 구현 방안]
*   **딱지표 붙이고 대기실 이동 (DB 저장):** 관리자가 유저의 KYC 확인 승인 버튼을 누르는 즉시, 백엔드 DB 대기실에 유저 채굴량 기록과 함께 "발송 기한: 오늘 날짜의 15일 뒤"라는 시간 딱지를 붙여 조용히 냅둡니다.
*   **매일 자정 자동 순찰대 가동:** 메인 서버에 봇(Bot) 스크립트를 하나 심습니다. 이 로봇은 매일 밤 자정 정각마다 대기실을 순찰합니다.
*   **실제 코인 무인 발사:** 순찰 로봇이 딱지를 확인하여 15일 기한이 도래한 유저를 발견하면, 앞선 5번의 '개인채굴자 통장(65%)'에서 실제 코인을 유저의 지갑으로 자동 입금시킨 후 기록을 완료 마킹합니다.
*   **작업 파일 위치:** 백엔드 서버쪽 `cron-scheduler.ts` 파일 신규 스크립트 작성 및 몽고 DB 데이터베이스 연결.

## 3. 초프리미엄 빗위시 경제 통합 대시보드 (Unified Dashboard) 구축 계획

### [분석 및 개선 철학]
*   **실패 원인 분석:** 이전의 작업물은 가독성을 무시한 단순 압축으로 인해 정보 전달력이 상실됨. 특히 수치 중복과 폰트 크기 불균형으로 빗위시의 품격을 훼손함.
*   **지향점:** "압축하되 숨통이 트이는 디자인." 발행량(65:15:20)과 기금(6:4)을 하나의 고품격 유리 패널에 **좌우 대칭형**으로 배치하여 기술적 신뢰도와 심미성을 동시에 확보함.

### [상세 작업 계획]
*   **하이브리드 그리드 아키텍처 (Hybrid Grid):**
    1.  **좌측 (Fixed Supply Center):** 상단에 총 발행량 캡슐을 배치하고, 하단에 [채굴65 / 파트너15 / 재단20] 통계를 수직 리스트 형식이 아닌 **'세련된 3분할 카드'**로 정렬.
    2.  **우측 (Live Treasury Dashboard):** 실시간으로 쌓이는 생태계 기금(60%)과 재단 기금(40%)을 **네온 게이지 바(Neon Gauge Bar)**로 제작하여, 기금이 차오르는 모습을 시각적으로 실시간 중계함.
*   **기술적 세부 마감 (Technical Finishing):**
    1.  **중복 0% 정책:** 라벨 내 수치 표기 방식 전면 재검토 및 중복 출력 원천 차단.
    2.  **레이어드 아웃라인:** 각 섹션에 미세한 아웃라인 광채를 부여하여 다크 모드에서의 깊이감을 유저가 직접 체감하도록 설계.
    3.  **지갑 생성 수 & 버튼 재배치:** 겹침이 발생했던 하단 영역을 플렉스 박스로 재정렬하여 여유 있는 클릭 영역 확보.
*   **작업 파일 위치:** 
    *   `src/components/Explorer/ExplorerPage.tsx`: 좌우 대칭형 엔진 구조로 소스 코드 전면 리팩토링.
    *   `src/components/Explorer/ExplorerPage.css`: 네온 게이지 효과 및 하이엔드 유리 질감 CSS 주입.
## 초프리미엄 빗위시 경제 통합 대시보드 (Unified Dashboard) 구축 계획 작업 완료 상세 보고서:

### 1. 하이브리드 경제 아키텍처 (Hybrid Grid Architecture) 실현
*   **좌우 대칭형 배치:** 발행량 정보(좌)와 수수료 기금 정보(우)를 1:1 대칭으로 배치하여 정보 간 시각적 간섭을 원천 차단하고 구조적 안정을 확보했습니다.
*   **수직 부피 획기적 축소:** 기존의 나열식 리스트 구조를 전면 해체하고 그리드 시스템을 도입하여, 유저가 스크롤 없이도 전체 경제 지표를 한눈에 파악할 수 있는 고밀도 UX를 완성했습니다.

### 2. 자산 배분 무결성 확보 및 시각적 오류 교정 (Asset Integrity)
*   **공급량 과잉 카운팅 제거:** 시각적 혼란을 주던 '생태계 80%' 항목을 삭제하고, 실제 작업 예정인 [채굴65 / 파트너15 / 재단20]의 3강 체제만 노출하여 합계가 정확히 100% 임체적으로 증명되도록 조치했습니다.
*   **라벨 중복 표기 원천 차단:** 번역 데이터와 코드 간의 충돌로 발생하던 퍼센티지 중복 표기(65% 65%) 문제를 정밀 교정하여 단일하고 명확한 텍스트 수치만 출력되도록 처리했습니다.

### 3. 하이엔드 테크니컬 비주얼 및 디테일링 (Premium UX)
*   **실시간 네온 게이지(Neon Gauge):** 6:4 비율로 적립되는 수수료 기금을 실시간으로 차오르는 네온 스타일의 프로그레스 바 형태로 제작하여 데이터의 생동감을 극대화했습니다.
*   **고광택 프리미엄 마감:** 유리 질감(Glassmorphism) 패널과 미세한 수평 스캔 라인 애니메이션을 적용하여 대시보드에 살아있는 기술적 신뢰도를 부여했습니다.
*   **JetBrains Mono 폰트 최적화:** 모든 수치 데이터에 해당 폰트를 적용하여 고도의 전문성과 시각적 정돈감을 확보했습니다.

### 4. 사후 무결성 검증 및 안정화 (Quality Assurance)
*   **CSS 문법 안정화:** 마감 과정에서 발생했던 오타 및 신택스 오류를 철저히 검증하여 시스템 전체의 스타일 시트 안정성을 100% 복구했습니다.
*   **테마 및 다국어 무결성:** 4개국 언어의 각기 다른 텍스트 길이에도 레이아웃이 붕괴되지 않는 유연함을 갖추었으며, 다크 모드에서의 선명도를 최상급으로 조율했습니다.
