## 1. 다국어 구현 모델 시스템 분석 (Benchmarking)

### [모범 사례] CreateWalletModal.tsx
*   **특징:** `LanguageManager` 인스턴스를 내부에서 관리하며 컴포넌트 독립성을 100% 유지함.
*   **핵심 로직:** `const t = (key: string) => languageManager.getTranslation(key, currentLanguage);`
    *   언어 코드(`currentLanguage`)를 번역 함수의 인자로 직접 주입하여 **개별 유저의 언어 변경을 지연 없이 즉각 반응**하도록 구현됨.
    *   중앙 제어 없이 유저의 세션에만 의존하는 '완벽 독립 저장소' 철학에 부합함.

## 2. 니모닉 파편 보안 인증 모달 결함 정밀 진단

이미지 1, 2번에서 확인된 번역 미구현 현상의 원인을 분석한 결과, 다음 구역이 수술 대상입니다.

### [결함 구역 1] 텍스트 하드코딩 (Hardcoded Strings)
*   **위치:** `MnemonicAuthModal.tsx` 내 UI 안내 문구 및 에러 메시지 일부.
*   **현상:** `getTranslation` 함수가 적용되어 있으나 연동된 `LanguageManager.ts` 데이터 상에 실제 번역문이 누락되어 한국어 기본값이 출력됨.

### [결함 구역 2] 어순 및 숫자 치환 로직 (Interpolation)
*   **위치:** 니모닉 단어 번호 표시 구역 (`{idx}번 단어`)
*   **현상:** 한국어 어순(`숫자 + 번 단어`)이 고정되어 있어 영어(`Word #숫자`) 등 타 언어 적용 시 문법적 오류 발생. 현재 구조로는 유저가 언어를 바꿔도 자연스러운 문장 구성이 불가능함.

## 3. 단계별 초정밀 구현 로직 및 계획

### [1단계] 번역 데이터 구조 보강 (LanguageManager.ts)
*   **목표:** 니모닉 모달에 필요한 모든 키(Key)를 4개국어별로 완벽 동기화.
*   **세부 내용:** 
    *   `ko`, `en`, `ja`, `zh` 섹션에 `mnemonicAuth` 오브젝트 데이터 전수 검수 및 삽입.
    *   특히 영어의 `cacheGuide`(경고 문구) 등 긴 문장이 UI를 깨뜨리지 않도록 최적화된 번역문 선정.

#### 🚩 [1단계 실행 완료 보고 - 2026-04-16]
*   **수행 결과:** `LanguageManager.ts` 내 4개국어 `mnemonicAuth` 키(15개) 동기화 완료.
*   **특이 사항:** 누락되었던 영어(`en`) 데이터 전체 복구 및 어순 교체를 위한 인프라(`wordLabel`) 구축 완료.
*   **상태:** 데이터 준비 완료. 2단계 컴포넌트 연결 대기 중.

### [2단계] 모달 컴포넌트 수술 (MnemonicAuthModal.tsx)
*   **목표:** 독립성 원칙을 유지하며 번역 유틸리티 최적화.
*   **세부 내용:**
    1. `const t = (key) => ...` 단축 함수 도입으로 오타 방지 및 가독성 확보.
    2. 생성자(`new`) 의존성 해결을 위해 `useMemo`를 통한 언어 인스턴스 고정.
    3. `placeholder` 및 `error` 메시지 상태값을 언어 변경 시 즉시 재랜더링되도록 트리거 보강.

#### 🚩 [2단계 실행 완료 보고 - 2026-04-16]
*   **수행 결과:** `MnemonicAuthModal.tsx` 내 모든 하드코딩 제거 및 `useMemo` 기반 번역 엔진 이식 완료.
*   **특이 사항:** `currentLanguage` 프로퍼티와 `t()` 함수를 직결하여 실시간 언어 전환 성능 최적화 완료.
*   **상태:** 컴포넌트 수술 완료. 3단계 어순 최적화 대기 중.

*   **세부 내용:** 
    *   예: `mnemonicAuth.wordLabel` 키의 값을 `en`에서는 `Word #{index}`로, `ko`에서는 `{index}번 단어`로 정의하여 어순을 유동적으로 제어.

*   **특이 사항:** 각 언어의 고유 문법(Preposition, Word Order)을 100% 충족하는 초정밀 UI 구현 완료.

## 4. 오류 예측 및 리스크 방어 전략 (Safety Guide)

1.  **동적 데이터 꼬임 방지**: 니모닉 검증 인덱스가 생성되는 시점과 번역이 적용되는 시점의 불일치를 방지하기 위해 `indices` 상태값을 번역 렌더링 최하단에 배치.
2.  **독립 저장소 안정성**: 다국어 작업 중 유저의 MongoDB 하이브리드 저장소에 담긴 시드 정보나 주소 정보가 오염되지 않도록 **UI 레이어와 데이터 레이어를 철저히 분리**하여 집도.
3.  **예외 상황**: 유저가 지갑을 처음 열 때 언어를 도중에 바꾸더라도 니모닉 인증 세션이 만료되지 않도록 상태 보존(State Persistence) 최적화.
