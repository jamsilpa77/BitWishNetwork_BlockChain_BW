## 3. [예정] [파트 C] 지갑 인증 모달(`WalletAuthModal`) 다국어 결함 수술 계획

### 🛠️ 분석 결과 (결함 진단)
현재 지갑 인증(시드문구 입력) 모달이 언어 변경 로직을 전혀 타지 않는 이유는 다음과 같은 3중 복합 결함 때문입니다.
1.  **[로직]** `HomePage.tsx`에서 모달 호출 시 현재 선택된 언어 정보(`currentLanguage`)를 Prop으로 넘겨주지 않아 모달이 한국어 기본값에 고정됨.
2.  **[데이터]** `LanguageManager.ts` 내 일본어(`ja`) 사전에 지갑 인증 관련 번역 객체(`walletAuth`)가 완전히 누락됨.
3.  **[하드코딩]** `WalletAuthModal.tsx` 내부에 '처리 중...', '지갑 복구 실패' 등의 메시지가 한국어 리터럴로 직접 박혀 있음.

### 🛠️ 작업 위치 및 계획
1.  **[Data] `LanguageManager.ts`:** 일본어 사전에 `walletAuth` 객체 주입 및 4개 국어 공통 메시지(로딩/에러) 보강.
2.  **[Link] `HomePage.tsx`:** `686행`의 `<WalletAuthModal />`에 `currentLanguage={currentLanguage}` Prop 주입.
3.  **[UI] `WalletAuthModal.tsx`:** 내부 하드코딩을 제거하고 Prop으로 받은 언어팩을 사용하여 동적 번역 구현.

### 🚩 [파트 C 작업 완료 상세 보고서 - 2026-04-18]
*   **작업 대상:** `LanguageManager.ts`, `HomePage.tsx`, `WalletAuthModal.tsx`
*   **상세 설명:** 
    1.  **[Data] 언어 사전 완벽 복원 및 에러 픽스:** 한국어/영어/중국어 사전의 `walletAuth` 객체에 하드코딩 교체용 메시지(`processing`, `recoveryFailed`)를 각 언어에 맞게 정밀 번역하여 주입 완료함.
        *   *특이사항:* 기존 보고서 진단 시 일본어(`ja`) 사전에 `walletAuth`가 누락된 것으로 오판되었으나, 전수 스캔 결과 파일 상단에 잘못된 위치로 은닉되어 있음을 식별함. 상단에 존재하는 원본 객체 내부에 누락된 변수만 정밀 주입하여 `TS1117`(중복 선언) 컴파일 에러를 완벽히 분쇄함. 추가로, 일본어 사전에 진짜로 증발해 있던 `miningAuth`와 `secondPassword` 번역 객체 전체를 색출하고 영문 베이스로 번역해 추가 이식함.
    2.  **[Link] 언어 파이프라인 개통:** `HomePage.tsx` 685행 부근 `<WalletAuthModal />` 컴포넌트 호출부에 `currentLanguage={currentLanguage}` Props를 주입하여, 메인 시스템의 언어 상태와 모달 창이 실시간으로 동기화되도록 끊어진 언어 연결고리를 복원함.
    3.  **[UI] 하드코딩 암 제거 및 다국어 치환:** `WalletAuthModal.tsx` 내부에 강제 삽입되어 있던 한글 하드코딩(`'처리 중...'`, `'지갑 복구에 실패했습니다...'`)을 드릴로 뚫어내고, 시스템 연동 사전 변수인 `t('walletAuth.processing')`, `t('walletAuth.recoveryFailed')`로 100% 치환함. (지갑 복구 코어 로직은 단 한 구절도 건드리지 않음)
