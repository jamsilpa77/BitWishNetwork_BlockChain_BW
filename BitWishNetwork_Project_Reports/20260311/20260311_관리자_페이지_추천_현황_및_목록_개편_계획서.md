## 1. 지시 사항 정밀 분석

### **(1) 💎 [추천 보상 현황] 레이아웃 개편**
- **현황:** 전체 지급 보상(BW)이 단독으로 넓게 자리 잡고 있음.
- **개편:** 중앙 요약 카드를 **50:50으로 분리**하여 좌측에 [전체 지급 보상], 우측에 [전체 지급 보너스 (%)] 창을 배치.
- **데이터:** 실시간 채굴 데이터와 연동하여 보너스 %가 즉시 합산 반영되도록 백엔드-프론트엔드 통신 강화.

### **(2) 🎁 [가입자 목록 관리] 메뉴 및 UI 최적화**
- **명칭 변경:**
    - 메뉴 탭: "추천 보너스" → **"가입자 목록"**
    - 페이지 제목: "🎁 추천 보너스 관리" → **"가입자 목록 관리"**
    - 입력창 문구: "추천인 지갑 주소 입력" → **"지갑 주소 입력"**
    - 지갑 주소 입력 창 우측 년, 월 선택 버튼 **"제거 필수"**
    - 검색 버튼: "월별 검색" → **"주소 검색"** (입력창 우측 배치)
    - **목록 노출 로직:** 지갑 주소 입력 후 검색 버튼 클릭 시, **검색한 주소만 목록에 출력**.
    - 목록 제목: "✅ 3월 추천 보너스 현황" → **"가입자 정보 확인"** (주소 검색 시에도 유지)

### **(3) 🔍 [실시간 채굴량] 멈춤 현상 해결 (핵심)**
- **문제점:** "전체 가입자 총 채굴량" 수치가 API 호출 시점의 고정값으로 정지되어 있음.
- **해결 방안:** 
    - 관리자 페이지 내부의 `useEffect`에 **1초 단위 인터벌(setInterval) 타이머** 구축.
    - 각 가입자의 초당 보너스 가산 수치를 합산하여 화면의 숫자가 실시간으로 올라가는 것을 시각적으로 구현.
