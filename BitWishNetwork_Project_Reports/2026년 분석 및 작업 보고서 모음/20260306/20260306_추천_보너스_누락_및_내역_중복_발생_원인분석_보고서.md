## 1. [장애 실태 보고]

사용자님의 제보를 바탕으로 시스템 내부 로직을 전수 조사한 결과, 다음 두 가지 치명적 결함이 확인되었습니다.

1.  **합산 오류**: 가입 시 받은 기본 2% 보너스가 추천인 영입 후 계산 과정에서 합산되지 않고 덮어씌워짐 (4%가 되어야 하나 2%로 표시).
2.  **데이터 중복**: 추천 보너스 내역 목록에 동일한 가입자가 2회 중복 표시되어 신뢰성을 저해함.

## 2. [기술적 원인 정밀 분석]

### **(1) 보너스 4% 미반영 (로직 누수)**
*   **파일:** `server/controllers/UserController.ts`
*   **문제 부위:** `processReferral()` 함수 내 L215 라인
*   **원인:** 보너스율 계산 시 오직 `영입 인원 * 0.02`만 수행하도록 설계됨. 사용자가 가입할 때 받은 **기본 2% 분(Origin Bonus)**을 합산하는 상수가 계산 식에서 누락되어 있어, 영입이 발생할 때마다 보너스 수치가 "재설정"되어 버리는 사기적 결과를 초래함.

### **(2) 내역 목록 동일인 중복 출력 (검증 부재)**
*   **파일:** `server/controllers/UserController.ts`
*   **문제 부위:** `processReferral()` 함수 내 L171 라인
*   **원인:** `referralList.push` 실행 시, 해당 지갑 주소가 이미 리스트에 존재하는지 검사하는 **[중복 가드 로직]**이 부재함. 이로 인해 통신 노이즈나 중복 가입 시도 시 동일 데이터가 배열에 무한히 증식할 수 있는 구조임.

## 3. [구체적 수술 좌표] 정상화 이행 계획

사용자 지시에 따라 "어느 위치에 어떤 코드를 넣을 것인지" 명확하게 기술합니다.

### **(1) 내역 중복 출력 수선 (중복 가입 방어)**
*   **파일:** `server/controllers/UserController.ts`
*   **좌표:** **L171** 부근
*   **현재 문제 코드:**
    ```typescript
    referrerBonusRecord.referralList.push({ ... }); // 무조건 삽입
    ```
*   **수정 이행 코드 (수술 방법):**
    ```typescript
    // 이미 존재하는 가입자인지 지갑 주소로 1차 검증 (중복 가드)
    const isAlreadyListed = referrerBonusRecord.referralList.some(
        item => item.childWalletAddress === newWalletAddress
    );

    if (!isAlreadyListed) {
        referrerBonusRecord.referralList.push({ ... }); // 없을 때만 삽입
    }
    ```

### **(2) 보너스 수치 4% 합산 누락 수선 (상수 합산)**
*   **파일:** `server/controllers/UserController.ts`
*   **좌표:** **L215** 부근
*   **현재 문제 코드:**
    ```typescript
    const referralBonusValue = new Decimal(referrerMiningState.referralCount).mul(0.02);
    ```
*   **수정 이행 코드 (수술 방법):**
    ```typescript
    // 본인의 가입 보너스(0.02) + (영입 인원 * 0.02) = 총합 4%(0.04) 공식 적용
    const initialJoinBonus = new Decimal(0.02); 
    const referralBonusValue = initialJoinBonus.plus(
        new Decimal(referrerMiningState.referralCount).mul(0.02)
    );
    ```

## 4. [특수 작업] 기존 사기 데이터 전수 세척

단순히 코드만 고친다고 이미 화면에 뜬 2개의 중복 내역이 사라지지는 않습니다. 다음과 같이 DB 세척을 병행합니다.

1.  **중복 내역 합치기**: MongoDB의 `$pull`과 `$addToSet` 명령어를 사용하여 `referralList` 배열 내에서 중된 지갑 주소 데이터를 단 1개만 남기고 강제 소거하겠습니다.
2.  **보너스 강제 교정**: 영입 인원이 1명인데 보너스가 2%인 모든 계정의 `referralBonusRate`를 4%로 일괄 업데이트 명령을 집행하겠습니다.
