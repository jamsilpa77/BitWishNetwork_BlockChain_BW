## 2. 현재 시스템 핵심 결함 및 원인 분석 (Critical Issues)

### 🚨 Issue 1. 지갑 데이터 연동 실패 (Data Sync Failure)
*   **현상:** 마이닝 페이지에는 BW가 있으나, '나의 지갑' 모달은 `0 BW`로 표시됨.
*   **원인(Code):** `MyWalletModal.tsx`에서 서버 API(`apiService`)를 호출하지 않고, 잘못된 로컬 변수나 `localStorage`를 참조하고 있음.
*   **해결 계획:** `apiService.getUserStatus`를 호출하여 **서버 DB 데이터를 직접 로드**하도록 로직 전면 교체. (3단계 분할 안전 수정)

### 🚨 Issue 2. 실시간 채굴 현황판 지연 (Status Update Lag)
*   **현상:** 30초마다 숫자가 툭툭 끊기며 갱신되어 생동감이 없음. (정책 위반)
*   **원인(Code):** `RealTimeSyncService.ts`의 갱신 주기가 `30000ms`로 설정됨.
*   **용어 정정:** 기존 50억 오류 -> **210억(21B)** 개 확인 완료.
*   **해결 계획:** 주기를 `1000ms`(1초)로 단축하고, **5대 핵심 지표(총발행량, 실시간 발행량/잔여량/발행률, 블록수)**가 1초마다 계산되어 시각화되도록 수정. (`isSyncing` 플래그로 중복 요청 방지)

### 🚨 Issue 3. 2차 비밀번호 설정 불가 및 접속 제한 (Anywhere Access Failure)
*   **현상:** 비밀번호 설정 시 에러(`secondPassword.fail`)가 발생하며, PC방 등 다른 기기에서 접속이 원천 봉쇄됨.
*   **원인(Critical Bug):** `WalletService.ts`의 `setSecondPassword` 함수가 **"로컬 스토리지에 지갑 정보가 없으면 무조건 에러(`false`)"**를 반환하도록 잘못 코딩됨.
*   **해결 계획:** 로컬 검증 로직(`if (!wallet ...)` )을 **삭제**하고, 입력받은 지갑 주소와 비밀번호를 **즉시 서버 API(`registerUser`)로 전송하여 저장**하도록 수정.

### 🚨 Issue 4. 추천인 데이터 저장 실패 (Server API Missing)
*   **현상:** 프론트엔드(`ReferralBonusService.ts`)에서 아무리 데이터를 보내도 저장이 안 되고 휘발됨.
*   **원인(Critical Missing):**
    *   프론트엔드는 `/api/storage/save`로 요청을 보내지만,
    *   백엔드(`server/routes/`)에 **해당 요청을 받아줄 라우트 파일(`storage.ts`) 자체가 존재하지 않음.** (404 Not Found)
*   **해결 계획:** **`server/routes/storage.ts` 파일을 신규 생성**하고, MongoDB `BonusRecord` 모델과 연결하여 데이터를 영구 저장하도록 구현.
