# [긴급] 일반 가입자 추천 보상(1BW) 오지급 원인 분석 보고서 (2026-03-01)
# Analysis Report: Referral Bonus Misallocation for General Users

**문서 번호:** BW-ERROR-REPORT-20260301-01
**작성 일자:** 2026년 3월 1일
**작성자:** BitWish AI Assistant

---

## 🏗️ 1. 장애 현황 및 이슈 개요
*   **현상:** 추천 코드 없이 가입한 일반 가입자(`BW29DFB93...9DAC`)에게 "추천 보상 보관함"으로 **1.00000000 BW**가 자동 지급됨.
*   **영향:** 추천 보너스 정책(추천 가입자에게만 지급) 위배 및 전체 발행량 데이터(Total Issued) 왜곡 발생.
*   **발견 사항:** 지갑 생성 시 서버 DB에 1명이 추가됨과 동시에 총 발행량도 정확히 1 BW 상승(6 -> 7)함.

## 🏗️ 2. 코드 정밀 분석 결과 (Root Cause)

파일 분석 결과, **백엔드 서버의 사용자 등록 로직 내부에 하드코딩된 기초 파라미터 오류**를 발견했습니다.

### **[핵심 문제 지점]**
*   **파일 위치:** `server/controllers/UserController.ts`
*   **코드 라인 (Line 71):**
```typescript
// server/controllers/UserController.ts

// 4. 보너스 레코드 초기화
const bonusRecord = new BonusRecord({
    walletAddress,
    referralBonusStorage: '0.00000000000000000000000000000000000000000000000000',
    // ⚠️ 아래 라인이 모든 신규 유저에게 1 BW를 무조건 지급하도록 설정되어 있음
    referralRewardStorage: '1.00000000000000000000000000000000000000000000000000', 
    referralList: [],
    attendanceHistory: []
});
```

### **[분석 상세 요약]**
1.  **무차별적 초기 보상 설정:** 위 코드의 71번 라인에서 신규 가입자의 보너스 데이터베이스를 생성할 때, 추천 코드 유무와 상관없이 **기본 발행량(referralRewardStorage)을 '1.0'으로 강제 설정**하고 있습니다.
2.  **정책 위배 로직:** 원래는 '0'으로 시작하고, 가입 시 제공된 `referrerCode`가 유효할 경우에만 별도의 추천 처리 로직(`processReferral`)을 통해 1 BW를 더해줘야 하지만, 현재는 **기본값이 1로 잘못 설정**되어 있습니다.
3.  **DB-홈페이지 동기화:** 홈페이지 현황판은 이 서버 DB의 `referralRewardStorage` 총합을 실시간으로 반영하므로, 일반 유저가 가입하자마자 즉시 발행량이 +1 상승하는 현상이 발생한 것입니다.

## 🏗️ 3. 장애 영향 분석
*   **데이터 오염:** 현재 가입된 모든 유저(6명)가 각 1 BW씩 기본적으로 가지고 시작하게 되어, 실제 의도한 발행량보다 초과 발행된 상태임.
*   **정합성:** 프론트엔드(`CreateWalletModal.tsx`)에서는 코드가 없을 때 `""`(빈 값)을 잘 넘겨주고 있으나, 백엔드에서 이를 무시하고 1점을 퍼주는 구조임.

## 🏗️ 4. 향후 조치 계획 (안)
*   **Step 1:** `server/controllers/UserController.ts`의 초기값을 `0.0`으로 수정. (완료)
*   **Step 2:** 추천 코드가 있을 때만 1 BW를 지급하는 조건문 검증 강화. (완료)
*   **Step 3:** **[물리적 정화 실행]** 기존 오지급된 유저는 강제 차감하고, 일반 유저 보관함은 `0`으로 즉시 초기화하는 물리적 행위(Script Run) 집행 및 검증. (진행 중)
