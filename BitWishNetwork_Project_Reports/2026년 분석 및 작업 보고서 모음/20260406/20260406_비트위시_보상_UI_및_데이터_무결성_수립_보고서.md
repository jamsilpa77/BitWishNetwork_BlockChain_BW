# 📋 2026-04-06 비트위시 보상 UI 시인성 및 데이터 무결성 수립 보고서

## [1] 채굴 보상 내역 UI 정밀 개선안 (Job 1)

숫자가 늘어나며 레이아웃이 뒤틀리는 현상을 원천 차단하고 보상의 정밀도를 확보하기 위한 UI 정례 지지서입니다.

### 1-1. 4자리/8자리 표기 원칙
- **기본 노출 (Table View):** 모든 수치는 **소수점 4자리**로 고정 표기합니다. (검은색 폰트 #111827 강제 적용)
- **정밀 노출 (Hover/Tooltip):** 마우스를 가져다 대었을 때(Hover), 시스템의 근간인 **소수점 8자리**를 말풍선(Tooltip) 형태로 노출하여 숫자의 '움직임' 없이 정밀도를 확인하게 합니다.

### 1-2. 텍스트 밀림 현상 방지 해결책
- **문제 진단:** 숫자의 너비가 유동적으로 변하며 테이블 칸(`<td>`)을 밀어내는 현상.
- **수정 로직:** 
    - 테이블 셀에 **`width` 고정값** 또는 **`table-layout: fixed`** 속성을 부여하여 틀을 유지.
    - 8자리 숫자는 부유 레이어(말풍선)로만 처리하여 물리적 공간을 차지하지 않도록 설계.

## [2] 추천 보너스 보관함 데이터 무결성 수복 (리셋 방지)

페이지 이동 시 '2% 추천 보너스 보관함' 수치가 0으로 초기화되는 결함에 대한 최종 분석 및 해결 구조입니다.

### 2-1. 결함의 핵심: 대소문자 미매칭 (Case-Insensitivity)
- **분석 결과:** 서버 검색(`MiningController.ts`) 시 대소문자를 구분하여 검색하므로, 주소가 소문자로 전달될 경우 '0' 값을 뱉고 화면이 이를 덮어쓰며 리셋이 발생함.

### 2-2. 해결 구조 (Blueprint)
1. **서버:** 모든 지갑 주소 검색 시 대소문자 무시 정규표현식을 적용하여 '진짜 금고' 한 곳만 바라보도록 고정.
2. **클라이언트:** 30초 동기화 시 서버에서 비정상적인 0이 올 경우 기존 수치를 보관하는 방어 로직 설계.
