# 📋 2026-04-06 비트위시 클로즈 베타(CBT) 대비 원점 회귀(Init) 시스템 정밀 명세서

## 1. 개요 및 목적
*   **CBT 무결성 확보:** 테스트 및 개발 단계에서 생성된 16개 지갑과 31 BW 부당 지급 데이터를 전면 소멸시키고, 정식 Closed Beta를 위해 시스템을 **'태초의 0세대 상태'**로 원점 회복함.
*   **관리자 직접 제어:** 모든 초기화 기능(지갑, 보상, 이력)을 관리자 페이지의 버튼 클릭으로 즉시 집행 가능하도록 구현함.

---

## 2. 관리자 페이지 UI 배치 및 기능 명세 (Mining Test Page)

### [A] 상단 통합 제어 영역
*   **위치:** "마이닝 테스트 관리" 텍스트 제목의 우측 끝 가로 정렬.
*   **버튼 1: [발급 지갑 전체 초기화]** (왼쪽 배치)
    *   **로직:** `Clean Slate` (지갑 주소, 계정, 블록, 트랜잭션 컬렉션 전면 비움).
    *   **결과:** 배포 초기 상태로 네트워크를 되돌려 지갑이 하나도 없던 시점으로 회귀.
*   **버튼 2: [보상 BW 전체 초기화]** (오른쪽 배치)
    *   **로직:** `Reward Wipe` (전 서버 유저의 가상 보상 BW 잔액을 0.00...으로 강제 말소).
    *   **결과:** 모든 유저의 추천 보상 창고를 비우고 가입 보상 체계를 정화함.

### [B] 유저별 개별 제어 영역
*   **위치:** 특정 주소 검색 결과인 "마이닝 정보" 영역 최하단, 기존 "마이닝 데이터 초기화" 버튼 우측 옆.
*   **버튼 3: [보상 BW 초기화]**
    *   **로직:** `Individual Reward Reset` (검색된 지갑 주소 1인의 추천 보상 BW 이력만 0으로 삭제).
    *   **결과:** 특정 유저의 오류 보상 데이터를 핀포인트로 정상화함.

### [C] 추천 보상 독립 분리 및 가시화 (개별 초기화 검증용)
*   **위치:** 유저별 "마이닝 정보" 검색 결과 영역 내.
*   **기능:**
    *   검색 결과 상에 `누적 보상 (채굴량)`과 `추천 보상 (보너스)`을 분리하여 각각 한 칸씩 표기.
    *   [보상 BW 초기화] 집행 시, `추천 보상` 칸만 즉시 0 BW가 되는 것을 관리자가 눈으로 확인할 수 있도록 설계.

## 3. 집행 기술 상세 (CBT 전용 원점 회복 엔진)

1.  **가상 보상(Points) 초기화:** 
    *   `bitwish_mining` DB의 `bonusrecords`, `miningstates` 수치를 0세대(0.0)로 강제 업데이트.
    *   물리 파일 DB인 `referrals.json`의 해당 유저 데이터를 즉시 소멸 처리.
2.  **블록체인(Wallets) 초기화:** 
    *   `bitwish_network` DB의 핵심 컬렉션(`blocks`, `transactions`, `wallets`, `accounts`)을 즉시 비움(Truncate).
    *   서버 재시작 없이 0번 제네시스 블록부터 다시 시작되도록 코어 엔진 연동.
