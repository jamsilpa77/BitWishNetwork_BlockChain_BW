## 🏗️ 1. 현 시스템의 치명적 장애 원인 (Root Cause Analysis)

사용자님이 제시하신 이미지와 실제 DB 데이터를 정밀 대조한 결과, 제가 보고했던 "개 헛소리"의 실체를 밝혀냈습니다.

### **가. 유저 1 (...E9DAC): 명수 중복 오류 (2명/4% 현상)**
- **기술적 팩트**: 추천 리스트(`referralList`)에 가입자 유저 2(`...7A626`)의 주소가 **중복으로 2회** 기록되어 있음.
- **원인**: 이전의 오판된 '가짜 데이터 생성 회로'가 동작할 때, 진짜 지갑 주소를 중복으로 삽입하거나 중복 체크 없이 리스트를 채웠기 때문임.
- **결과**: 엔진은 리스트 개수(2개)를 정직하게 세어 4%라고 내보냈으나, 데이터 자체가 오염되어 1명이 아닌 2명으로 표기된 것임.

### **나. 유저 2 (...7A626): 가입자 보너스 누락 (0% 현상)**
- **기술적 팩트**: 마이닝 페이지 UI(`MiningStatusModal.tsx`)의 연산 로직 결함.
- **원인**: UI가 서버 DB에 저장된 보너스율(2.0%)을 직접 읽지 않고, 화면에 표시된 명수(0명)를 가져와 `0 * 2 = 0%`라고 **중복 재연산**하여 덮어씌움.
- **결과**: 가입자가 받아야 할 영구 보너스 2%가 명수가 0명이라는 이유로 UI에 의해 차단됨.

## 🏗️ 2. 사용자 지시 기반 정책 구현 기준 (Policy Standard)

1. **1 BW 보상**: 현재 정상 작동 중인 1 BW 지급 회로는 **절대 수정하지 않음.**
2. **보유자 혜택**: 가입자 발생 즉시 **실제 가입인원(Unique) 반영** 및 **명당 2%** 누적 연산.
3. **가입자 혜택**: 추천 코출 가입 즉시 **2% 영구 보너스** 수령. (단, 본인 추천 명수는 0명 유지)
4. **시스템 중심**: 임시방편이 아닌, **백엔드 엔진 개조**를 통해 앞으로의 모든 가입자가 이 혜택을 즉각 누리도록 함.

## 🏗️ 3. 완결 정규화 집행 계획 (Execution Plan)

### **🚀 Step 1: 데이터 정합성 100% 회복 (Cleanup & Normalization)**
- **작업**: 전체 유저의 추천 리스트를 전수 조사하여 **중복된 지갑 주소** 및 유령 주소를 완전히 삭제함.
- **목표**: 유저 1의 명수를 실제 진실인 **1명(2%)**으로 되돌림.

### **🚀 Step 2: 백엔드 정책 엔진 완비 (Systemization)**
- **작업**: 가입자 본인의 **2% 수익권 엔진**을 백엔드에 영구 장착.
- **목표**: 어떤 유저든 가입 즉시 DB에 2%가 시스템적으로 귀속되도록 함.

### **🚀 Step 3: UI 표시 엔진 교정 (UI Integrity)**
- **작업**: UI가 화면의 명수로 보너스를 재계산하는 오류 로직을 삭제하고, **서버 DB의 보너스 수치를 그대로 신뢰**하여 표출하도록 수정.
- **목표**: 가입자 본인의 2%가 명수와 상관없이 화면에 즉시 나타나게 함.
