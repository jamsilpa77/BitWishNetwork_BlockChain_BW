# [구현 계획] 나의 지갑 '추천 보너스 내역' 시스템 정밀 구현 계획서

## 1. [작업 개요] 나의 지갑 내 '추천 보너스 내역' 기능 구현

본인은 사용자님이 지시하신 [추천 보너스 내역]의 핵심 명세를 무시하고 엉뚱한 결론을 도출했습니다. 이에 지시하신 사항을 100% 반영하여, 시스템이 나아가야 할 단 하나의 방향을 정의합니다.

### (1) 구현 메뉴 및 위치
*   **메뉴 명칭:** 추천 보너스 내역
*   **위치:** '나의 지갑' 모달 내 [추천 보너스 내역] 탭
*   **원칙:** 가입자 발생 즉시 코드 보유자의 지갑 내역에만 자동 등록.

---

## 2. [데이터 명세] 내역서 표기 항목 (5대 필수 요소)

사용자님이 불러주신 명세 그대로, 테이블 컬럼과 색상을 다음과 같이 확정합니다.

| 표기 항목 | 상세 내용 | UI 색상 구분 |
| :--- | :--- | :--- |
| **1. 가입자** | 가입자의 지갑 주소 | 기본 텍스트 |
| **2. 가입날짜** | 가입 완료 시점 (년.월.일.시.분.초) | 기본 텍스트 |
| **3. 지급 보상** | 나 1BW / 가입자 1BW 지급 상태 | **지급(파란색)** / **미지급(빨간색)** |
| **4. 지급 보너스** | 나 2% / 가입자 2% 지급 상태 | **지급(파란색)** / **미지급(빨간색)** |
| **5. KYC 승인 현황** | 가입자의 실시간 KYC 승인 상태 | **승인(파란색)** / **미승인(빨간색)** |

## 3. [안전 작업 계획] 기존 실행 플랜 (히스토리 보존)

기존 P2P 지갑 기능을 절대 건드리지 않고, 독립적인 내역 시스템만 구축합니다.

### **[Step 1] 내역 기록용 독립 데이터 구조 생성**
*   **작업:** 사용자님이 지시한 5가지 정보를 저장할 수 있도록 유저 개개인의 저장소(`BonusRecord`)에 필드 생성.
*   **원칙:** 다른 컴포넌트나 전역 변수와 절대 공유하지 않음.
*   **결과:** (2026-03-05 집행 완료)

### **[Step 2] 가입 즉시 내역 등록 엔진 적용**
*   **작업:** 추천 코드로 가입이 완료되는 즉시, 위 5가지 정보를 파란색/빨간색 상태값과 함께 DB에 박아넣는 로직 구현.
*   **소급 적용:** 현재 0%로 누락된 가입자들도 이 내역 시스템에 맞춰 과거 로그를 생성하여 권리 복구.
*   **결과:** (2026-03-05 엔진 수정 및 데이터 1차 정화 완료)

### **[Step 3] 지갑 UI 내역판 완성**
*   **작업:** 지갑의 "Coming Soon"을 제거하고, DB에서 불러온 내역을 지시하신 색상(파/빨)대로 출력.
*   **결과:** (2026-03-05 테이블 UI 구축 및 데이터 매핑 완료)

## 4. [긴급 정밀 진단] 발생 결함 분석 및 보정 계획 (3월 5일 추가)

작업 완료 후 발견된 비정상 현상에 대해 원인을 분석하고 후속 조치 계획을 수립합니다.

### **(1) 지갑 내역 공백 및 UI 가시성(명암) 오류**
*   **진단:** DB는 정화되었으나 API 통신 규격 불일치로 데이터가 지갑에 도달하지 못함. 또한 헤더 색상 대비 미흡으로 글자가 보이지 않음.
*   **수정 계획 (Step 4):** 백엔드 API 수선 및 헤더 텍스트 색상(`#374151`) 하드코딩 적용.

### **(2) 추천 보너스 수치(48%) 및 보상률 폭증 오류**
*   **진단:** 정규화 명령어의 중첩 연산 버그로 인해 기본 6%여야 할 보너스가 48%로 폭증함(8배 오차).
*   **수정 계획 (Step 5):** 기존 수치를 전면 무시하고 `가입자 수 * 0.02`라는 **[절대 공식]**을 사용하여 전 유저 데이터 강제 재교정(Reset).

## 6. [추가 기능 분석 및 계획] 추천 내역 요약(Summary) 실시간 표기 (3월 5일 추가)

사용자 지시에 따라, 가입자 목록 상단에 전체 합산 수치를 실시간으로 노출하여 가독성을 극대화합니다.

### **(1) 기능 분석**
*   **배치 위치:** '추천 보너스 내역' 탭 내 [- 가입자 목록 -] 제목의 우측 끝 라인.
*   **표기 항목:** 
    1.  **총 보상:** 가입 보상(1BW)의 전체 합산 수치 (예: `총 보상: 3.0000 BW`)
    2.  **총 보너스:** 채굴 보너스(2%)의 전체 합산 퍼센트 (예: `총 보너스: 6.0000 %`)
*   **연동 원리:** `referralList`의 데이터 개수와 서버에서 전달받은 `referralRewardStorage`, `referralBonusRate` 값을 실시간 바인딩하여 가입자 발생 즉시 수치가 변동되도록 구현.

### **(2) 작업 계획 (Step 6)**
*   **UI 작업:** `MyWalletModal.tsx` 내 가입자 목록 헤더 영역에 요약 정보(Summary) 섹션 추가 및 우측 정렬 레이아웃 적용.
*   **데이터 작업:** 지갑 데이터(`walletData`) 상태값을 활용하여 합산 수치가 누락 없이 출력되도록 로직 안착.
