## 1. 아키텍처 V2 설계도 vs 실제 코드 현황 비교

관리자님께서 승인하신 **V2 아키텍처 (`BitWishNetwork_System_Architecture.md`)**는 완벽한 모듈화를 지향하고 있습니다. 그러나 현재 파일 시스템을 전수 조사한 결과, **설계된 폴더 구조가 실제로는 존재하지 않음**을 확인했습니다.

### **1.1. 설계도 (Ideal: V2 Architecture)**
문서의 **4.1 디렉토리 구조** 섹션에 따르면, 백엔드는 다음과 같이 역할별로 방이 나누어져 있어야 합니다.

*   📂 **`server/routes/`**: API 주소와 요청을 받아주는 접수처 (예: `mining.ts`, `admin.ts`)
*   📂 **`server/controllers/`**: 실제 계산과 일처리를 담당하는 작업장 (예: `MiningController.ts`)
*   📂 **`server/models/`**: 데이터를 담는 그릇(Schema)을 정의하는 창고 (예: `User.ts`, `MiningState.ts`)

### **1.2. 실제 현황 (Reality: Current File System)**
`ls` 명령어로 `deploy_backend/src` 내부를 확인한 결과는 처참합니다.

*   📂 **`src/server/`**: 이 폴더 안에 **`storage.ts` (파일 저장소)** 파일 딱 하나만 덩그러니 있습니다.
*   📄 **`src/server.ts`**: 모든 로직이 이 파일 하나에 뒤섞여 있습니다. (접수처, 작업장 구분이 없음)
*   ❌ **누락된 것:** `routes`, `controllers`, `models` 폴더 자체가 **없습니다.**

### **1.3. 진단 결과 (Diagnosis)**
**"설계도는 아파트인데, 실제로 지어진 건 원룸입니다."**
방이 하나(`server.ts`)밖에 없어서 모든 짐(코드)을 거기에 다 쌓아두고 있습니다. 그래서 관리자 기능을 추가하려 해도 넣을 공간이 없고, 추천 보너스 로직을 짜려 해도 엉킬 수밖에 없는 구조입니다.

## 2. 복구 실행 계획: 원룸을 아파트로 리모델링

V2 설계도대로 시스템을 복구하기 위해, 물리적으로 **방(Folder)**을 만드는 작업이 선행되어야 합니다.

### **2.1 [Step 1] 구조 정상화 (폴더 생성)**
*   **작업 내용:** 누락된 디렉토리를 `mkdir` 명령어로 생성합니다.
    *   `deploy_backend/src/server/routes`
    *   `deploy_backend/src/server/controllers`
    *   `deploy_backend/src/server/models`
*   **이유:** 코드를 옮겨 담을 그릇(폴더)이 있어야 리팩토링(이사)이 가능하기 때문입니다.

### **2.2 [Step 2] 코드 이식 (파일 생성 및 분리)**
*   **작업 내용:** `server.ts`에 뭉쳐있는 코드를 찢어서 각 방으로 보냅니다.
    *   **Mining API 관련 코드** → `routes/mining.ts` 파일 생성 후 이동.
    *   **보너스 계산 로직** → `controllers/MiningController.ts` 파일 생성 후 이동.
    *   **DB 스키마 정의** → `models/MiningState.ts` 파일 생성 후 이동.

## 3. 관리자님께 드리는 보고
이 작업은 제가 임의로 폴더를 늘리는 것이 아닙니다.
**"원래 있어야 했지만 누락된 폴더"**를 V2 설계도에 맞춰 복원하는 **필수 기초 공사**입니다.

이 보고서 내용을 확인해 주시고, **"구조 정상화(폴더 생성)"** 작업을 승인해 주신다면 즉시 실행에 옮기겠습니다.
