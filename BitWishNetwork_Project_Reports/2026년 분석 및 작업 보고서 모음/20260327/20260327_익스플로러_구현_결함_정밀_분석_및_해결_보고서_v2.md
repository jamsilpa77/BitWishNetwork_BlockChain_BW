## 1. 현재 결함 정밀 분석 (Why the error persists?)

### ⚠️ [치명적 결함] 반복되는 런타임 에러(`TypeError: blockHeight`)
- **원인 분석:** 
    1. 코드 상에 `?.` (선택적 체이닝)을 적용했음에도 에러가 지속되는 것은, **데이터가 들어오지 않는 찰나의 순간(undefined/null)** 또는 **DB에서 예외적인 데이터(객체가 아닌 값)**가 넘어올 때 리액트가 렌더링을 시도하기 때문입니다.
    2. 특히 `blocks.map()` 내부에서 배열의 요소가 `blocks[0]` 등과 형식이 다르거나, 예상치 못한 `null` 값이 섞여 있을 가능성이 매우 큽니다.
    3. **핵심 원인:** 기존의 부분적인 수정(코드 한두 줄 변경)으로는 빌드 캐시나 데이터 예외 상황을 완벽히 통제하지 못하고 있습니다.

## 2. 익스플로러 단계별 개선 및 해결 계획 (Priority & Code Logic)

### **[1순위] 런타임 에러 원천 봉쇄 (Atomic Defense)**
- **목표:** 어떤 상황에서도 페이지가 멈추지 않고 즉시 열리도록 구현.
- **수정 위치:** `src/components/Explorer/ExplorerPage.tsx`
- **코드 로직:** 
    - **배열 무결성 검증:** `fetchBlocks` 후 `setBlocks` 시점에 `Array.isArray()`를 통한 타입 체크 강제.
    - **렌더링 필터링:** `blocks.filter(b => !!b && !!b.header).map(...)` 로직을 사용하여 정상이 아닌 데이터는 렌더링 시도조차 하지 않음.
    - **초강력 하드 가드:** 모든 객체 접근에 `block?.header?.blockHeight || 0` 등 3중 선택적 체이닝 적용.

### **[2순위] 파이 네트워크 벤치마킹 UI 고도화 (Premium Aesthetics)**
- **목표:** 파이 네트워크 익스플로러 수준의 극도로 깔끔하고 가시성 높은 디자인 구현.
- **수정 위치:** `src/components/Explorer/ExplorerPage.css`
- **코드 로직:** 
    - **카드 시스템 도입:** 통계 지표(Height, Blocks 등)를 상단에 배치하고, 가독성 높은 산세리프 폰트(Inter)와 네온 효과 적용.
    - **테이블 간소화:** 불필요한 테두리를 제거하고 행(Row) 사이의 여백과 호버 효과를 강화하여 프리미엄 느낌 강조.

### **[3순위] 전역 테마 및 4개국 언어 완벽 연동**
- **목표:** 홈페이지 설정(다크모드, 한/영/일/중)이 익스플로러에 즉각 반영되도록 구현.
- **수정 위치:** `ExplorerPage.tsx` & `LanguageManager.ts`
- **코드 로직:** 
    - **테마 감시:** `document.body`의 `class` 변화를 감지하는 `MutationObserver`를 장착하여, 페이지 새로고침 없이도 테마가 즉시 전환됨.
    - **다국어 매핑:** `LanguageManager.getTranslation('explorer.xxx')`를 통해 모든 하드코딩된 영문 텍스트를 제거하고 4개국어 자동 변환 구조로 개편.

### **[4순위] 실시간 검색 엔진 적용 (Search Engine)**
- **목표:** 주소/TXID 입력 시 데이터를 즉시 추적하는 검색 보드 활성화.
- **수정 위치:** `ExplorerPage.tsx` (handleSearch 함수)
- **코드 로직:** 
    - 정규식을 통해 `BW` 주소와 `64자리 16진수` 해시를 판별하고, 해당 데이터로 스크롤하거나 상세 뷰로 전환하는 로직 주입.

## 3. [긴급 추가] 3순위 테마 및 언어 동기화 결함 보강 대책

### ⚠️ [현상] 홈페이지 설정 변경이 익스플로러에 즉각 반영되지 않음
- **원인 1 (테마):** `ExplorerPage` 진입 시 `localStorage` 값은 존재하지만, 이를 실제 `document.body`의 클래스에 반영하는 초기 주입 과정이 누락되었습니다.
- **원인 2 (언어):** `MutationObserver`는 DOM 변화만 감지할 뿐, `localStorage`의 언어 키(`bw_lang`) 변화를 직접 추적하지 못합니다.

### 🛠️ [보강 코드 로직]
1.  **Mount 시점 강제 동기화 (Initial Force Sync):**
    - `useEffect`의 시작점에서 `localStorage.getItem('bw_theme')` 값을 읽어와 `document.body.classList.toggle('dark-mode', isDark)`를 강제로 실행합니다.
    - `languageManager.setLanguage(localStorage.getItem('bw_lang'))`을 호출하여 인스턴스 언어를 즉각 초기화합니다.
2.  **전역 설정 변화 실시간 트래킹 (Global Storage Listener):**
    - `window.addEventListener('storage', ...)` 리스너를 추가하여, 홈페이지 탭이나 다른 경로에서 설정이 바뀌는 순간 익스플로러의 상태(`darkMode`, `currentLang`)를 즉시 업데이트하도록 구현합니다.
3.  **컴포넌트 리렌더링 유도:**
    - 언어 설정 변경 시 `setCurrentLang` 상태를 갱신하여 시스템 전체 텍스트가 번역된 값으로 즉각 다시 그려지도록 설계합니다.

## 4. [최종 보강] 홈페이지 저장 기능 복구 및 전역 규격 단일화

### ⚠️ [결정적 결함 분석] 홈페이지의 "데이터 저장 주체" 기능 누출
- **홈페이지(`HomePage.tsx`):** 언어 선택 시 상태값으로만 들고 있을 뿐, 브라우저 스토리지(`localStorage`)에 저장(`setItem`)하는 로직이 원천적으로 누락되어 전역 연동이 불가능한 상태입니다.
- **규격 불일치 (Key Mismatch):** 홈페이지는 `bw-theme`(하이픈)을 쓰는 반면, 익스플로러는 `bw_theme`(언더바)를 읽으려 시도하여 초기 테마 연동이 실패합니다.

### 🛠️ [최종 해결 가이드]
1.  **홈페이지(`HomePage.tsx`) 수리:** 
    - `handleLanguageChange` 함수 내에 `localStorage.setItem('bw_lang', language)` 코드를 강제 주입하여 데이터의 원천을 복원합니다.
2.  **익스플로러(`ExplorerPage.tsx`) 규격 단일화:** 
    - 모든 저장 키를 홈페이지의 정격 규격인 `bw-theme` 및 `bw_lang`으로 완전히 일치시켜 0.1초 만에 설정이 반영되도록 조치합니다.
3.  **최종 결과:** 
    - 홈페이지에서 유저가 선택한 영어/일어/중어 및 다크모드 설정이 브라우저 데이터를 거쳐 익스플로러에 즉각적(Real-time)으로 전달됩니다.
