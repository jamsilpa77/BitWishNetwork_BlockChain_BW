## 2. 주요 중점 분석 항목 (Deep Dive Checkpoints)

### A. 추천 보너스 시스템 (피라미드 및 저장소)
- **무한 종속성 (Infinite Dependency):** A가 B를 초대하고, B가 C를 초대했을 때, C의 채굴 활동이 B뿐만 아니라 A에게까지 영향을 미치는 구조적 로직이 `server.js`와 데이터 구조에 존재하는가?
- **보관함 완전 분리:**
    - `rewardStorage` (1BW 가입 보상)
    - `bonusStorage` (2% 채굴 보너스 누적분)
    - 이 두 데이터가 서로 간섭 없이 독립적으로 계산되고 저장되며, UI에 명확히 구분되어 표시되는가?
- **즉시 전파 (Real-time Propagation):** 하위 라인에서 이벤트(가입, 채굴) 발생 시 상위 라인으로 `Recursive`(재귀적) 업데이트가 즉시 일어나는가?

### B. 독립 저장소 엔진 (BISE)
- **파일 시스템 구조:** 통합 `user_data.json`이나 메모리 배열 의존성을 탈피하고, `database/users/BW지갑주소.json` 형태의 **개별 독립 파일 시스템**이 실제로 구현되어 있는가?
- **데이터 무결성:** 파일 생성 및 갱신 시 데이터 손실을 방지하는 안전장치(Atomic Write 등)가 있는가?

### C. 프론트엔드 정합성
- **서버 절대 복종:** 클라이언트(`MiningStatusModal.tsx` 등)가 로컬 스토리지 값을 맹신하거나 임의로 `setInterval`을 돌려 서버 데이터를 덮어쓰는 "하극상" 코드가 제거되었는가?
- **UI 디테일:** 소수점 4자리 표시 및 호버 시 8자리 확장 기능, 로딩 처리 등이 지시서대로 구현되었는가?

### D. 인증 및 접근성 (최신 지시사항)
- **접속 제한 해제:** "내 기기가 아니어도 내 지갑+비번이면 접속 가능"하도록 로컬 차단 로직이 해제되었는가? (계획서 30번 반영 여부)
