## 2. 통합 검증 내역 (Cross Check)

### A. [Step 1] 기능 유지 확인
1.  **BISE 엔진 유지:** `getUserFilePath`, `loadUserData`, `saveUserData` 함수가 그대로 존재하며, 파일 기반 입출력을 수행합니다. (Line 73~110)
2.  **피라미드 로직 유지:** `/api/referral/join` 엔드포인트 내 `while (currentAncestorId && depth < MAX_DEPTH)` 루프가 살아있으며, 상위 추천인에게 2% 보너스를 전파하는 로직(Line 356~366)이 건재합니다.

### B. [Step 2] 신규 기능 구현 확인
1.  **틱 로직 고도화 (`/api/mining/tick`):**
    -   기존: 틱 요청 시 아무것도 안 하거나 통계만 올림.
    -   **현재 (Line 386):**
        -   클라이언트가 1초마다 신호를 보내면,
        -   서버는 해당 유저 파일(`userId`)을 엽니다.
        -   유저의 `totalBonusRate`(피라미드로 쌓인 보너스율)를 확인합니다.
        -   **공식:** `(0.25 / 3600) * (Rate / 100)` 만큼 계산하여 `bonusStorage`에만 핀셋으로 집어넣듯 정확히 더합니다.
    -   결과: `rewardStorage`(가입 선물 1BW)와 섞이지 않고, 순수하게 채굴 보너스만 `bonusStorage`에 쌓이게 됩니다.
