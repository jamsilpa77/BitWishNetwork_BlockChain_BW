## 🏗️ 1. 구현 목표
*   마이닝 페이지의 1초 주기 채굴량 데이터를 지갑(Wallet)과 홈페이지(HomePage)에 실시간으로 접목.
*   50단위 부동소수점 정밀도 유지 및 UI 8자리 표기 일치.

## 🏗️ 2. 상세 작업 계획

### **Task A. MyWalletModal 실시간 연동 (최종 개선안)**
*   **분석 (실패 원인):** 기존에는 채굴기 모달이 닫히면 로컬 타이머가 멈춰 방송이 중단되었음. 또한 제멋대로 8자리를 강제하여 UI 가독성을 해침.
*   **해결책 1 [서비스 주도 티커]:** 1초 단위의 채굴량 증가 로직을 UI 모달이 아닌 `RealTimeSyncService`(싱글톤) 내부로 이전. 채굴 시작 시 서비스에 `baseRate`를 전달하면, 모달이 닫혀도 서비스가 자체 티커로 초당 방송을 유지함.
*   **해결책 2 [UI 표준 복구]:** 사용자 지시 사항인 **"기본 4자리 표시 / 호버 시 8자리 툴팁"** 규칙을 박제함.
*   **해결책 3 [정밀도 연산]:** 모든 초당 증가분 계산은 `Decimal.js`를 사용하여 50단위 정밀도를 유지하며, UI 출력시에만 위 규칙에 따라 포맷팅함.

### **Task B. HomePage 실시간 발행 현황 연동**
*   **로직:** 사용자의 실시간 채굴량 증가분(Delta)을 홈페이지 전역 통계(`globalStats`)에 즉시 합산/차감.
    *   실시간 현재 BW 발행량: `+ Delta`
    *   실시간 BW 잔여 발행량: `- Delta`
    *   실시간 BW 발행률: 재계산 표기 (1,000,000 단위 정밀도 유지)
*   **동작:** 초당 50단위 부동소수점 계산기 엔진 사용.
