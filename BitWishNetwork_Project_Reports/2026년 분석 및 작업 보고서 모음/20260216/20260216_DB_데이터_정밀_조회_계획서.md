# [계획] DB 데이터 정밀 조회 및 검증 계획서

**작성일:** 2026년 2월 16일
**작성자:** BitWish AI Assistant
**목표:** 관리자 리셋 후에도 사라지지 않는 "6 BW" 등의 보상 데이터가 DB 내 어떤 필드에 저장되어 있는지 **물리적 원본 데이터(Raw Data)**를 조회하여 규명함.

---

## 1. 조회 대상 및 방법
*   **대상:** MongoDB `bitwish_mining` 데이터베이스 내 `BonusRecord` 컬렉션.
*   **방법:** `server/routes/admin.ts`에 **임시 조회 API**를 추가하여 실행.
*   **핵심 전략:** Mongoose Schema(`BonusRecord.ts`)를 통하지 않고, **MongoDB Native Driver**(`mongoose.connection.db.collection`)를 직접 사용하여 **스키마에 정의되지 않은 "유령 필드(Hidden Fields)"까지 전부 조회**한다.

---

## 2. 구현할 조회 코드 (Source Code)

`server/routes/admin.ts` 파일에 아래 코드를 추가한다.

```typescript
// [임시] DB 원본 데이터 정밀 검증용 API
// 사용법: GET /api/admin/inspect/raw/:walletAddress
router.get('/inspect/raw/:walletAddress', async (req, res) => {
    try {
        const { walletAddress } = req.params;
        
        // 1. Mongoose Schema 검증 없이 날것 그대로 가져오기
        // (스키마에 없는 'bonusStorage' 같은 옛날 필드도 다 보임)
        const rawData = await mongoose.connection.db?.collection('bonusrecords').findOne({ walletAddress });

        if (!rawData) {
            return res.json({ success: false, message: '데이터 없음 (No Record Found)' });
        }

        // 2. 전체 필드 목록 및 값 출력
        return res.json({
            success: true,
            explain: "DB에 저장된 물리적 원본 데이터입니다.",
            data: rawData // JSON 전체 리턴
        });

    } catch (error) {
        return res.status(500).json({ error: error.message });
    }
});
```

## 3. 실행 및 확인 절차
1.  위 코드를 `admin.ts`에 삽입한다.
2.  서버가 재시작되면 웹 브라우저 주소창에 다음을 입력한다.
    *   `http://localhost:5001/api/admin/inspect/raw/BWD6CCB861E43DA7B213B9871CEC8C49E0F17577E5`
3.  **결과 확인:**
    *   화면에 표시된 JSON 데이터에서 `referralRewardStorage`(공식) 외에 **`bonusStorage`** 또는 **`recommend...`** 같은 **불필요한 필드**가 있는지 확인한다.
    *   그 필드에 **"6"**이라는 숫자가 들어있는지 확인한다.

## 4. 후속 조치 (데이터 확인 후)
*   만약 유령 필드가 발견되면, 리셋 함수(`reset`)에 해당 필드를 삭제(`$unset`)하는 코드를 추가하여 해결한다.
