## 1. 개요 및 목표 (Mission)
관리자님의 **"단순한 표시판이 아닌, 신뢰를 증명하는 지표여야 한다"**는 철학에 따라, 홈페이지 메인에 위치한 **[실시간 BW 채굴 상태 현황]** 6개 패널을 가짜(Mock) 데이터가 아닌 **실제 시스템 데이터와 100% 동기화**된 살아있는 유기체로 구현한다.

### **핵심 원칙 (The 3 Realities)**
1.  **Real Action:** 마이닝 시작 버튼을 누르면 즉시 반응해야 한다.
2.  **Real Data:** DB에 저장된 실제 지갑 수와 채굴량을 1:1로 반영해야 한다.
3.  **Real Time:** 1초의 지체 없이 시스템의 심장 박동(Heartbeat)을 보여줘야 한다.

---

## 2. 시스템 정밀 분석 및 데이터 매핑 (Deep Analysis)

각 UI 요소가 백엔드의 **어느 핏줄(Data Source)**과 연결되어야 하는지 정밀 분석했습니다.

### ① 지갑 생성 수 (Total Wallets)
*   **현재 상태:** 5명 (파일 시스템 기준)
*   **데이터 원천:** MongoDB `users` 컬렉션의 문서 개수 (`countDocuments`)
*   **검증:** 새 지갑 생성 (`createWallet`) -> DB Insert -> 즉시 카운트 +1 상승 확인.

### ② 실시간 생성 블록 (Total Blocks)
*   **현재 상태:** 제네시스 블록 + 유저 채굴 활동
*   **데이터 원천:** 실제 블록체인 엔진의 블록 높이 (`BlockChain.length` or DB `blocks` count)
*   **검증:** 마이닝 진행 시 10분(또는 설정된 주기)마다 블록 수 증가 확인.

### ③ 실시간 현재 BW 발행량 (Current Supply)
*   **데이터 원천:** `MiningState` 컬렉션의 모든 유저 `accumulatedReward` 총합 + `BonusRecord` 총합.
*   **연산:** 서버에서 `aggregate` 쿼리를 통해 소수점 50자리 정밀도로 실시간 합산.

### ④ 실시간 BW 잔여 발행량 (Remaining Supply)
*   **공식:** `21,000,000,000 (총 공급량) - 현재 BW 발행량`
*   **특징:** 발행량이 0.00000001이라도 늘어나면, 잔여량은 즉시 그만큼 줄어들어야 함. (오차 허용 0)

### ⑤ 실시간 BW 발행률 (Issuance Rate)
*   **공식:** `(현재 BW 발행량 / 21,000,000,000) * 100` (%)
*   **표기:** 소수점 2자리까지 표시 (예: 0.00%)

### ⑥ 네트워크 연결 상태 (Network Status)
*   **로직:** 클라이언트가 API 서버에 `ping`을 보내 응답이 오면 `🟢 연결됨`, 실패하면 `🔴 연결 끊김`.

## 3. 단계별 작업 계획 (Execution Steps)

### **[Step 1] 백엔드(Server) - 실시간 집계 API 구축**
*   **목표:** 프론트엔드가 요청하면 현재 시스템의 정확한 상태를 0.01초 만에 리턴하는 초고속 API 생성.
*   **파일:** `src/server/routes/stats.ts` (신규 생성)
*   **구현 내용:**
    *   `GET /api/stats/realtime`
    *   MongoDB Aggregation Pipeline을 사용하여 `users`, `miningStates`의 총합을 실시간 계산.

### **[Step 2] 프론트엔드(Client) - 유기적 데이터 바인딩**
*   **목표:** `HomePage.tsx`의 정적인 HTML을 살아있는 컴포넌트로 교체.
*   **파일:** `src/components/HomePage/HomePage.tsx`
*   **구현 내용:**
    *   `useRealTimeStats` 커스텀 훅 생성 (데이터 폴링 및 상태 관리).
    *   숫자 카운팅 애니메이션(CountUp) 효과 적용으로 "살아있음" 시각화.
    *   소수점 8자리까지 끊김 없이 표시.

### **[Step 3] 통합 검증 (Integration Test)**
*   **테스트 시나리오:**
    1.  현재 지갑 수(5) 확인.
    2.  새 브라우저(시크릿 모드)에서 **[지갑 만들기]** 수행.
    3.  메인 화면의 **[지갑 생성 수]**가 5 -> 6으로 실시간 변경되는지 확인.
    4.  마이닝 시작 후 **[발행량]**이 초 단위로 증가하는지 확인.
