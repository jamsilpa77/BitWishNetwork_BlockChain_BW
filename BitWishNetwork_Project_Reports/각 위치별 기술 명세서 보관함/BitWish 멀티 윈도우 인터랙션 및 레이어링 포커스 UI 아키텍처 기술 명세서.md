## 1. 🌐 시스템 개요 (System Overview)
본 명세서는 BitWishNetwork 마이닝 플랫폼의 사용자 경험을 단순한 웹 페이지 수준에서 **'고성능 멀티 태스킹 윈도우 환경'**으로 격상시키기 위해 설계된 프론트엔드 인터랙션 아키텍처를 정의합니다.
사용자가 여러 개의 정보 모달을 동시에 제어하고, 위치를 자유롭게 이동시키며, 작업의 맥락(Context)을 잃지 않고 끊김 없이 데이터를 모니터링할 수 있는 **'레이어드 윈도우 시스템(Layered Window System)'**의 물리적/논리적 구조를 상세히 기록합니다.

## 2. 🖱️ 드래그 인터랙션 엔진 (Draggable Interaction Engine)

### 2.1. 하드웨어 가속 기반 상태 관리
모든 모달 창은 브라우저의 가속 성능을 최대로 활용하도록 설계된 독립적 드래그 엔진을 내장하고 있습니다.
*   **상태 정의**: `position({x, y})`, `isDragging`, `dragOffset` 상태를 통한 위치값 제어.
*   **이벤트 파이프라인**: 
    1.  `mousedown`: 헤더 영역 클릭 시 드래그 세션 개시 및 클릭 좌표와 창 좌표 간의 오프셋 산출.
    2.  `mousemove`: 브라우저 전역(`window`) 이벤트를 구독하여 마우스 이동량을 픽셀 단위로 추적, 창 위치 즉각 업데이트.
    3.  `mouseup`: 세션 종료 및 이벤트 리스너 클린업을 통한 메모리 누수 방지.

### 2.2. 유령 잔상(Ghosting) 원천 차단 알고리즘
창이 열릴 때 이전 위치에서 날아오는 듯한 시각적 노이즈를 제거하기 위한 **'오픈-클로즈 동기화 리셋'** 로직이 적용되었습니다.
*   **2중 방어 메커니즘**:
    - `useEffect`를 통해 `isOpen` 상태 변화 감지 시, 실제 렌더링 직전에 좌표를 중앙(300px, 450px 등)으로 강제 초기화.
    - `transitionProperty`를 `opacity, visibility`로 국한시켜, 좌표 이동(`left`, `top`) 시에는 애니메이션이 개입하지 못하도록 물리적으로 차단하여 100% 즉각적인 응답성 확보.

## 3. 📑 레이어링 및 포커스 거버넌스 (Layering & Focus Governance)

### 3.1. 중앙 집중식 포커스 컨트롤러 (Centralized Focus Controller)
모든 윈도우의 깊이(Depth) 우선순위는 상위 컴포넌트인 `HomePage.tsx`에서 중앙 집중식으로 관리됩니다.
*   **상태 제어**: `activeModal` 문자열 상태를 통해 현재 유저가 인터랙션 중인 모달 식별.
*   **Z-Index 다이내믹 뱅킹**:
    - **Active Window**: `z-index: 10100` (최상위 레이어 격상 및 그림자 강조)
    - **Inactive Window**: `z-index: 10000` (하위 레이어 유지)
*   **포커스 스위칭 트리거**: 모달 내부 어디든 클릭(`onMouseDownCapture`)하거나, 새로운 기능을 실행할 때 상위 `handleFocus` 콜백을 호출하여 즉각적으로 레이어 순위 재조정.

## 4. 🔗 멀티 태스킹 레이어링 UX (Multi-tasking Layering)

### 4.1. 지갑-마이닝 연쇄 계층 구조
사용자가 자산을 관리하면서 동시에 채굴 상태를 확인해야 하는 핵심 여정을 위해 **'영구 레이어링'** 로직을 구축했습니다.
*   **비-폐쇄형 흐름 (Non-Closing Flow)**: 지갑 창에서 '마이닝 시작' 버튼 클릭 시 지갑을 닫지 않고, 마이닝 창을 그 위에 레이어 형태로 추가 렌더링.
*   **시각적 오프셋 설계**: 
    - 지갑 위치: `(Center, 300px)`
    - 마이닝 위치: `(Center, 450px)`
    - 이 기하학적 차이를 통해 두 창이 겹쳤을 때 정보가 상충하지 않고 계단식으로 배열되어 시각적 밀도를 최적화합니다.

## 5. 🚀 시스템 성능 및 보안 (Performance & Security)

*   **독립성 보장**: 모든 드래그 로직과 포커스 연산은 전역 상태 관리자(Redux, Recoil 등) 없이 순수 props 전달과 로컬 상태만으로 구동되어, 컴포넌트 간의 결합도를 0에 수렴시켰습니다.
*   **경계 방어**: 드래그 시 브라우저 뷰포트를 벗어나 창을 잃어버리는 현상을 방지하기 위해, 열기/닫기 시점에 항상 좌표를 안정권으로 수복하는 'Safe-Landing' 로직이 포함되어 있습니다.

본 멀티 윈도우 아키텍처는 단순한 UI 기능을 넘어, BitWish 플랫폼이 추구하는 **'전문적인 금융/채굴 환경'**의 시각적 근간을 형성합니다. 이를 통해 사용자는 모바일 기반의 단편적인 앱 경험을 뛰어넘어, 강력한 멀티 태스킹 능력을 갖춘 데스크톱 수준의 마이닝 환경을 누리게 됩니다.

## 7. 🌐 독립 브라우징 프로세스 및 네임드 윈도우 거버넌스 (Standalone Window Governance)

### 7.1. 네임드 윈도우 컨트롤 (Named Window Control - NWC)
단순한 `_blank` 방식의 팝업 생성을 지양하고, 시스템 전역에서 유일하게 식별되는 윈도우 네임을 부여하여 브라우저 인스턴스를 프로세스 단위로 제어합니다.
*   **고유 식별자 엔진**: 
    - **BW_EXPLORER_WINDOW**: 블록 익스플로러 전용 독립 프로세스 아이디.
    - **BW_RANKING_WINDOW**: 실시간 랭킹 게시판 전용 독립 프로세스 아이디.
*   **기술적 이점**: 동일한 창이 여러 번 열리는 리소스 낭비를 원천 차단하며, 브라우저 메모리 관리 효율을 극대화합니다.

### 7.2. 지능형 포커스 릴레이 (Intelligent Focus Relay)
이미 열려 있는 독립 윈도우를 감지하여 유저의 시선을 즉각적으로 해당 창으로 유도하는 고성능 포커싱 알고리즘을 탑재했습니다.
*   **알고리즘 워크플로우**:
    1.  `window.open` 호출 시 기존 인스턴스 존재 여부 체크.
    2.  존재할 경우, 새로운 창을 생성하지 않고 기존 창의 핸들을 획득.
    3.  `win.focus()` 메서드를 트리거하여 OS 수준에서 해당 브라우저 창을 최상위로 팝업.
*   **UX 가치**: 유저가 수많은 탭 사이에서 길을 잃지 않도록 하며, 마이닝 시스템의 각 기능이 유기적으로 연결된 하나의 운영체제처럼 느껴지게 합니다.

### 7.3. 독립 세션 브릿지 (Independent Session Bridge)
메인 창의 React 생명주기와 완전히 분리된 독립 페이지들이 `localStorage`를 매개체로 하여 실시간 상태 정보를 공유하는 비동기 데이터 브릿지를 구축했습니다.
*   **공유 데이터 무결성**: 
    - **Auth Bridge**: 메인 창에서 인증된 지갑 주소 동기화.
    - **UI Bridge**: 유저가 선택한 언어(KO, EN, JA, ZH) 및 테마(Dark/Light) 상태 실시간 계승.
*   **아키텍처 강점**: 메인 창이 예기치 않게 종료되거나 새로고침되어도, 독립 창들은 각자의 런타임에서 데이터를 유지하며 지속적인 모니터링 기능을 수행합니다.

### 7.4. 마이닝 세션 격리 보호 (Mining Session Isolation)
독립 페이지의 구동은 메인 창의 채굴 연산 프로세스에 단 1%의 사이드 이펙트도 주지 않도록 설계되었습니다.
*   **Zero-Interference**: 독립 창의 데이터 페칭은 별도의 네트워크 커넥션과 메모리 풀을 사용하여, 메인 창의 채굴 해시 속도나 안정성에 간섭하지 않습니다.
*   **Sandboxed Environment**: 독립 창 내의 어떤 오류나 리소스 과부하도 메인 마이닝 창의 생존에 영향을 주지 않는 완벽한 샌드박스 환경을 지향합니다.
