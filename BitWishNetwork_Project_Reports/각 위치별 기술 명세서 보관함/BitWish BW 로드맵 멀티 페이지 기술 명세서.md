체계적으로 기록하였습니다.

BitWish Network (BW) 로드맵 기술 명세서

Technical Specification for BitWish Network Roadmap System

1. 개요 (System Overview)

본 시스템은 BitWish Network(BW) 고정밀 금융 영구 채굴 플랫폼의 비전과 단기/장기 마일스톤을 사용자에게 직관적으로 제시하기 위해
개발된 **독립형 로드맵 페이지(RoadmapPage)**입니다.

플랫폼의 핵심 기술인 50단위 부동소수점 정밀 연산 메커니즘 및 하이브리드 MongoDB 데이터 모델에 대한 신뢰도를 높이기 위해 고급
UI/UX 요소와 다국어 실시간 번역, 관리자 원격 문구 제어 기능이 통합 설계되었습니다.

2. 시스템 아키텍처 및 설계 원칙

2.1 완전 독립성 보장 (State Isolation Rules)

플랫폼 내부의 다른 기능(마이닝 모달, 지갑 모달 등)과의 메모리 및 상태 간섭을 원천 차단하기 위해 다음 원칙을 준수하여 단독 컴포넌트로
빌드되었습니다.

  - 전역 변수/공통 함수 사용 금지: 컴포넌트 내에 캡슐화된 상태(State)와 로컬 함수만을 사용합니다.
  - 상태 공유 배제: Redux, Recoil 등 전역 상태 관리 라이브러리를 사용하지 않고 독자적인 React State 라이프사이클을
    가집니다.
  - 컴포넌트 캡슐화: 스타일시트(.bw-roadmap-page) 범위를 제한하여 홈페이지 및 모달의 CSS 클래스 오염을 차단합니다.

2.2 하이브리드 데이터 동기화 프로토콜

사용자에게 보여지는 로드맵 정보는 다음 순서에 따라 우선순위 동기화(Fallback Chain)가 수행됩니다.

1.  관리자 커스텀 데이터 (BW_CUSTOM_ROADMAP): 관리자 페이지에서 텍스트를 수정하여 로컬 저장소(LocalStorage)에
    저장한 경우, 최우선적으로 해당 데이터를 주입합니다.
2.  다국어 사전 데이터 (LanguageManager): 관리자 지정 값이 없을 경우, 로컬에 준비된 다국어 번역 리소스에서 값을
    가져옵니다.

3. 핵심 기술 명세 (Technical Specifications)

3.1 파일 구성 및 인터페이스 사양

  - 뷰 컴포넌트: RoadmapPage.tsx
  - 스타일 엔진: RoadmapPage.css (다크/라이트 테마 통합 지원)
  - 다국어 디바이스: LanguageManager.ts 연동

3.2 다국어 및 테마 지원 규격 (Multi-language & Theme Matrix)

  - 지원 언어: 한국어(ko), 영어(en), 일본어(ja), 중국어(zh)
  - 동기화 매커니즘:
      - 유저 인터페이스 언어 상태(currentLanguage) 변경 시 로컬 저장소(bw_lang)에 즉시 기록됩니다.
      - 다크 모드 동기화 상태(bw-theme)는 사용자의 시스템 테마 또는 홈화면 설정을 이어받아 .dark 및 .light 클래스로
        자동 분기 렌더링됩니다.

3.3 로드맵 데이터 모델 구조 (BW_CUSTOM_ROADMAP)

관리자 도구 및 다국어 사전에서 로딩되는 데이터 객체의 명세는 다음과 같습니다.

interface RoadmapMilestone {
  title: string;          // 단계별 메인 타이틀
  date: string;           // 예정일 및 마일스톤 일정 안내
  desc: string;           // 세부 개발 요약 설명
  tooltipTitle?: string;  // [1단계 전용] 마우스 호버 시 출력되는 기술 명세 타이틀
  tooltipSubtitle?: string; // [1단계 전용] 서브프레이즈
  tooltipContent?: string; // [1단계 전용] 50단위 부동소수점 연산 기술 명세서 본문
}

interface CustomRoadmapConfig {
  step1: RoadmapMilestone;
  step2: RoadmapMilestone;
  step3: RoadmapMilestone;
  step4: RoadmapMilestone;
  step5: RoadmapMilestone;
  step6: RoadmapMilestone;
}

4. UI/UX 디자인 및 레이아웃 설계

4.1 S자형 벡터 트랙 (S-Curve SVG Vector Track)

컴포넌트의 배경에는 시간의 흐름을 상징하는 구불구불한 S자 형태의 선로(Road)가 SVG 패스로 실시간 렌더링됩니다.

  - 사용 기술: SVG <path> 요소 및 베지에 곡선(Cubic Bezier curve) 좌표 제어.
  - 디자인 사양:
      - 기본 넓은 선폭 테두리 위에 점선(stroke-dasharray="12,12") 스타일의 중앙 차선을 올려 도로 느낌을
        구현하였습니다.
      - 테마 변경에 따라 다크모드는 시안/에메랄드 블루 계열, 라이트모드는 인디고 퍼플 계열의 조명색으로 실시간 동적 매핑됩니다.

4.2 프리미엄 가변 툴팁 (Premium Adjustable Tooltip)

마일스톤 1단계(베타 테스트 출시) 영역에 마우스를 올릴 경우, 내용의 분량에 맞춰 유연하게 리사이징되는 동적 툴팁이 활성화됩니다.

  - 마운트 조건: RoadmapPage.css 내부에서 .roadmap-node-card.first-milestone:hover 가동 시
    CSS 3D 가속 애니메이션을 통해 노출됩니다.
  - 광채 효과 (Neon Glow Filter): 툴팁 외곽에 백그라운드 그라디언트 필터(filter: blur(12px))를 배치하여
    어두운 화면에서도 기술 명세서가 강조되어 보이도록 심미성을 부여하였습니다.
  - 가로/세로 가변성: 내부 텍스트 용량 변화에 유연하게 대응하도록 높이가 자동으로 늘어나는 반응형 구조를 채택하였습니다.

5. 6대 글로벌 마일스톤 상세 정의 (The 6 Milestones)

| **01** | **베타 테스트 공식 출시**       | 플랫폼에 실제 진입하는 최초 관문입니다. 50단위 부동소수점 연산 기술을 설명하는 프리미엄 안내 툴팁이 내장되어 있습니다. |
| **02** | **폐쇄형 메인넷 시범 가동**      | 외부 침입을 차단하는 제네시스 메인넷 기반 모의 마이그레이션 테스트 단계입니다.                         |
| **03** | **글ローバル 인프라 파트너십**     | 오프라인 가맹점 결제 모듈 연계 및 마이닝 엔진 결합을 통한 실질적 유틸리티 확보 단계입니다.                 |

| **05** | **블록 반감기 및 유통 제어 가이드** | 발행량 통제를 통한 가치 보호와 자동 인플레이션 억제 엔진이 가동되는 분기점입니다.                       |
| **06** | **오픈 메인넷 및 2차 백서 공표**  | 크로스체인 지갑 연동, 실제 코인 전송 마이그레이션 적용 및 글로벌 거래소 거래 정식 개시 단계입니다.            |

6. 예외 처리 및 환경 안정성 (Error Handling)

1.  LocalStorage 데이터 유실 시 대처: 관리자 저장 데이터가 깨지거나 비어있는 경우, try-catch 구문을 통해 예외 처리되며
    사용자 화면에는 중단 없이 로컬 다국어 자원(LanguageManager)의 기본 텍스트가 정상적으로 대체 출력됩니다.
2.  반응형 뷰포트 대응 (Responsive Viewport):
      - 모바일 해상도 (768px 이하): S자 도로 레이아웃이 미디어 쿼리에 의해 안전하게 차단되며, 카드와 툴팁이 아래로 길게
        내려가는 1열 스택 형태로 재정렬됩니다. 또한 호버 툴팁은 모바일 환경의 터치 조작 특성을 감안하여 화면 내에 상시
        고정 노출되는 형태로 자동 변환됩니다.

도구(AdminPage) 인터페이스에 준하여 개정되어야 합니다.
