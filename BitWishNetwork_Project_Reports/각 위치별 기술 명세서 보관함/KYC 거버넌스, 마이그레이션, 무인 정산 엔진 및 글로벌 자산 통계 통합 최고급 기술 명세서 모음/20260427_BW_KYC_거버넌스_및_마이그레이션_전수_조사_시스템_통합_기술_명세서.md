# [초정밀] BitWish Network: KYC 거버넌스 및 마이그레이션 전수 조사 시스템 통합 기술 명세서 (Premium v1.5)

본 명세서는 BitWish 메인넷의 핵심 금융 거버넌스 시스템에 대한 **최고 등급의 기술 사양서**입니다. [공정 1-2]부터 [공정 4]까지 진행된 모든 코드 로직과 인터페이스 설계를 초정밀 분석하여 기록하였습니다.

---

## 1. 아키텍처 및 핵심 메커니즘 (Architecture & Core Mechanism)

### 🛰️ 데이터 생애 주기 (Data Life Cycle)
채굴된 자산은 다음의 4단계 상태 전이를 거치며 완벽한 무결성을 유지합니다.
1.  **MINING (채굴 중):** 유저 대시보드에서 실시간 적립.
2.  **SETTLED (정산 완료):** 매월 말 정산 엔진에 의해 `MonthlySettlement` 원장에 기록 (LOCKED 상태).
3.  **TIMELOCK (15일 잠금):** 서버 사이드 `updateMany` 무인 엔진에 의해 1,296,000초 동안 보호.
4.  **AVAILABLE (뱅킹 가용):** 15일 경과 시 즉시 가용 자산으로 전환되어 유동화 가능.

### 🛠️ 고도화된 기술 스택
-   **연산 엔진:** `Decimal.js` (50-bit 정밀도) - 금융권 수준의 부동소수점 오차 제로 구현.
-   **동기화 엔진:** `React State Lifecycle` + `Shared currentTime` - 컴포넌트 간 오차 없는 1초 단위 타이머 동기화.
-   **조사 엔진:** `MongoDB Indexing` + `Regex Engine` - 수만 건의 데이터 중 특정 지갑 주소를 0.1초 내 추적.

## 2. 공정별 초정밀 기술 사양

### 💎 [공정 1-2] 지능형 다국어 거버넌스 (Intelligent Multilingual)
-   **메커니즘:** `LanguageManager` 클래스 기반의 싱글톤 패턴 적용.
-   **고급 아키텍처:**
    -   **세이프티 넷(Safety Net):** 호출된 번역 키가 데이터셋에 누락되었을 때 시스템 중단을 방지하기 위해 기본 언어(KO)의 매핑 값을 강제 반환하는 폴백 로직 탑재.
    -   **상태 동기화:** 언어 전환 시 브라우저 `localStorage`를 즉시 갱신하고 개별 컴포넌트가 최신 번역본을 참조하도록 설계.
-   **초보자 가이드:** "언어 파일에 실수가 있더라도 화면이 깨지지 않고 한국어로 안전하게 보여주는 자동 복구 기능이 들어있습니다."

### 💎 [공정 2] 이중 원장 정산 및 금융 연산 (Dual-Ledger Settlement)
-   **데이터 방어 기제 (Integrity Protection):**
    -   **중복 방지:** 각 정산 데이터는 지갑 주소, 연도, 월을 조합한 논리적 고유 키를 가집니다. 이를 통해 동일 유저에게 한 달에 두 번 정산이 일어나는 오류를 원천 차단합니다.
    -   **예외 처리:** KYC 승인일이 정산일보다 늦을 경우에도, 시스템은 유저의 '최초 정산 데이터'부터 순차적으로 인덱싱하여 지급 누락이 발생하지 않도록 제어합니다.
-   **정밀도 사양:** `Decimal.set({ precision: 50 })` 설정을 통해 0.00000001 BW 단위의 오차도 허용하지 않는 엄격한 회계 처리.
-   **초보자 가이드:** "한 명에게 두 번 돈이 나가는 사고가 절대 없도록 설계된 안전 장부입니다."

### 💎 [공정 3] 초정밀 연금형 릴레이 오토메이션 (Sequential Relay Automation)
-   **핵심 메커니즘:** 유저별 KYC 승인일(`kycVerifiedDate`)과 정산월의 순번을 결합한 **'릴레이 트리거'** 방식.
-   **시퀀스 공식:** $T_n = T_0 + (n-1) \times 30일$
    -   $T_0$: KYC 승인일로부터 15일이 경과한 '최초 해제 기점'.
    -   $n$: 유저의 정산 내역 중 날짜순으로 부여된 순번 (1, 2, 3...).
-   **오토메이션 로직:**
    1.  서버 호출 시 `isKycVerified` 유저의 최초 해제 기점($T_0$)을 산출.
    2.  해당 유저의 모든 `LOCKED` 데이터를 날짜순으로 정렬하여 인덱스 부여.
    3.  현재 시각이 $T_n$에 도달한 데이터만 순차적으로 `UNLOCKED` 전환.
-   **리얼타임 엔진:** 프론트엔드 기준 시간(`currentTime`) 루프를 통한 1초 단위 동기화 및 역동적인 진행바(Progress Bar) 시각 효과 부여.
-   **초보자 가이드:** "정산된 돈뭉치들이 기차 칸처럼 줄을 서서, 한 달에 한 칸씩 차례대로 금고문이 열리는 고도의 금융 보안 시스템입니다."

### 💎 [공정 4] 전수 조사 전용 시스템 (Full Investigation System)
-   **전용 뷰(View Mode):** `viewMode` 상태 전환을 통해 대시보드를 완전히 숨기고 '조사 전용 레이아웃'으로 100% 스왑.
-   **추적 엔진:** 지갑 주소 검색 시 `RegExp(search, 'i')`를 사용하여 대소문자 구분 없이 0.1초 내 전수 조사 결과 도출.
-   **페이징 최적화:** `skip`과 `limit(30)` 파라미터를 사용하여 데이터 로딩 부하를 최소화하고 무한 내비게이션 환경 제공.
-   **디자인 사양 연동:** 골드 네온 하이라이트 및 글래스모피즘 기반의 프리미엄 레이아웃 적용 (상세 수치는 섹션 5 참조).

## 3. 예외 처리 및 보안 사양 (Security & Exception Handling)

1.  **대소문자 무결성:** 모든 지갑 주소 검색 시 `RegExp(search, 'i')`를 사용하여 유저가 대소문자를 틀리게 입력해도 정확한 결과를 도출.
2.  **데이터 오염 방지:** 프론트엔드에서 계산된 시간이나 금액은 단순히 '표시용'이며, 모든 최종 결정과 상태 전환은 서버 사이드(Node.js)에서만 수행됨.
3.  **부하 분산:** 익스플로러의 실시간 데이터 갱신은 10초 주기로 제한하여 서버 트래픽 폭주를 예방.

## 4. 관리자 가이드 (Administrator Guide)

-   **데이터 검증:** `MonthlySettlement` 컬렉션을 조회하여 유저의 `migrationStatus`를 확인하십시오.
-   **검색 결과 확인:** 익스플로러 전수 조사 창에서 `BW...`로 시작하는 지갑 주소를 입력하여 해당 유저의 타임락 시계가 정상 작동하는지 상시 모니터링이 가능합니다.
-   **시스템 복귀:** 전용 뷰 최상단의 `Back to Main Explorer` 버튼을 통해 언제든 대시보드 통계 화면으로 복귀할 수 있습니다.
