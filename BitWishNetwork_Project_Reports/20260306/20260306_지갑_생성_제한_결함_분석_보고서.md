## 1. [장애 현상]
지갑 생성 시 시드 문구 검증 단계에서 **"Wallet creation limit exceeded (Max 2 per IP)"** 팝업이 뜨며 생성이 중단됨.

---

## 2. [기술적 원인 분석]

본인은 지갑 생성의 무분별한 어뷰징을 막기 위해 `WalletService.ts`에 로컬 기반의 생성 제한 로직을 심어두었습니다.

### **(1) 문제의 소스 코드 좌표**
*   **파일 위치:** `BitWishNetwork_MiningSystem/src/services/BlockchainService/WalletService.ts`
*   **핵심 로직:** `checkIpLimit()` 함수 (L517-L521)
*   **제한 방식:** 브라우저 `localStorage`의 `bw_ip_creation_count` 값을 읽어 **2**와 비교함.

### **(2) 상세 로직 분석**
*   지갑이 하나 성공적으로 생성될 때마다 `incrementIpCount()`가 실행되어 숫자를 1씩 올립니다.
*   현재 사용자님의 브라우저 저장소에 이 숫자가 이미 **2**로 기록되어 있어, 시스템이 정상적인 보안 절차로 인식하고 차단한 것입니다.

---

## 3. [즉시 해결 방안]

이 제한을 풀고 자유롭게 테스트하시려면 다음 두 가지 방법 중 하나를 선택해야 합니다.

*   **방법 A (소스 수정):** `WalletService.ts`의 L520 라인에 정의된 `count < 2`를 `count < 100` 등으로 확장.
*   **방법 B (데이터 삭제):** 브라우저 개발자 도구(F12) -> Application -> Local Storage에서 `bw_ip_creation_count` 항목을 수동으로 삭제.
