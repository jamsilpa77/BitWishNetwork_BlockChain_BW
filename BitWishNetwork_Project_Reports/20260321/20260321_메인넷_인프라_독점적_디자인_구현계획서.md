## 1. 벤치마킹 분석 및 비트위시 차별화 전략

사용자님께서 지시하신 참조 사이트들을 세포 단위로 분석한 결과, 기능적 장점을 흡수하되 디자인은 **비트위시 네트워크만의 '프리미엄 메인넷' 감성**으로 전면 재해석하여 구현합니다.

### 📊 [분석 및 독점 디자인 포인트]
1.  **BW 블록 익스플로러 (v.s. Pi Explorer):**
    -   **벤치마킹:** Pi의 카드형 레이아웃과 직관적인 검색 기능을 수용합니다.
    -   **BitWish만의 혁신:** 단순히 숫자를 나열하는 대신, 배경이 투명하게 비치는 **글래스모피즘(Glassmorphism)** 카드와 블록 생성 시 네온 심장박동(Heartbeat) 효과를 적용하여 '살아있는 체인'을 시각화합니다.
2.  **BW 커뮤니티 (v.s. Coinness):**
    -   **벤치마킹:** 카테고리별 직관적인 게시판 구조와 실시간 반응 시스템을 차용합니다.
    -   **BitWish만의 혁신:** 광고성 배너를 배제하고 **Council(의결권)** 시스템을 도입하여, 게시물에 대한 투표가 비트위시 네트워크의 향후 정책에 반영되는 '거버넌스 광장' 형태로 디자인합니다.
3.  **BW 노드 및 메인넷 명세:**
    -   **벤치마킹:** Pi의 단계별 노드 가이드의 구조적 우수성을 참고합니다.
    -   **BitWish만의 혁신:** 3D 노드 맵과 실시간 TPS/Hashrate 터미널 위젯을 삽입하여, 기술적 전문성이 시각적으로 압도되는 디자인을 구현합니다.

## 2. 초정밀 단계별 구현 로직

### [1단계] 상단 네비게이션 호버 메뉴 및 컨트롤 바 최적화
- **작업 위치:** `BitWishNetwork_MiningSystem/src/components/HomePage/HomePage.tsx`
- **로직:**
    - **Mega-Hover:** "BW 메인넷" 호버 시 하위 메뉴(익스플로러, 노드)가 부드러운 Fade-in 효과와 함께 확장되는 메가 메뉴 시스템 구축.
    - **오른쪽 순서:** `[지갑(💼)] -> [언어(🌍)] -> [테마(☀️/🌙)]` 순서를 1px 오차 없이 배치 고정.

### [2단계] 하이브리드 익스플로러 엔진 (ExplorerPage.tsx)
- **작업 위치:** 
    - **UI:** `src/pages/Explorer/ExplorerPage.tsx` 신규 개발.
    - **통신:** `server/routes/stats.ts`의 블록 조회 경로를 `bitwish_network` DB로 브릿지 연결.
- **로직:** 
    - `bwChainCore`가 생성한 최신 20개 블록 리스트를 실시간 폴링(Polling) 방식으로 노출.
    - 검색창에 주소 입력 시 `BitWish-256` 해시 기반의 트랜잭션 기록을 즉시 추적하여 상세 명세 전시.

### [3단계] 거버넌스 커뮤니티 엔진 (CommunityPage.tsx)
- **작업 위치:** 
    - **DB:** MongoDB `community` 컬렉션(Post, Comment, Vote) 설계.
    - **UI:** `src/pages/Community/CommunityPage.tsx`.
- **로직:** 
    - 지갑 인증과 연동하여 본인 확인이 완료된 사용자만 의결권(Vote) 행사가 가능하도록 로직 설계.
    - 댓글 및 대댓글 시스템에 실시간 알림 로직 주입.

## 3. 최종 작업 순위 (Priority)

1.  **P1 (네비게이션 호버 연동):** 지시서의 UI 규칙을 최우선으로 적용하여 홈페이지의 얼굴을 완성합니다.

3.  **P3 (하트비트 블록 생성):** 10초마다 실제 블록이 생성되는 하트비트 로직을 서버에 안착시켜 시스템 기동성을 입증합니다.
