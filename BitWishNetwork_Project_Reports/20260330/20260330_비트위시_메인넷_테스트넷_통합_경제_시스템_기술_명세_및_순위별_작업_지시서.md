## 🛡️ 1. [핵심 실체] 시스템 원천 지갑 및 자산 관리 (BIP-44)

사용자님께서 지시하신 '진짜' 블록체인 지갑과 자산 생성 메커니즘입니다.

### 1.1 BIP-44 결정적 지갑 생성 및 주소 고정
- **기술적 근거:** 재단 마스터 시드를 기반으로 특정 경로(`m/44'/BW'/...`)를 통해 추출하여 시스템 전용 지갑을 생성합니다. (단순 변수가 아닌 실제 지갑 계정)
- **지갑 정체:** 모든 시스템 지갑은 **`BW` + 40자리 16진수** (총 42자리) 형식을 갖는 실제 블록체인 지갑 주소입니다.
- **고정 지갑 목록 (금고):**
    - `GENESIS_MASTER`: **210억 BW (메인넷)** 및 **210,000,000 TBW (테스트넷 2.1억 개)** 원천 발행 및 보관처.
    - `ECOSYSTEM_TREASURY_NET`: 수수료의 **60%**가 물리적으로 자동 적립되는 생태계 지원 금고.
    - `FOUNDATION_TREASURY_NET`: 수수료의 **40%**가 물리적으로 자동 적립되는 재단 운영 금고.

### 1.3 테스트넷(TBW) 원천 격리 시스템 (Isolation)
- **자체 폐쇄 시스템:** TBW는 오직 비트위시 네트워크 내에서만 존재하며, 어떠한 외부 거래소나 메인넷으로의 유출도 불가능하도록 `isTestnetTransfer` 플래그로 원천 차단합니다.
- **시스템화:** `WalletService.ts` 내 송금 로직에서 체인 ID 검증을 필수로 수행하여, TBW와 BW 간의 오송금을 물리적으로 불가능하게 작업합니다.

## ⚖️ 2. [경제 엔진] 수수료 절차 및 발행량 정교화

### 2.1 수수료 60/40 실시간 분기 (Fee Routing)
- **트랜잭션(0.01 BW)** 및 **블록(0.001 BW)** 발생 즉시 인터셉터가 실시간으로 쪼개어 위 지갑으로 전송합니다.
- **익스플로러 투명화:** 모든 분해 내역을 "System Activity"로 표기하여 감춘 자금 이동이 없음을 증명합니다.

### 2.2 반감기 정책 업데이트 (지시서 기반)
- **시작:** 사용자 지시대로 **2027년 1월 1일**부터 시스템 적용, 실제 수치는 2027년 1월부터 **매월 5%씩** 점진적 감소.
- **기술적 산식:** `현재 채굴률 = 0.25 * (1 - (0.05 * 경과개월))` (2027년 1월 기준).
- **코드 수정 위치:** `src/constants/index.ts` 내 `HALVING_START_DATE`를 2027-01-01로 갱신.

## 💧 3. [테스트넷] 수도꼭지(Faucet) 및 UI/UX 자동화

### 🖱️ 1일 3회 100 TBW 지급 시스템

- **코드 위치:** `MyWalletModal.tsx` 내 `header-address-row` div 내부의 `mining-start-btn` 바로 앞에 배치.
- **내역 관리:** `WalletHistory.tsx`의 신설된 **[TBW 거래내역] 탭**으로 자동 연동되어 Tx_Hash와 함께 기록됨.
- **내역 관리:** `WalletHistory.tsx` 연동을 통해 테스트 코인 수령 시 실제 **Tx_Hash**를 포함한 거래 내역이 지갑 목록에 표시됨.
- **제한 로직:** 서버 DB의 `faucet_usage` 컬렉션 연동으로 24시간 내 3회 초과 요청 시 차단 알림 송출.

### 3.3 지갑 전송 버튼 및 거래내역 UI 대개편
- **전송 버튼 재배치 (MyWalletModal.tsx):**
    1. **최상단:** [BW 송금받기] 버튼 유지.
    2. **중간 (기존 송금하기 자리):** [TBW 송금받기] / [TBW 송금하기] 버튼을 상하(Vertical Stack)로 신설.
    3. **최하단:** 기존 [BW 송금하기] 버튼을 최하단으로 이동.
- **거래내역 탭 신설 (MyWalletModal.tsx):**
    - [거래내역] 메뉴 우측에 **[TBW 거래내역]** 메뉴 추가.
    - **목록 구현:** 기존 거래내역과 동일한 디자인으로 구축하되, TBW 전용 데이터만 필터링하여 **YYYY.MM.DD HH:mm:ss** 형식으로 실시간 기록되도록 `WalletHistory.tsx` 로직 확장.
