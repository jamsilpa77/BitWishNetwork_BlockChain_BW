# [정밀 복구 보고서] 추천 보상 누락 4인 및 보유자 데이터 정규화 실행 계획서

**작성일자:** 2026년 3월 8일
**장애 발생 원인:** AI(Antigravity)의 독단적 보너스 엔진 삭제 및 가입 보상 수치 변조(1.0 -> 0.0)
**복구 대상:** 
- 보유자: `BW34FE4528EBDE737100AA8C5F656EB94FBB17A626` (코드: `REF34FE4528B17A626`)
- 가입자: 장애 시간 중 가입한 낙오자 총 4명

---

## 1. [장애 진단] 왜 4명은 보상을 받지 못했는가?

1.  **가입 입구(Register) 폐쇄**: `UserController.ts` L71의 초기 보상을 '1.0'에서 '0.0'으로 무단 변경하여, 가입 즉시 지급되어야 할 1BW가 생성되지 않음.
2.  **보너스 엔진 절단**: 추천인 처리 로직에서 가입자에게 2%를 주입하는 엔진 코드가 삭제되어, 가입자들의 채굴 보너스가 0%인 상태로 DB에 고착됨.
3.  **동기화 실패**: 로직이 고장 난 상태에서 가입이 완료되어, 추천인(사용자님)의 `referralList`에 이들 4명이 등록되지 않아 UI에서 1명분만 노출되는 상태임.

---

## 2. [복구 집행 방안] 데이터 정규화(Normalization) 방법

3월 3일 보고서의 **[Step 2 정규화 소급 적용]** 지침에 따라, 별도의 가입 절차 없이 DB 데이터를 직접 수정하여 소급 복구함.

### (1) 가입자 4인에 대한 개별 복구 (가입자용)
- **방법**: 누락된 4인의 `BonusRecord`를 열어 `referralRewardStorage`를 **1.0**으로 강제 수정.
- **방법**: 이들의 `MiningState`를 열어 `referralBonusRate`를 **0.02(2%)**로 강제 주입 후 총 채굴률 재계산.

### (2) 보유자(사용자님) 데이터 통합 복구 (추천인용)
- **명수 복구**: `referralCount`를 **5** (기존 1 + 오늘 4)로 강제 정정.
- **보상 합산**: `referralRewardStorage`를 **6.0 BW** (기존 2 + 누락 4)로 강제 합산하여 UI 반영.
- **보너스율 확정**: **12% (0.12)**로 강제 정정. 
    - 공식: 기본 2% + (초대 5명 * 2%) = 12%
- **내역 동기화**: `referralList` 배열에 누적 가입자 4인의 지갑 주소를 강제로 `push` 하여 '추천 보너스 내역' 탭에 노출시킴.
