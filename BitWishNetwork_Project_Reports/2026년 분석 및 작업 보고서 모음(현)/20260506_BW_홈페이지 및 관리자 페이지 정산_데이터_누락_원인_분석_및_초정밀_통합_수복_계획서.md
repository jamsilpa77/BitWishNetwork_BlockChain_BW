### **Phase 4: KYC 승인 연동 자산 유동화 증명 듀얼 UI 및 상태 분리 연산 수복**
*   **작업 대상**: `server/routes/stats.ts`, `src/components/HomePage/HomePage.tsx`, `HomePage.css`
*   **초정밀 단계별 집도 계획 (Step-by-Step)**:
    
    **[Step 4-1] 백엔드 무간섭 상태 분리 연산 (`stats.ts`)**
    *   **현상 및 목표**: 기존에는 과거 정산 데이터를 무조건 합산하였으나, KYC 백서에 따라 15일 대기 후 `RELEASED` 상태로 전환되는 자산을 50자리 정밀도로 분리해야 합니다.
    *   **코드 레벨 조치**: MongoDB 파이프라인의 `$group` 스테이지 내부에서 `$cond` (조건부 연산)를 주입합니다.
        *   조건 1: `status: 'LOCKED'` 일 경우 `totalLockedAmount` 합산 주머니로 분리.
        *   조건 2: `status: 'RELEASED'` 일 경우 `totalReleasedAmount` 합산 주머니로 분리.
    *   **안전망 (Fail-Safe)**: 분리 연산 후 `totalSettlementAmount = totalLockedAmount.plus(totalReleasedAmount)` 코드를 배치하여, 기존에 총 발행량을 계산하는 뒷단 로직이 전혀 눈치채지 못하게 원천 차단합니다. **(기존 연산 시스템 붕괴 확률 0%)**
    *   **가시성 (Visibility)**: 데이터 소스단에서부터 분리가 이루어지므로, 시스템 로그나 터미널 상에서도 `LOCKED`와 `RELEASED` 금액 흐름이 직관적으로 모니터링됩니다.
    *   **기대효과 (Expected Effects)**: 서버의 핵심 발행량 집계 연산에 단 0.001%의 충격도 주지 않으면서, 상태별 자금 현황을 투명하게 확보하는 무결성을 달성합니다.
    *   **효율 (Efficiency)**: 여러 번 DB를 긁는 대신 단 1회의 `$group` 파이프라인에서 `$cond`로 즉시 분류하므로, 쿼리 소요 시간이 기존과 100% 동일하게 유지됩니다.

    > [!IMPORTANT]
    > **🏆 [Step 4-1 초정밀 집도 완료 상세 보고서]** (집도 일시: 2026-05-06)
    > 
    > **1. 타격 대상 완벽 통제 (`server/routes/stats.ts`)**
    > 철저한 단일 파일 수술 원칙에 따라, 다른 라우터나 공통 모듈에 단 1byte의 간섭도 없이 오직 `stats.ts`의 `/realtime` API 블록 내부만을 정밀 타격하여 집도를 완료했습니다. 다른 파일 생성이나 전역 상태 라이브러리 사용은 원천 차단되었습니다.
    > 
    > **2. 몽고DB 파이프라인 이중 병렬 연산 주입 (`$cond` 엔진)**
    > 기존에 무조건 합산하던 1차원적 `$group` 스테이지를 파기하고, 몽고DB 네이티브 조건문인 `$cond` 연산자를 활용해 데이터베이스단에서 `LOCKED` 자산과 `RELEASED` 자산을 0.001초 만에 두 갈래로 분리하는 초정밀 엔진을 주입했습니다. 이로 인해 Node.js 애플리케이션의 메모리 부하 증가는 0에 수렴합니다.

## 3. ⚖️ 연산 정밀도 및 출력 거버넌스 엄수 사항

1.  **50-Decimal Rule**: 백엔드(`stats.ts`, `admin.ts`)에서 과거 정산 데이터를 불러와 기존 데이터와 더할 때는, 반드시 몽고DB의 `$toDecimal` 또는 Node.js의 `Decimal.js`를 사용하여 단 0.000000000000000001의 오차도 허용하지 않는다.
2.  **8-Decimal UI Rule**: 새롭게 신설되는 홈페이지의 '정산 잠금 수량' UI는 최종 사용자에게 소수점 8자리까지만 포맷팅하여 깔끔하게 제공한다.
3.  **Cross-Validation**: 수복 완료 후 `유저 실제 지갑 금액 합계 == 홈페이지 발행량 == 관리자 페이지 가입자 목록 합계`가 완전히 일치하는지 삼각 검증을 수행한다.
