# 블록 생성 수수료 실시간 카운팅 연동 작업 계획서

블록 생성 시 0.001 BW 수수료가 `network_stats` DB에 정상 적립되고 있으나, 프론트엔드 UI가 하드코딩 `0.0000`으로 표시되는 문제를 수정한다.
거래 수수료(P2P)는 본 작업 범위에서 제외한다.

---

## Proposed Changes

### 백엔드 API

#### [MODIFY] [stats.ts](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/server/routes/stats.ts)

`/api/stats/realtime` 라우트에 `network_stats` 컬렉션 조회를 추가하여 기금 데이터를 응답에 포함시킨다.

**추가 내용 (기존 응답 `data` 객체 안에 3개 필드 추가):**
```typescript
// 기존 코드의 nativeClient 연결 재활용하여 network_stats 조회
const fundStats = await networkDb.collection('network_stats').findOne({ id: 'global_fund_stats' });

// 응답 data에 추가:
blockCreationFee: fundStats?.totalAccumulatedFees || '0',
ecosystemFund: fundStats?.ecosystemFund || '0',
foundationFund: fundStats?.foundationFund || '0',
```

### 프론트엔드 UI

#### [MODIFY] [ExplorerPage.tsx](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/src/components/Explorer/ExplorerPage.tsx)

**변경 1: state 추가** (L53 부근)
```typescript
const [fundStats, setFundStats] = useState({
    blockCreationFee: '0', ecosystemFund: '0', foundationFund: '0'
});
```

**변경 2: fetchBlocks 내 데이터 수신** (L98 부근, statsResponse 처리 내)
```typescript
if (statsResponse.data.data.blockCreationFee) {
    setFundStats({
        blockCreationFee: statsResponse.data.data.blockCreationFee,
        ecosystemFund: statsResponse.data.data.ecosystemFund,
        foundationFund: statsResponse.data.data.foundationFund,
    });
}
```

**변경 3: 하드코딩 제거 + UI 바인딩** (L332-350)

현재 구조:
```
블록 생성 수수료: 0.0000 BW  (하드코딩)
[주황색 바]
총 누적 기금 합계: 0.00000000 BW  (하드코딩)
```

변경 후 구조:
```
블록 생성 수수료: {실시간 값} BW  ← API 바인딩
[주황색 바 - 동적 width]
  ┌─────────────────────┬──────────────────────┐
  │ 생태계 기금 (60%)    │ 재단 운영비 (40%)     │  ← 신규 서브 행
  │ {실시간 값} BW       │ {실시간 값} BW        │
  └─────────────────────┴──────────────────────┘
총 누적 기금 합계: {실시간 값} BW  ← API 바인딩
```

> [!IMPORTANT]
> 거래 수수료(파란색 바)는 현재 P2P 거래가 없으므로 `0.0000 BW` 표시를 유지하되, 추후 P2P 수수료 구현 시 동일 패턴으로 연동 가능하도록 구조만 준비해 둔다.
