# [배포/이전 매뉴얼] BitWishNetwork 헤츠너(Hetzner) 서버 이전 및 배포 실행 매뉴얼

본 매뉴얼은 기존 Vultr 서버 등에서 헤츠너(Hetzner) 신규 서버로 프로젝트를 성공적으로 이전하고 실행하기 위한 모든 필수 명령어와 백업/복구 절차를 담고 있습니다.

---

## 1. 헤츠너 신규 서버 초기 환경 구축

헤츠너 Linux(Ubuntu 22.04 LTS 권장) 서버 접속 후 아래 명령어를 순서대로 실행하여 필수 런타임을 구성합니다.

```bash
# 1) 패키지 업데이트
sudo apt update && sudo apt upgrade -y

# 2) Node.js LTS (v18 또는 v20) 설치
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 3) PM2 프로세스 매니저 및 Webpack 전역 설치
sudo npm install -g pm2 webpack webpack-cli

# 4) MongoDB (데이터베이스) 설치
sudo apt-get install gnupg curl
curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | \
   sudo gpg -o /usr/share/keyrings/mongodb-server-keyring.gpg \
   --dearmor
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-keyring.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
sudo apt-get update
sudo apt-get install -y mongodb-org

# 5) MongoDB 서비스 시작 및 자동 실행 등록
sudo systemctl start mongod
sudo systemctl enable mongod
```

## 2. 데이터베이스 & 설정 파일(.env) 백업 및 복구 방법

회원 정보와 마이닝 내역이 담긴 중요 데이터베이스 및 환경 변수(.env) 백업/복구 절차입니다.

### 1) 주요 파일 및 데이터 저장 위치
* **설정 파일 (.env)**: `/BitWishNetwork_MiningSystem/.env`
* **마이닝 및 가입 데이터베이스**: MongoDB 내 `bitwish_network` 데이타베이스

### 2) 기존 서버에서 데이터 백업 (Export)
기존 서버 터미널에서 다음 명령어를 실행하여 압축 아카이브 형태로 데이터를 추출합니다.

```bash
# DB 백업 (bitwish_network 데이터베이스 전체 백업)
mongodump --db bitwish_network --out /root/db_backup/

# 백업 폴더 압축
tar -zcvf bitwish_db_backup.tar.gz /root/db_backup/
```
* 압축 완료 후 `bitwish_db_backup.tar.gz` 파일과 `BitWishNetwork_MiningSystem/.env` 파일을 `scp`나 FileZilla 등을 이용하여 로컬 컴퓨터나 신규 헤츠너 서버로 전송합니다.

### 3) 신규 헤츠너 서버에서 데이터 복구 (Import)
신규 서버로 백업 파일을 옮긴 후 아래 명령어로 압축을 해제하고 복구를 진행합니다.

```bash
# 백업 파일 압축 해제
tar -zxvf bitwish_db_backup.tar.gz

# DB 복구 수행
mongorestore --db bitwish_network /root/db_backup/bitwish_network/
```

## 3. 프로젝트 빌드 및 실행 명령어 (헤츠너 배포)

신규 서버에서 소스 코드를 내려받은 후 구동하는 릴리즈 명령어 모음입니다.

```bash
# 1) 프로젝트 디렉토리 이동
cd /path/to/BitWishNetwork_MiningSystem

# 2) 백업해둔 .env 파일 배치
# (이 위치에 기존 .env 파일을 그대로 업로드하거나 재생성합니다)

# 3) 의존성 라이브러리 설치
npm install

# 4) 프론트엔드 Webpack 빌드 (배포용 파일 생성)
npm run build

# 5) PM2를 사용한 백엔드 서버 백그라운드 무중단 실행
# 포트 5001 충돌을 방지하기 위해 기존 프로세스가 있다면 종료 후 시작합니다.
pm2 start server/index.ts --name "bitwish-backend" --interpreter ts-node

# 6) 서비스 포트 방화벽 개방 (5000: 프론트엔드 서비스, 5001: 백엔드 API)
sudo ufw allow 5000/tcp
sudo ufw allow 5001/tcp
sudo ufw allow 27017/tcp   # MongoDB 외부 관리 툴 접속 시 개방 (평소엔 닫아두는 것을 권장)
```

## 4. 모니터링 및 문제 해결 명령어
* **실시간 서버 구동 상태 확인**: `pm2 status`
* **실시간 에러/로그 출력**: `pm2 logs`
* **백엔드 서버 재시작**: `pm2 restart bitwish-backend`
* **Nginx 재구동 (포워딩 설정 변경 시)**: `sudo systemctl restart nginx`
