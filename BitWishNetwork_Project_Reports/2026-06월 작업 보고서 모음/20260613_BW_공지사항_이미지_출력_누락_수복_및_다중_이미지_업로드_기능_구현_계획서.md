## 1. 현재 문제점 분석 및 원인 파악

### ① 로컬/실서버 공지사항 이미지 출력 누락 원인
* **원인 위치:** [BWCommunityWindow.tsx](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/src/components/BWCommunityWindow/BWCommunityWindow.tsx) (프론트엔드 상세 뷰)
* **상세 분석:** 
  * 관리자 도구(`AdminPage.tsx`)에서 공지사항 등록 시 Base64 이미지 텍스트가 백엔드 API를 거쳐 DB(`BWCommunityPost`의 `image` 필드)에 정상 저장 및 조회가 완료됩니다.
  * 그러나 커뮤니티 프론트엔드 화면의 **게시글 상세 보기 영역(라인 738~756)**에서 저장된 `selectedPost.image` 데이터를 렌더링해 주는 `<img>` 태그가 완전히 누락되어 있어 화면에 그림이 나타나지 않은 것입니다. (UI 렌더링 누락 결함)

### ② 대용량 이미지 등록 시 서버 통신 실패 원인
* **원인 위치:** 실서버 Nginx 설정 파일 (`/etc/nginx/sites-available/default`)
* **상세 분석:** 
  * Nginx의 기본 바디 제한(`client_max_body_size`)이 `1MB`로 기입 생략되어 있어 고화질 이미지를 Base64로 변환할 때 1MB를 초과하면 Nginx가 백엔드로 가기 전에 `413 Payload Too Large` 에러로 차단하기 때문입니다.

## 2. 수복 및 고도화(다중 이미지 업로드) 구현 계획

개당 2MB 이하의 이미지 파일 최대 10개까지 첨부 및 출력이 가능하도록 데이터베이스 모델, API, 프론트엔드 UI/UX를 다음과 같이 전면 개조합니다.

```
[구현 아키텍처 및 데이터 흐름]
관리자 페이지 (파일 10개 멀티 업로드 및 용량 2MB 제한 검증)
   ↓ (Base64 인코딩 배열 전송)
Nginx 프록시 (client_max_body_size 30M 허용)
   ↓
백엔드 API (배열 개수 검증 && images DB 저장)
   ↓
리액트 프론트엔드 (selectedPost.images 배열 순회하며 이미지 출력)
```

## 3. 세부 작업 공정 (계획)

### [공정 1] 데이터베이스 모델 개조 (Mongoose Schema)
* **수정 파일:** [bwCommunityModels.ts](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/server/models/bwCommunityModels.ts)
* **변경 사양:** 
  * 기존 단일 이미지 보관용 `image: { type: String }` 필드를 다중 이미지 보관용 **`images: [{ type: String }]` 배열 스키마**로 개편합니다.

### [공정 2] Nginx 인프라 용량 제한 해제 설정
* **수정 대상:** 실서버 `/etc/nginx/sites-available/default`
* **변경 사양:** 
  * 최대 10장의 이미지(2MB × 10 = 약 20MB, 인코딩 고려 약 27MB)를 안전하게 통과시키기 위해 Nginx 설정 내 `server` 블록에 아래 설정을 삽입하고 Nginx를 리로드합니다.
  ```nginx
  client_max_body_size 30M;
  ```

### [공정 3] 백엔드 API 보강 및 유효성 검증
* **수정 파일:** [bw_community_api.ts](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/server/routes/bw_community_api.ts)
* **변경 사양:**
  * `POST /api/admin/notices` 및 `PUT /api/admin/notices/:id` 라우터가 `images` 배열을 받도록 수정합니다.
  * **서버 검증 추가:** 
    * 업로드 개수 검증: `images.length > 10` 일 경우 차단 에러 반환.
    * 용량 초과 검증: 수신한 Base64 배열 데이터 중 개당 크기가 2MB를 상회하는 패턴이 존재하는지 실시간 정밀 필터링 후 위반 시 업로드 거부.

### [공정 4] 관리자 페이지 이미지 관리 UI 고도화
* **수정 파일:** [AdminPage.tsx](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/src/components/AdminPage/AdminPage.tsx)
* **변경 사양:**
  * 첨부파일 입력 폼에 `multiple` 속성을 추가하여 다중 선택을 가능하게 합니다.
  * **클라이언트 사전 검증:** 사용자가 이미지를 선택할 때 파일별 용량(`file.size`)이 `2,097,152 bytes` (2MB) 이하인지 확인하여 초과 시 즉각 경고창을 띄우고 첨부를 제외합니다.
  * 업로드된 이미지들의 미리보기(썸네일 그리드) 및 개별 삭제 버튼을 추가하여 편의성을 제고합니다.

### [공정 5] 커뮤니티 상세 화면 이미지 출력 적용
