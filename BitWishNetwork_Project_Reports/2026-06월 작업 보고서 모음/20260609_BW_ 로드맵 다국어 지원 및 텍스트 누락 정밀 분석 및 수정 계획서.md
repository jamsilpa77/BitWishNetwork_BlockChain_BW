## 2. 다국어 (4개국어) 변경 미작동 원인 분석

### ❌ 원인: 단일 언어(한국어) 구조 저장 및 강제 오버라이드
관리자 페이지에서 로드맵을 수정할 때 다국어별 입력창 없이 **단일 텍스트(한글)**로만 로컬스토리지에 저장하고 있으며, 로드맵 페이지에서는 다국어 설정과 관계없이 이 단일 텍스트를 무조건 화면에 우선 표기하고 있습니다.

* **로드맵 페이지 렌더링 방식 (`RoadmapPage.tsx:75-80`)**:
  ```typescript
  const fetchVal = (step: string, field: string, defaultValue: string): string => {
      // 로컬스토리지(BW_CUSTOM_ROADMAP)에 데이터가 존재하면 다국어 기본값(defaultValue)을 무시하고 반환
      if (customRoadmap && customRoadmap[step] && customRoadmap[step][field]) {
          return customRoadmap[step][field];
      }
      return defaultValue;
  };
  ```
  관리자 페이지에서 한국어로 문구를 수정 및 저장하면 `customRoadmap`에 한글 데이터만 저장됩니다. 이후 유저가 로드맵 페이지에서 **EN, JA, ZH 버튼**을 눌러 `currentLanguage`를 바꾸더라도, `fetchVal`이 다국어 번역본인 `defaultValue` 대신 로컬스토리지에 저장된 **한글 텍스트를 강제로 리턴**하기 때문에 번역이 작동하지 않게 됩니다.

이 문제를 근본적으로 해결하기 위해 다음의 2단계 패치를 제안합니다.

### [Phase 1] 관리자 폼 초기 텍스트 정합성 맞추기
* [AdminPage.tsx](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/src/components/AdminPage/AdminPage.tsx)의 `roadmapForm` 초기 상태값을 [LanguageManager.ts](file:///c:/BitWishNetwork_BlockChainMainnet/BitWishNetwork_MiningSystem/src/utils/LanguageManager/LanguageManager.ts) 내 한국어(`ko.roadmap`) 원본과 100% 동일하게 확장 수정합니다.

### [Phase 2] 로드맵 데이터 모델의 다국어(ko/en/ja/zh) 지원 구조 변경
* **관리자 페이지 수정**:
  * 전광판 메시지 수정 창처럼 **언어 선택 탭(KO, EN, JA, ZH)**을 추가하여 관리자가 4개 국어별로 로드맵을 각각 입력할 수 있도록 수정합니다.
  * 데이터 저장 포맷을 `{ step1: { title: { ko: "...", en: "..." } } }` 형태로 확장하여 `BW_CUSTOM_ROADMAP`에 저장합니다.
* **로드맵 페이지 수정**:
  * `fetchVal` 함수에 현재 선택된 언어(`currentLanguage`) 인자를 추가하여 해당 국가의 커스텀 텍스트를 우선적으로 가져오도록 수정합니다:
    `fetchVal('step1', 'title', currentLanguage, getTxt('roadmap.step1.title'))`
