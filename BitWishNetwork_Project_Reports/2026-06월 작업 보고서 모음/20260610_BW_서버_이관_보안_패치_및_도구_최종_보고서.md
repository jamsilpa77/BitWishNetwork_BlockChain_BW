# 🛡️ BitWish Network 서버 이관, 보안 패치 및 도구 최종 보고서 (2026-06-10)

본 보고서는 BitWish Network의 성공적인 실서버 이관, 보안 강화 작업, 일시 장애 긴급 복구 경위, 그리고 이번 공정에 활용된 각종 원격 진단/배포 도구들의 기능과 향후 편리한 서비스 업데이트를 위한 원클릭 자동화 매뉴얼을 집대성한 통합 최종 보고서입니다.

## 1. 금일 작업 및 배포 내역 요약

### 1) 🔐 실서버 보안 고도화 패치 완료
실서버 내 소스코드 유출 및 무단 접속을 원천 차단하기 위해 아래와 같은 다중 보안 체계를 수립하였습니다.
* **Nginx HTTP 보안 헤더 적용:** 클릭재킹, XSS 공격, MIME Sniffing을 방어하기 위한 보안 헤더 (`X-Frame-Options`, `X-Content-Type-Options`, `X-XSS-Protection`, `Strict-Transport-Security` 등)를 웹서버 설정에 삽입했습니다.
* **민감 파일 외부 접근 차단:** 브라우저를 통해 `.env`, `.git`, `tsconfig.json`, `package.json` 및 `server/` 내 TypeScript 원본 소스코드에 다이렉트로 접속하지 못하도록 Nginx와 Express 미들웨어 이중으로 접근을 완벽히 차단했습니다. (시도 시 404 Not Found 강제 반환)
* **방화벽(UFW) 강화 및 포트 차단:** 내부 API 통신용 포트인 `5001` 및 `4001` 포트에 대한 외부 직접 접근을 방화벽에서 거부(`DENY`) 처리하였습니다.
* **무단 침입 방어(Fail2ban) 적용:** 외부 세력의 SSH 무차별 대입 공격(Brute Force)을 탐지하고 차단하는 Fail2ban 서비스를 설치 및 활성화했습니다.

### 2) 🛠️ 일시 장애(500 Error) 원인 분석 및 해결
* **원인:** Nginx 설정 파일을 새로 고치는 과정에서 내부 변수인 `$uri`, `$host` 앞의 달러(`$`) 기호에 백슬래시(`\`)가 자동으로 이스케이프 처리되며 오작동을 유발, Nginx 엔진이 파일 경로를 못 찾아 무한 내부 리다이렉션에 빠진 결함이었습니다.
* **해결:** 리눅스 쉘 변수 확장 간섭이 배제되는 `cat << 'EOF'` 기법을 적용하여 온전한 설정 파일로 복구 후 Nginx 서비스를 정상 재기동하였습니다.

### 3) 🐛 지갑 연결(Restore) 잠재적 버그 원인 분석 및 해결
* **원인:** MongoDB 8.0 데이터베이스 스키마에서 유저 IP 주소(`ipAddress`)를 필수값(`required: true`)으로 요구하나, 유저가 기존 24단어를 입력해 지갑을 연결(불러오기)할 때는 프론트엔드 API 요청에 IP가 누락되어 DB 정합성 검증 에러로 지갑 불러오기가 실패하는 현상이 존재했습니다.
* **해결:** 백엔드 컨트롤러(`UserController.ts`)의 회원가입/등록 로직에서 `ipAddress`가 누락되더라도 실제 유저의 접속 헤더 정보(`x-forwarded-for` 또는 Express `req.ip`)에서 접속 IP를 자동 추출해 매핑하는 폴백 코드를 적용하여 안전하게 해결했습니다.

| 도구 파일명 | 위치 | 역할 및 기능 |
| :--- | :--- | :--- |

| **`migrate_db.js`** | `scratch/` | 로컬 MongoDB에 쌓여있던 정합성 자산 데이터(17개 지갑 주소, 누적 채굴 기록, 144개 체인 블록 및 트랜잭션 내역)를 실서버 MongoDB 데이터베이스로 완벽하게 복사해 준 **데이터베이스 이관기**입니다. |

| **`walkthrough.md`** | `artifacts/` | Vultr 서버 환경 구축, MongoDB 설치, 포트 충돌 해결, PM2 무중단 구동, 그리고 DB 마이그레이션 결과 수치를 상세하게 수록해 둔 **작업 공정 완수 보고서**입니다. |

## 3. 향후 유지보수를 위한 간편 배포 도구 (Continuous Deployment)

향후 개발을 진행하고 업데이트를 편리하게 진행할 수 있도록 영구 장착한 원클릭 자동 배포 가이드입니다.

### 1) 📂 스마트 원격 배포기 (`deploy_backend.js`)
* **위치:** `BitWishNetwork_MiningSystem/scripts/deploy_backend.js`
* **역할:** 실서버에 코드를 Git Pull 하고, 프론트엔드(`src/`)에 수정 사항이 있을 때만 웹팩 빌드를 수행(약 2분 소요)하고 백엔드 로직만 수정된 경우는 즉시 빌드를 건너뛰어 초고속으로 백엔드를 무중단 재기동해 줍니다.

### 2) 📂 원클릭 실행 런처 (`deploy.bat`)
* **위치:** 프로젝트 최상위 루트 폴더 (`c:\BitWishNetwork_BlockChainMainnet\deploy.bat`)
* **역할:** 로컬 소스코드를 깃허브에 자동으로 커밋/푸시하고, 서버에 원격 로그인하여 위 `deploy_backend.js`를 즉각 실행해 주는 **관리자 PC용 원클릭 자동화 실행 파일**입니다.
* **사용법:** 로컬 포트 서버에서 작업 완료 후 이 파일을 **더블 클릭**만 하시면 됩니다.
