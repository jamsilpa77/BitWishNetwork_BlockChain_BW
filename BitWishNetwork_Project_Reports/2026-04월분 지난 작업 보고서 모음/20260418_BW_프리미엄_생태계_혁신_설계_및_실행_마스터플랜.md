# 📊 BitWishNetwork 시스템 개편 및 마스터 실행 계획서 (2026.04.18)

본 보고서는 복잡한 기술 용어와 거창한 표현을 싹 걷어내고, 사용자님께서 지적하신 4가지 핵심 개선 사항을 **누구나 이해하기 쉽고 직관적으로** 정리한 문서입니다. 각 항목별로 **문제점, 개선 방안, 실제 작업할 파일 위치**를 알아보기 쉽게 쪼개어 분리했습니다.

## 1. 익스플로러 화면 UI 전면 개편 (파이코인 차별화)

### 1-1. 파이(Pi) 핵심 기능(Metrics, Operations, Transactions)의 프리미엄 UI 재설계

#### [문제점]
현재 빗위시 네트워크에는 파이 익스플로러를 지탱하는 핵심 3대 기능(`Mainnet metrics`, `Latest Operations`, `Latest Transactions`)이 아예 누락되어 있습니다. 또한, 이 기능들을 파이코인과 똑같이 딱딱하고 복잡한 텍스트(Hash) 리스트 형태로 그대로 복사해 오면 일반 사용자들은 복잡함에 지쳐 즉각 이탈해 버리는 치명적 문제가 존재합니다.

#### [개선 및 구현 방안 (파이코인 압살 UI 개편)]
*   **[1] Mainnet Metrics (메인넷 생태계 지표창 구축):**
    *   단순히 블록 높이 등 숫자만 나열하는 파이코인식 표를 버립니다. 익스플로러 화면 상단에 **'메인넷 종합 대시보드 위젯'**을 만들어, ① 총 유통된 코인량, ② 타임락 무인 대기 코인량, ③ 개설된 나의 지갑(Wallet) 등록 수 등을 **주식 앱(토스증권 등)의 상단 요약 지수창**처럼 아주 심플한 타일(Tile) 디자인으로 시각화합니다.
    **[1] Mainnet Metrics (메인넷 생태계 지표창 구축) 작업 완료 상세 보고서:**
        *   **정밀 타겟 타격 파일:** `ExplorerPage.tsx`, `ExplorerPage.css`, `LanguageManager.ts` (계획 밖의 다른 파일 무단 침범 절대 없음)
        *   **UI 구조 대개편 (결과):** 파이 익스플로러의 투박한 숫자 나열을 폐기하고, 토스증권 뱅킹형 프리미엄 타일 위젯 5개를 이식했습니다.
        *   **특수 시각화 (소각 뷰):** 비전과 가치 방어의 핵심인 [🔥 소각 가치 타일]을 최상단 하이라이트로 구성하고 붉은빛의 Shine 빛 반사 네온 애니메이션을 부여하여 신뢰를 극대화했습니다.
        *   **절대 규칙 [4개국어 연동]:** `LanguageManager.ts`의 한국어/영어/일본어/중국어 사전 영역에 5가지 신규 메트릭스 번역 키를 추가하여, 하드코딩 없이 `t('explorer.xxx')`로 전면 동작하게 만들었습니다.
        *   **절대 규칙 [다크모드 완벽 연동]:** 일반 모드의 밝고 산뜻한 Glassmorphism뿐 아니라 전역 다크모드 진입 시 심연의 고급스러운 네온 투명도가 즉각 살아나도록 CSS 분리 설계를 완벽히 적용했습니다.

*   **[2] Latest Operations (최근 네트워크 활동 타임라인 구축):**
    *   파이코인식의 개발자용 언어(Call/Contract)가 섞인 텍스트 기록을 모조리 폐기합니다. SNS 피드(페이스북, 카카오스토리) 형태의 **'모듈형 타임라인 세로 UI'**로 바꿉니다. `"🛡️ 어드민 다중서명을 통해 펀드 송금이 완료됨"`, `"👤 무인 봇이 새로운 KYC 유저에게 코인을 입금함"` 처럼 누구나 읽기 편한 한글 문구와 아이콘이 달린 직관적인 카드형 리스트로 교체합니다. (View All 클릭 시 모달창으로 열림)
    **[2] Latest Operations (최근 네트워크 활동 타임라인 구축) 작업 완료 상세 보고서:**
        *   **정밀 타겟 타격 파일:** `ExplorerPage.tsx`, `ExplorerPage.css`, `LanguageManager.ts` (계획 이탈 파일 0%)
        *   **UI/UX 구조 개편 (결과):** 불친절하고 어지러운 텍스트 해시 로그(Call/Contract)를 전면 폐기하고, 토스나 페이스북과 동일한 친화적이고 직관적인 [SNS 모듈형 세로 타임라인 피드 UI]를 완벽하게 이식했습니다.
        *   **메시지 친화력 향상:** "🛡️ 펀드 송금", "👤 KYC 봇 입금", "🔥 수수료 자동 소각" 등 누구나 읽기 쉬운 명시적 아이콘(Emoji) 기반 친화적 직관형 카드로 재설계했습니다.
        *   **절대 규칙 [4개국어 연동]:** 신규 창설된 생태계 활동 메시지들과 '전체 보기' 버튼 등을 `LanguageManager.ts` 내 4개국어(ko, en, ja, zh) 사전에 구조 훼손 없이 완벽히 주입하여, `t()` 훅으로 하드코딩의 흔적을 말끔히 지웠습니다.
        *   **절대 규칙 [다크모드 완벽 연동]:** 타임라인 카드가 전역 다크 모드 속으로 들어올 경우, 배경은 눈의 피로를 최소화하는 묵직한 반투명 색상으로 전환되며 카드 좌측 보더 엣지(Edge)에 스포트라이트 컬러가 살아나는 고급 CSS 렌더링을 적용 수행했습니다.

*   **[3] Latest Transactions (빗위시 뱅킹 영수증 스타일 이체 내역 구축):**
    *   복잡한 트랜잭션 해시(TxHash) 목록을 완전히 걷어냅니다. 은행 송금 확인증처럼 `[보낸 유저 앞 6자리...뒤 6자리] ➡️ 💸 10.00 BW 전송 ➡️ [받은 유저 앞 6자리...뒤 6자리]` 형태의 **확실하고 깔끔한 영수증 레이아웃**으로 재창조합니다. 거래 주체에 대한 시각적 안도감을 완벽히 제공하면서도, 불필요하게 긴 전체 해시나 블록 증명 데이터는 유저가 **'상세보기(클릭)'했을 때만 아래로 펼쳐지는(아코디언 형태) UI** 공간 속에 숨겨서 처리합니다. (View All 클릭 시 전체 거래 풀페이지 이동)
    **[3] Latest Transactions (빗위시 뱅킹 영수증 스타일 이체 내역 구축) 작업 완료 상세 보고서:**
        *   **정밀 타겟 타격 파일:** `ExplorerPage.tsx`, `ExplorerPage.css`, `LanguageManager.ts` (구조적 손상 0%)
        *   **UI 구조 대혁명 (결과):** 파이코인의 딱딱한 개발자용 테이블(Table) 형식을 영구 폐기하고, 토스/카카오뱅크 스타일의 [프리미엄 뱅킹 영수증 카드 UI]로 전면 교체 완료했습니다.
        *   **직관적 주소 포맷팅:** 모든 지갑 주소는 사용자님의 지시대로 [앞 6자리...뒤 6자리]만 노출하여 시각적 피로도를 낮추고 뱅킹 앱과 같은 안도감을 부여했습니다.
        *   **아코디언 디테일 엔진:** 트랜잭션 해시, 블록 높이, 수수료 등 복잡한 데이터는 평소 숨겨두었다가 클릭 시 부드럽게 펼쳐지는 [아코디언 상세 보기] 시스템을 구축했습니다.
        *   **절대 규칙 [4개국어 연동]:** '출금', '입금', '상세 보기', '시스템 기반(Coinbase)' 등 모든 UI 텍스트를 4개국어 사전에 정밀 동기화하여 하드코딩을 원천 차단했습니다.
        *   **절대 규칙 [다크모드 완벽 연동]:** 영수증 카드가 다크 모드 속에서 우아한 네이비-블랙 톤으로 전환되며, 세밀한 그림자와 보더 라인을 통해 프리미엄 감성을 유지하도록 설계했습니다.
        *   **최종 보정:** 메인넷 메트릭스 구역에서 발생했던 210억 유통량 숫자 오버플로우 현상을 긴급 수술하여, 모든 디스플레이 환경에서 '칸에 딱 맞는 정갈함'을 확보하며 프로젝트를 완수했습니다.
    **[4] 차세대 익스플로러 신뢰도 강화 (실제 할당량 수치화 및 지갑 생성 수 동기화) 작업 보고:**
        *   **핵심 철학:** 홈페이지의 '포인트용 채굴 현황'을 배제하고, 익스플로러만의 본질인 '공식 백서 할당량(Tokenomics Allocation)'을 최상단에 배치하여 시스템의 투명성과 신뢰도를 100%로 끌어올림.
        *   **작업 완료 목표:** 5대 핵심 할당량(재단, 생태계, 개인채굴자 등) 정적 주입 및 실시간 지갑 생성 수(Wallet Count) 데이터 스트림 연동.

    **[5] 토큰이코노미 절대 진실 구현 및 실시간 지갑 연동 분석 보고서:**
        *   **분석 (진실 vs 포인트):** 홈페이지의 채굴량은 KYC 전 '포인트' 개념이므로, 익스플로러에서는 이를 과감히 걷어내고 빗위시 네트워크의 근간이 되는 **공식 할당 수량**을 노출함으로써 익스플로러의 본질인 '팩트 체크' 기능을 강화함.
        *   **구현 레이아웃 (세로 행 시스템):** 기존 가로형 그리드 타일을 폐기하고, 사용자가 위에서 아래로 읽으며 생태계 규모를 한눈에 파악할 수 있도록 **세로 행(Vertical Rows) 기반의 프리미엄 리스트 UI**로 전면 재설계. (불필요한 순번 숫자는 제외하여 심미성 극대화)
        *   **데이터 연동 로직:** 
            1. BW코인 총 발행량 (21,000,000,000 BW) - 백서 기준 고정값.
            2. 재단/운영팀 할당량 (4,200,000,000 BW) - 백서 기준 고정값.
            3. 생태계 할당 수량 80% (16,800,000,000 BW) - 백서 기준 고정값.
            4. 개인채굴자 할당량 65% (13,650,000,000 BW) - 백서 기준 고정값.
            5. 파트너/가맹/사업자 할당량 15% (3,150,000,000 BW) - 백서 기준 고정값.
            6. **지갑 생성 수 (Dynamic)** - 홈페이지 오픈 API 및 DB 연동을 통해 실제 생성된 지갑 수를 1초 단위로 실시간 카운팅하여 출력.
        *   **작업 타격 위치:** `ExplorerPage.tsx` (UI 뼈대), `ExplorerPage.css` (세로 행 프리미엄 디자인), `LanguageManager.ts` (명칭 보정 및 4개국어 완벽 대응).

    **[5] 토큰이코노미 절대 진실 구현 및 실시간 지갑 연동 작업 완료 상세 보고서:**
        *   **절대적 진실의 데이터 주입:** 홈페이지의 유동적인 '포인트' 수치를 배제하고, 백서 공인 수치(`210억`, `42억`, `168억`, `136.5억`, `31.5억`)를 익스플로러 심장에 고정 주입하여 시스템의 권위를 확보함.
        *   **실시간 지갑 동기화 엔진:** "지갑 생성 수" 항목은 홈페이지 실제 데이터 소스(`/api/stats/realtime`)와 1초 단위로 직결하여, 신규 지갑 생성 시 익스플로러에서도 즉각 카운팅되도록 완벽 구현함.
        *   **프리미엄 세로 행(Vertical) UI:** 기존의 파이 조각 같은 그리드 방식을 폐기하고, 위에서 아래로 정갈하게 자산 가치가 읽히는 **세로형 리스트 레이아웃**으로 전면 개편하여 심미성을 극대화함.
        *   **4개국어 명칭 보정:** "등록된 지갑 수"를 사용자님 지시대로 **"지갑 생성 수"**로 4개 국어(KO, EN, JA, ZH) 모두 정밀 교체 완료하여 글로벌 신뢰도를 확보함.

#### [작업 파일 위치]
`src/components/Explorer/ExplorerPage.tsx` 화면 컴포넌트 파일을 열람하여, 내부 화면의 뼈대를 `메트릭스 구역(Grid UI)`, `오퍼레이션 구역(Flex UI)`, `트랜잭션 구역(Flex UI)` 크게 3단의 프리미엄 레이아웃으로 화면을 분할 코딩합니다.
