# [분석 보고서] 시스템 전반 다국어 시스템 결함 진단 및 UI 개선 계획서

**보고 일자:** 2026-04-16
**상태:** 분석 완료 및 작업 대기

---

## 1. 다국어(i18n) 시스템 정밀 분석 결과

### [결함 1] 보안 인증 모달 (보안 유예 만료 등)
*   **원인:** `MnemonicAuthModal.tsx`에서 `LanguageManager`를 컴포넌트 본문 내부에서 매번 생성함. 리액트의 특성상 언어 프롭스(`currentLanguage`)가 들어와도 첫 렌더링 시에는 기본값인 한국어('ko') 인스턴스가 사용되어 번역이 지연되거나 누락됨.
*   **해결책:** `LanguageManager`를 싱글톤(Singleton)으로 관리하거나, `useMemo`를 통해 인스턴스 생성을 최적화하고 언어 변경 시 즉각적으로 `setLanguage`가 반영되도록 보정해야 함.

### [결함 2] 나의 지갑 (Dashboard & Tabs)
*   **원인:** 
    1. `LanguageManager.ts` 내에 데이터 불일치: `ko` 섹션에는 존재하는 `actions`(송금, KYC 등) 키가 `en`, `ja`, `zh` 섹션에는 통째로 누락됨. 이로 인해 화면에 키값(`wallet.dashboard.actions.receive` 등)이 그대로 노출됨.
    2. 다수의 하드코딩: "잔액", "추천 보너스", "채굴 보상 내역" 제목과 표 헤더(가입자, 날짜 등)가 번역 함수를 거치지 않고 한국어 텍스트로 직접 작성되어 있음.
*   **해결책:** 번역 데이터 전수 조사 및 4개국어 키 매칭 완료. 하드코딩된 모든 JSX 코드를 `getTranslation` 함수로 치환.

## 2. 나의 지갑 UI/UX 개선 분석

### [조치 A] 불필요 기능 제거
*   **대상:** '2차 비번 초기화' 버튼.
*   **사유:** 현재 시스템 아키텍처상 더 이상 필요하지 않은 중복 기능임.

### [조치 B] 마이닝 시작 버튼 고도화
*   **변경안:** 
    1. 위치: 기존 '2차 비번 초기화' 버튼이 있던 우측 상단 핵심 영역으로 이동.
    2. 디자인: 폰트 크기 확대, `background: linear-gradient` 등을 적용하여 강조.
    3. 색상: 가시성이 뛰어난 형광 녹색 계열 또는 골드 계열로 배경색 지정.

## 3. 정밀 작업 계획 및 단계별 로드맵 (Priority)

### [1단계] 번역 데이터(LanguageData) 전수 동기화
*   **작업:** `LanguageManager.ts`를 열어 한국어 기준으로 모든 키를 `en`, `ja`, `zh` 섹션에 복사 및 실제 번역 내용 기입.
*   **주의:** 키 이름 오타 발생 시 다시 키값이 노출될 수 있으므로 정밀 매칭 확인.

### [2단계] MyWalletModal 구조 수술 (UI 및 번역)
*   **작업:** 
    1. UI: '2차 비번 초기화' 코드 제거 및 '마이닝 시작' 버튼 CSS 고도화.
    2. 번역: 하드코딩된 약 20여 군데의 한국어 텍스트를 번역 키로 교체.
*   **예측 오류:** 버튼 위치 변경 시 플렉스(Flex) 레이아웃이 깨질 수 있으므로 절대 규칙에 따른 독립적 디자인 적용 필요.

### [3단계] 보안 인증 및 내역 페이지(Tabs) 완결
*   **작업:** `MnemonicAuthModal`, `MiningRewards`, `ReferralRewards` 내의 하드코딩 문자열 전수 교체.
*   **고급 기능:** 날짜 형식(`YYYY.MM.DD`) 표시 로직을 현재 언어 세팅에 맞게 멀티 포맷팅 처리.

## 4. 작업 중 발생 가능한 오류 예측 및 방어 전략
1.  **JSON 데이터 누락 오류**: 신규 추가된 키가 `en`에만 있고 `ja`에 없으면 일본어 환경에서 에러 발생 가능. -> `getAllLanguageTexts` 검증 로직으로 방어.
2.  **상태 업데이트 지연**: 언어 변경 버튼 클릭 시 즉시 화면이 안 변하는 현상. -> `useEffect` 의존성 배열에 `currentLanguage`를 명확히 선언하여 강제 렌더링.
3.  **레이아웃 뒤틀림**: 버튼 크기를 키울 경우 모바일이나 작은 화면에서 헤더 영역이 겹칠 수 있음. -> CSS `flex-wrap` 및 미디어 쿼리(선택적) 고려.
