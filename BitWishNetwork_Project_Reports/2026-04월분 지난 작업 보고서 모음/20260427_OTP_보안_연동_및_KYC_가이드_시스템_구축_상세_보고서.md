# [20260427] OTP 보안 연동 및 KYC 가이드 시스템 구축 상세 보고서

## 1. 개요 및 목적
본 보고서는 BitWishNetwork 지갑 시스템의 보안 무결성을 확보하기 위해 **KYC 인증 상태와 구글 OTP(2FA) 설정을 유기적으로 연동**하고, 사용자에게 직관적인 안내를 제공하는 가이드 시스템 구축 계획을 수립하는 데 목적이 있음.

## 2. 관리자 요구사항 분석 (상세 이해)
관리자 지시 사항에 의거하여, 비트위시 네트워크의 보안 순환 구조와 사용자 가이드 시스템을 다음과 같이 정의함:

### 2.1 핵심 거버넌스 철학: "신뢰 기반 보안 파이프라인"
비트위시의 자산 이동은 반드시 **[KYC 승인] → [OTP 설정] → [송금 활성화]**의 순차적 경로를 따라야 함. 이는 자산 가치가 확정된 유저에게만 최상위 보안(2FA) 권한을 부여하여 시스템 리소스를 최적화하고 금융 무결성을 확보하는 정책임.

### 2.2 KYC 상태별 동적 가이드 및 UI 레이아웃 정책
1.  **KYC 승인자 (15일 마이그레이션 등록 상태):**
    *   [OTP 설정] 버튼 클릭 시, 중간 가이드 없이 **즉시 OTP 설정 엔진(QR 코드 생성 및 6자리 검증창)**으로 연결.
2.  **KYC 미승인자 (미인증 또는 심사 중):**
    *   [OTP 설정] 버튼 클릭 시, 브라우저 기본 `alert` 대신 **전용 커스텀 UI(Small Message Window)**를 호출함.
    *   **배치(Positioning):** 나의 지갑 모달 내 [OTP 설정] 버튼과 [KYC 인증] 버튼 사이의 **정중앙 공간**에 작고 고급스러운 메시지 창을 팝업.
    *   **출력 메시지:** "KYC승인 후 다시 진행 하세요." (다국어 관리자를 통한 4개국어 대응)
    *   **액션:** 메시지 하단 [확인] 버튼 배치. 클릭 시 가이드 창이 닫히며 자연스럽게 [KYC 인증] 버튼으로 유저의 시선을 유도함.

### 2.3 기술적 마감 표준 (Supreme Standard)
*   **UI 독립성:** 가이드 창은 독립된 상태값(`showGuideWindow`)으로 관리하며, 버튼 사이의 레이아웃을 해치지 않는 Absolute Positioning 기술 적용.
*   **시각적 무결성:** 비트위시 전용 글래스모피즘 테마를 적용하여, 단순한 알림이 아닌 시스템의 일부로 느껴지는 프리미엄 디자인 고수.
