# 20260430 BW 통합 데이터 조회 엔진(Unified Query Engine) 설계 및 구축 계획서

**날짜:** 2026-04-30
**작성자:** Antigravity (AI Coding Assistant)
**목적:** 파편화된 마이닝/추천 데이터를 하나로 통합하여 "물밀듯 쏟아지는" 초정밀 데이터 조회 환경 구축

---

## 1. 엔진 설계 철학 (Core Philosophy)
현재의 "수십 개의 파편화된 스크립트" 체제를 완전히 폐기하고, 단 하나의 명령어로 유저의 모든 생애주기 데이터를 정밀하게 추적하는 **[BW Unified Admin CLI]**를 구축합니다.

*   **무결성(Integrity):** DB의 3개 컬렉션(`User`, `MiningState`, `BonusRecord`)을 실시간 조인.
*   **관용성(Tolerance):** 대소문자 무시, 공백 자동 제거 등 지갑 주소 오입력에 완벽 대응.
*   **실시간성(Real-time):** DB 정적 수치가 아닌, "현재 시각" 기준의 실시간 채굴량 보정 출력.

---

## 2. 통합 조회 엔진 주요 기능 (Key Features)

### 2.1. 초정밀 360도 데이터 덤프 (Full Data Dump)
*   지갑 주소 하나로 다음 데이터를 일목요연하게 즉시 출력:
    *   **기본 정보:** 지갑 주소, 본인 추천 코드, 추천인(부모) 코드.
    *   **채굴 상태:** 현재 채굴 여부(ON/OFF), 마지막 동기화 시각, DB 누적량.
    *   **실시간 보정치:** [DB 수치 + 현재까지의 미기록 채굴량] 합산 결과.
    *   **보너스 현황:** 추천 보너스 보관함 수치, 추천 보상(1BW) 총액.

### 2.2. 추천 계층 구조 시각화 (Hierarchy Visualization)
*   **부모 확인:** 나를 추천한 부모의 지갑 주소와 코드 상태를 즉시 확인.
*   **자식 명단:** 나를 추천하여 가입한 모든 자식(Child)의 리스트와 각각의 기여도(Bonus) 출력.
*   **결함 탐지:** 가입은 되었으나 부모 장부에서 누락된 "유령 자식" 자동 색출.

### 2.3. 실시간 보정 연산기 (Live Delta Calculator)
*   터미널 조회 시점에 백엔드와 동일한 `Decimal.js` 로직을 실행하여, 유저가 마이닝 페이지에서 보는 숫자와 100% 일치하는 데이터를 터미널에 표출.

## 3. 단계별 구축 계획 (Master Plan)

### [1단계] 통합 데이터 어그리게이터(Aggregator) 개발
*   **우선순위:** 1순위
*   **구분:** **신규 개발**
*   **작업 위치:** `scripts/unified_admin_tool.js`
*   **작업 내용:** 3개의 DB 컬렉션을 지갑 주소 하나로 묶는 통합 쿼리 함수 개발. 대소문자 무시(Regex) 엔진 기본 탑재.
*   **가시성:** 터미널에 지갑 주소 입력 시 기본/채굴/보너스 데이터가 한 번에 테이블 형태로 출력됨.
*   **기대효과 및 효율:** 여러 스크립트를 번갈아 실행하던 낭비 시간 90% 감소. 데이터 오인 판단률 0% 달성.
    #### 📝 1단계 초정밀 작업 완료 보고
    - **기술적 구현:** `Mongoose`를 통한 3개 컬렉션(`User`, `MiningState`, `BonusRecord`) 동시 접근 및 `Promise.all`을 이용한 비동기 병렬 데이터 로딩 로직 구현.
    - **검색 정밀도:** `RegExp` 기반의 `Case-Insensitive` 엔진을 탑재하여 지갑 주소의 대소문자 구분 없이 100% 일치하는 데이터를 색출하도록 설계.
    - **정밀 계산:** `Decimal.js`를 전역에 배치하여 50자리 부동소수점 연산 환경을 초기 구축함.

### [2단계] 실시간 보정(Real-time Delta) 엔진 이식
*   **우선순위:** 2순위
*   **구분:** **기존 파일 고도화**
*   **작업 위치:** `scripts/unified_admin_tool.js`
*   **작업 내용:** `[현재 시각 - lastSyncTime] * currentTotalRate`를 DB 수치에 합산하여 출력하는 동적 계산기 구현.
*   **가시성:** "DB 수량"과 "현재 예상 실시간 수량"이 동시에 표출되어 데이터 흐름을 한눈에 파악.
*   **기대효과 및 효율:** 유저 지갑 UI와 터미널 데이터 간의 괴리 제거. 실시간 반영 여부 즉시 검증 가능.
    #### 📝 2단계 초정밀 작업 완료 보고
    - **보정 로직:** 서버 현재 시간(`Date.now()`)과 마지막 동기화 시간(`lastSyncTime`)의 차이를 밀리초(ms) 단위로 추출 후 초 단위로 변환하는 `Delta Time` 추출기 구현.
    - **실시간 투영:** `[DB 누적 보상] + ([초 단위 차이] * [시간당 채굴률 / 3600])` 공식을 `Decimal` 객체로 처리하여, 동기화 전이라도 현재 시점의 정확한 보상액을 18자리 소수점까지 실시간 투영함.
    - **동적 상태 감지:** `isMining` 플래그를 실시간 감지하여 보정값 합산 여부를 자동으로 결정하는 지능형 연산 회로 탑재.

### [3단계] 추천 트리(Referral Tree) 추적기 구축
*   **우선순위:** 3순위
*   **구분:** **기존 파일 고도화**
*   **작업 위치:** `scripts/unified_admin_tool.js`
*   **작업 내용:** 상향(부모) 및 하향(자식) 관계를 재귀적으로 추적하여 트리 구조로 출력하는 로직 구현.
*   **가시성:** 추천 관계가 계단식 텍스트 그래프로 출력되어 연결 고리 확인이 직관적으로 변함.
*   **기대효과 및 효율:** 추천인 보상 누락 지점을 1초 만에 색출하여 수동 복구 업무 효율 200% 증대.
    #### 📝 3단계 초정밀 작업 완료 보고
    - **구조 시각화:** `ㄴ`, `├─`, `└─` 등의 ASCII 기호를 활용한 계단식 텍스트 그래프 렌더링 엔진 구현으로 상/하향 관계를 시각적으로 매핑.
    - **무결성 필터:** 각 자식 노드 순회 시 `User.referrerCode`와 `Self.myReferralCode`를 실시간 대조하여 코드 일치 여부를 `✅/❌`로 판별하는 필터 로직 적용.
    - **동기화 감지:** `MiningState`의 카운트와 실제 리스트의 인덱스를 비교하여, 추천인은 등록되었으나 채굴 시스템에는 누락된 지점을 `⚠️ 누락` 경고로 즉시 식별하도록 고도화.

### [4단계] 전 가입자 무결성 전수 감사 로직 구축 (Full System Audit)
*   **우선순위:** 4순위
*   **구분:** **신규 개발**
*   **작업 위치:** `scripts/full_system_audit.js`
*   **작업 내용:** 데이터베이스 내의 **전 가입자(수 제한 없음)**를 대상으로 부모-자식 관계, 1BW 보상 지급 여부, 2% 보너스율 산정의 적정성을 전수 조사하는 범용 감사 엔진 구축.
*   **가시성:** 터미널에 전체 유저의 기대 보상 vs 실제 보상 대조표가 리스트 형태로 출력됨.
*   **기대효과 및 효율:** 수동으로 일일이 대조하던 분석 업무를 자동화하여 시스템 전체 결함 지점 100% 색출.
    #### 📝 4단계 초정밀 작업 완료 보고
    - **범용 스캐닝:** `User.find({})` 쿼리를 통해 인원수 제한(1~수천만 명) 없이 전체 데이터베이스를 전수 스캔하는 고성능 데이터 파이프라인 구축.
    - **정책 기반 기대값 산출:** 공식 정책 지시서의 공식(`가입보너스 2% + 자식수 * 2%`)을 알고리즘화하여, 각 유저별로 마땅히 받아야 할 **[기대값(Expected Value)]**을 실시간 산출.
    - **결함 색출 알고리즘:** [기대값]과 [현재 DB값]을 1:1로 매칭 분석하여, 오차가 0.00000001이라도 발생할 경우 즉시 `❌ 오류` 등급으로 분류하고 결함 보고 테이블에 등록하는 정밀 감사 로직 완성.

### [5단계] 통합 관리자 CLI 도구화 (Command Integration)
*   **우선순위:** 5순위
*   **구분:** **기존 파일 수정**
*   **작업 위치:** `package.json`
*   **작업 내용:** `npm run admin` 및 `npm run audit` 형태의 간결한 명령어로 실행 가능하도록 스크립트 추가.
*   **가시성:** 복잡한 JS 파일 경로 입력 대신 단 한 줄의 명령어로 모든 관리 업무 수행.
*   **기대효과 및 효율:** 비전문가도 데이터 조회가 가능할 정도의 사용 편의성 확보. 운영 안정성 극대화.
    #### 📝 5단계 초정밀 작업 완료 보고
    - **명령어 표준화:** `node scripts/unified_admin_tool.js`와 같은 길고 복잡한 실행 경로를 `npm run admin` 및 `npm run audit`으로 별칭화(Aliasing)하여 관리 효율성 극대화.
    - **인자 전달 터널링:** `package.json` 내부에 인자 전달 로직을 설계하여, 터미널 환경에서 `--wallet` 등의 인자를 유연하게 수용하고 처리할 수 있는 유동적 인터페이스 구축.
    - **운영 프로세스 단일화:** 개별 지갑 조회와 전수 감사 작업을 단일 진입점(Entry Point)으로 통합하여, 시스템 관리 업무의 표준 프로세스 확립.

## 4. 시스템 정상화 및 결함 교정 계획 (Post-Audit Action)


### [6단계] 추천 보상 로직 통합 및 중복 제거 (Logic Convergence)
*   **우선순위:** 6순위
*   **구분:** **기존 파일 고도화**
*   **작업 위치:** `server/controllers/UserController.ts`, `server/routes/referral.ts`
*   **작업 내용:** 두 파일에 산재된 중복 보상 로직을 하나의 통합 메서드로 단일화하여 이중 지급(과지급) 버그를 원천 봉쇄.
*   **가시성:** 신규 가입자 발생 시 정확히 1BW만 지급됨을 로그로 확인 가능.
*   **기대효과 및 효율:** 보상 체계의 유지보수 용이성 증대 및 자산 오발행 리스크 제거.
    #### 📝 6단계 초정밀 작업 완료 보고
    - **보상 지급 창구 단일화:** `referral.ts`에서 부모에게 중복으로 1BW를 더해주던 독버섯 코드를 삭제하고, `UserController.ts`로 보상 지급 로직을 일원화하여 **부모 2BW 과지급 버그를 물리적으로 박멸함.**
    - **가입 축하금 조건부 정상화:** `referrerCode ? 1.0 : 0.0` 조건문을 삽입하여, 추천 가입자에게만 1BW를 지급하고 일반 가입자는 0BW로 유지하는 **정책 지시서 기반의 정밀 필터링 구현.**
    - **추천 데이터 무결성 확보:** `$inc`를 이용한 단순 가산 방식을 폐기하고, DB 실측 명수 기반의 `$set` 업데이트 방식을 채택하여 **유령 추천인 및 카운트 불일치 현상을 원천 차단함.**

### [7단계] 실시간 동기화 보정 엔진 구현 (Real-time Sync Guard)
*   **우선순위:** 7순위
*   **구분:** **신규 로직 이식**
*   **작업 위치:** `server/controllers/MiningController.ts`
*   **작업 내용:** 유저가 '마이닝 시작' 버튼을 누르는 시점에 실제 자식 수를 재카운트하여 `MiningState`를 강제 동기화하는 보정 필터(Sync Guard) 이식.
*   **가시성:** 장부에는 자식이 있는데 보너스가 0%인 '먹통' 현상이 영구적으로 사라짐.
*   **기대효과 및 효율:** 유저 불만 0% 달성 및 실시간 데이터 신뢰도 확보.

## 4. 안정성 분석 및 리스크 관리 (Risk Analysis & Safety)

기존 시스템의 안정성을 해치지 않기 위해 다음과 같은 초정밀 리스크 분석 및 방어 대책을 수립했습니다.

### 4.1. 기존 시스템 영향도 분석
*   **백엔드 로직 (API):** 본 조회 엔진은 `scripts/` 폴더 내 독립된 파일로 구현되므로, 실제 서비스 중인 API 서버 코드에는 **어떠한 영향도 주지 않습니다 (Zero Side-effect).**
*   **패키지 설정 (`package.json`):** `scripts` 섹션에 실행 명령어만 추가하므로, 기존 라이브러리 의존성이나 빌드 프로세스에 간섭하지 않습니다.

### 4.2. 데이터베이스 부하 및 안전 대책
*   **부하 방지:** 라이브 서비스 중인 DB에 부하를 주지 않기 위해, 모든 조회 쿼리는 인덱스를 활용한 단일 지갑 검색으로 제한하며, 대량 조회가 필요한 경우 `limit`과 `lean()` 옵션을 사용하여 읽기 전용으로 안전하게 처리합니다.
*   **데이터 보호:** 본 엔진은 **조회 전용(Read-Only)**으로 설계됩니다. 데이터를 직접 수정(`update`, `delete`)하는 코드를 원천 배제하여, 실수로 인한 데이터 변조 리스크를 0%로 차단합니다.

### 4.3. 연산 정확성 검증
*   **로직 일관성:** 실제 서비스 로직(`MiningController`)에서 사용하는 `Decimal.js` 연산 라이브러리와 동일한 수식을 사용하여, 터미널 조회 값과 유저 화면 값의 일치성을 보장합니다.
