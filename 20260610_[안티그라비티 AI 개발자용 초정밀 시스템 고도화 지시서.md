### [안티그라비티 AI 개발자용: 초정밀 시스템 고도화 지시서]

**[Role & Objective]**
당신은 BitWish Network의 전문 풀스택 개발자입니다. 아래 요구사항은 시스템의 무결성과 보안, UX 최적화를 위한 필수 기능입니다. 유추하거나 축소하지 말고, 모든 기능적 요구사항을 100% 명시된 로직대로 구현하십시오.

**[1. 보안 및 다계정 방지]**

* **IP 제한:** 사용자 계정 생성 시 서버는 클라이언트의 IP 주소를 획득하여 DB 내 `ip_address` 필드와 대조합니다. 동일 IP가 이미 3개의 지갑을 보유 중일 경우, 생성 요청을 즉시 차단하고 "IP당 최대 지갑 생성 개수를 초과하였습니다"라는 오류 메시지를 출력하십시오.

**[2. 크롬 확장프로그램 (상세 명세)]**

* **실시간 연동:** 홈페이지의 [BitWish 총 공급량 (210억개), 실시간 현재 BW 발행량, 실시간 BW잔여 발행량] 데이터를 실시간으로 크롤링하거나  API로 호출하여 팝업 내 카드형 UI에 출력하십시오.
* **출석 보너스(5%):**
* 상태값(출석여부)을 DB에서 불러와 버튼 UI를 동적으로 렌더링하십시오.
* 출석 미완료: 빨간색 그라데이션 버튼 / 클릭 시 출석 처리 + 24시간 타이머 시작.
* 출석 완료: 녹색 그라데이션 버튼 / 24시간 동안 보너스 적용 상태 유지.
* 0%~5% 보너스 적용 상태를 팝업 상단에 실시간 텍스트로 표시하십시오.

* **마이닝 데이터 연동 섹션:**
* '마이닝 페이지'의 핵심 데이터 4종을 팝업창 내 고급스러운 별도 카드 레이아웃으로 출력.
* 표시 항목: 1) 실시간 누적 보상, 2) 진행 시간, 3) 추천 보너스 %, 4) 추천 보너스 보관함.
* 데이터 업데이트: 웹사이트와 동일하게 실시간으로 동기화되어 카운팅되도록 구현.

* **니모닉 기반 30% 부스트:**
* 팝업 내 '지갑 인증' 섹션에서 24단어 니모닉 입력란 구현.
* 인증 성공 시: 즉시 `bonus_multiplier`를 1.1배로 계산하여 마이닝 페이지의 [기본 보상률, 누적 보상]에 즉시 반영.
* 설치/제거 감지: 확장프로그램 삭제 시 해당 브라우저 세션의 보너스 적용을 즉시 회수하고 DB에 즉시 업데이트하십시오.


* **설치 유도 배너:** 홈페이지 메인 공지창 바로 밑에 1번 스샷의 노란 영역을 지정하고, 눈에 띄는 디자인(애니메이션이 포함된 배너/아이콘)으로 확장프로그램 설치 페이지로 연결하십시오.

**[3. 커뮤니티 및 게시판 UI/UX]**

* **회원가입:** 비밀번호 입력 필드 바로 아래에 '비밀번호 확인' 필드를 추가하고, 불일치 시 가입 버튼 비활성화.
* **닉네임:** 닉네임 입력 시 [중복확인] 버튼을 강제하고, DB 쿼리를 통해 사용 가능 여부를 즉시 피드백하십시오.
* **네비게이션:** 로그인/가입/커뮤니티 상세 페이지를 포함한 모든 Depth에 [이전으로(뒤로가기)] 버튼을 상단에 필수 배치하십시오.
* **게시글 기능:** 글 수정(Edit) 버튼 추가, 작성자 본인만 수정/삭제 가능하도록 권한 설정.
* **이모지 로직:** 조회수(눈 아이콘)는 페이지 로드 시 마다 증가, 좋아요(하트)는 사용자당 1회 클릭 시에만 고유 카운트 증가.

**[4. 관리자 페이지 개편 ([bitwishnetwork.com/bitwish/testadmin](https://www.google.com/search?q=https%3A%2F%2Fbitwishnetwork.com%2Fbitwish%2Ftestadmin))]**

* **메뉴 구조:**
1. [가맹점 관리] 메뉴를 [KYC 관리] 메뉴 하위 호버 항목으로 이동.
2. '가맹점 관리'가 있던 자리에 [BW 커뮤니티 관리] 메뉴 신설.
3. 기존 [가입자 목록] 메뉴를 [추천 보상 현황] 대시보드 내부로 통합.


* **관리자 공지 관리:**
* 공지 작성/수정/삭제 기능을 구현하고, 업로드된 사진/스크린샷이 글에 포함되도록 구성.
* 공지 목록: 10개 단위 페이징(이전/다음 버튼), 전체 선택 및 개별 삭제 체크박스 구현.


* **금칙어 필터링:**
* [BW 커뮤니티 관리] 메뉴 내 [금칙어 관리] 섹션 추가.
* 관리자가 금칙어 입력 후 저장 시, 사이트 전역(게시판, 댓글)에 해당 단어 사용 불가 로직(정규식 처리) 실시간 적용.

1. 사용자 경험을 최우선으로 하되, 보안이 필요한 데이터(니모닉 등)는 로컬 스토리지에 평문 저장하지 마십시오.
2. 요구사항 중 하나라도 구현이 모호할 경우, 임의로 판단하지 말고 반드시 다시 질문하십시오.
3. 각 기능을 별도의 모듈 파일로 나누어 작성하십시오.
